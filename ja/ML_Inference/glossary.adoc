= 用語集: 機械学習推論

本用語集は、機械学習推論チュートリアルシリーズ全体で使われる主要用語の定義を提供します。

[glossary]
Activation Function（活性化関数）::
ニューラルネットワークレイヤの出力に適用される数学関数で、非線形性を導入します。これによりネットワークは複雑なパターンを学習できます。ReLU、Sigmoid、Tanh などが一般的な例です。

AI-based Upscaling（AI ベースのアップスケーリング）::
機械学習を使って低解像度で画像をレンダリングし、知的に高解像度へアップスケールする技術（DLSS や FSR など）。性能を向上させながら視覚品質を維持します。

Artificial Neural Network (ANN)（人工ニューラルネットワーク）::
生物の脳の構造と機能に触発された計算モデルで、情報をレイヤで処理する相互接続された「ニューロン」から成ります。

Backpropagation（逆伝播）::
学習フェーズで損失関数のネットワーク重みに対する勾配を計算し、誤差を減らす方向に重みを更新するために使われるアルゴリズムです。

Batch size（バッチサイズ）::
単一パスでまとめて処理する学習例または推論リクエストの数。大きいバッチサイズはスループットを改善できますが、メモリ使用量とレイテンシを増やします。

Bias（バイアス）::
活性化関数をシフトさせ、モデルが根底のデータによりよくフィットできるようにする、ニューラルネットワークレイヤの追加パラメータです。

Channel Ordering（チャネル順序）::
テンソルデータのメモリレイアウト。「Channels Last」（NHWC）はピクセル色をまとめて格納（RGBRGB...）し、「Channels First」（NCHW）は一チャネル分の値をすべて格納してから次へ（RRR...GGG...BBB...）格納します。

Compute Shader（コンピュートシェーダ）::
Vulkan のシェーダステージで、GPU 上の汎用並列計算に使われ、カスタム機械学習推論の実装に不可欠です。

Convolution（畳み込み）::
多くのニューラルネットワーク（特に画像処理）の基本演算で、小さな行列（カーネル）を入力上でスライドさせ、エッジやテクスチャのような特徴を抽出します。

Denoising（デノイジング）::
機械学習モデルを使って画像からノイズを除去するプロセス。特にリアルタイムレイトレーシングで、少ないサンプル数でもクリーンな結果を得るのに有用です。

Element-Wise Operation（要素ごとの演算）::
テンソルの各要素に独立して関数を適用する演算。活性化関数が典型的な例です。

Feature Extractor（特徴抽出器）::
入力データのパターン（エッジ、テクスチャ、形状）を識別するニューラルネットワークの初期部分。転移学習では、事前学習済みモデルの特徴抽出器がしばしば再利用されます。

Fine-tuning（ファインチューニング）::
事前学習済みモデルを新しい小さなデータセットでさらに学習し、特定タスクに適応させる転移学習技法です。

FlatBuffers::
効率的なクロスプラットフォームシリアライズライブラリ（TFLite が使用）。別途パースせずにシリアライズデータへ直接アクセスでき、メモリ制約のある環境に最適です。

Forward Pass（順伝播）::
入力データをニューラルネットワークのレイヤを通して出力を生成するプロセス。推論では順伝播のみが実行されます。

Frozen Layers（凍結レイヤ）::
ファインチューニング中に重みが更新されない事前学習済みモデルのレイヤ。元の学習で学んだ汎用特徴を保持します。

Fully Connected Layer（全結合レイヤ）::
前レイヤのすべてのニューロンが次レイヤのすべてのニューロンに接続されるレイヤ。Dense レイヤとも呼ばれます。

GPU (Graphics Processing Unit)::
大規模並列処理向けに設計された専用ハードウェアアクセラレータで、グラフィックスレンダリングと機械学習ワークロードの両方に非常に有効です。

Gradient Descent（勾配降下）::
学習中に損失を最小化する最適化アルゴリズムで、損失を最も減らす方向へ重みを反復的に調整します。

Inference（推論）::
*学習済み* 機械学習モデルを使って、新しい未見データに対して予測や分類を行うプロセス。本チュートリアルは GPU 上での効率的な推論に焦点を当てます。

Kernel / Filter（カーネル / フィルタ）::
畳み込み演算で入力データの特定特徴を検出するために使われる重みの小さな行列です。

Latency（レイテンシ）::
単一入力がモデルで処理され出力が得られるまでの時間。ゲームや対話型 AR のようなリアルタイムアプリケーションでは低レイテンシが重要です。

Layer（レイヤ）::
ニューラルネットワーク内で、入力に対して特定種類の計算（畳み込み、プーリング、活性化など）を行う機能ブロックです。

Machine Learning (ML)（機械学習）::
データからパターンを学び、明示的なプログラミングではなく経験を通じて性能を向上させるシステムを構築することに焦点を当てた人工知能の一分野です。

Memory Reuse / Memory Pooling（メモリ再利用 / メモリプール）::
中間テンソル用バッファを共有または再利用し、GPU メモリフットプリント全体を削減する最適化技法です。

Model（モデル）::
学習したパターンの数学的表現で、ネットワークアーキテクチャと学習中に得た重み/バイアスから成ります。

Neuroplasticity（神経可塑性）::
脳が新しい神経接続を形成することで自己を再編成する生物学的能力。人工ネットワークでは、学習中の接続重みの調整にこの概念が反映されます。

Normalization（正規化）::
入力データまたは中間活性化を標準範囲（例: 0〜1、または平均 0・分散 1）へスケールする技法。学習の安定性と推論性能を改善します。

ONNX (Open Neural Network Exchange)::
PyTorch や TensorFlow など異なるフレームワーク間の相互運用を可能にする、機械学習モデル表現のオープンソース形式です。

Operation Fusion（演算融合）::
複数の演算（例: 畳み込みの後の ReLU）を単一のコンピュートシェーダにまとめ、メモリ帯域とディスパッチオーバーヘッドを削減する性能最適化技法です。

Padding（パディング）::
畳み込みカーネルが端を正しく処理し出力サイズを制御できるよう、入力テンソルの端に余分な値（通常はゼロ）を追加するプロセスです。

Perceptron（パーセプトロン）::
複数の二値入力を受け取り、重み付き和に基づいて単一の二値出力を生成する、最も単純な人工ニューロンの一種です。

Pooling（プーリング）::
テンソルの空間次元（幅と高さ）を縮小するダウンサンプリング演算。計算負荷を減らし、入力位置の小さな変化に対してモデルをより頑健にします。

Postprocessing（後処理）::
モデルの生出力を使える形式へ変換する演算（例: Softmax で確率を得る、座標を画像空間へ戻す）。

Preprocessing（前処理）::
生入力データ（例: 画像のリサイズ、ピクセル値の正規化）を機械学習モデル向けに準備する演算です。

Pre-trained Model（事前学習済みモデル）::
ImageNet のような大規模汎用データセットで既に学習され、推論に直接使ったり転移学習の出発点にできるモデルです。

Quantization（量子化）::
モデルの重みと活性化の数値精度を下げるプロセス（例: 32 ビット浮動小数点から 8 ビット整数へ）。モデルサイズを削減し実行速度を向上させます。

Reduction Operation（リダクション演算）::
テンソルの複数要素を単一値（合計、平均、最大など）にまとめる演算。プーリングや正規化レイヤでよく使われます。

ReLU (Rectified Linear Unit)::
`f(x) = max(0, x)` と定義される一般的な活性化関数。計算効率が高く、学習中の勾配消失問題の防止に役立ちます。

Semantic Segmentation（セマンティックセグメンテーション）::
画像の各ピクセルにカテゴリ（「道路」「車」「空」など）を付けるタスク。シーンの詳細な理解を提供します。

Sigmoid::
任意の入力値を 0 から 1 の範囲へ写す活性化関数。二値分類や特定のゲーティング機構で使われます。

Skip Connection / Residual Connection（スキップ接続 / 残差接続）::
ResNet のようなネットワークの構造要素で、データが一つ以上のレイヤをバイパスでき、非常に深いネットワークでの勾配消失問題の防止に役立ちます。

Softmax::
分類モデルの最終レイヤでよく使われる活性化関数。数値ベクトルを合計 1 の確率分布へ変換します。

Stride（ストライド）::
畳み込みやプーリングでカーネルが各ステップで移動するピクセルまたは要素数。1 より大きいストライドはダウンサンプリングを生みます。

Tanh (Hyperbolic Tangent)::
入力値を -1 から 1 の範囲へ写す活性化関数。再帰ニューラルネットワークでよく使われます。

Tensor（テンソル）::
数値の多次元配列（スカラー、ベクトル、行列の一般化）。機械学習の主要データ構造です。

TFLite (TensorFlow Lite)::
低レイテンシと小さなフットプリントに焦点を当て、モバイル、組み込み、IoT デバイス上で機械学習モデルを実行するための軽量フレームワークです。

Throughput（スループット）::
単位時間あたりにモデルが処理できる入力の総数（例: フレーム毎秒）。大規模データセットを効率的に処理するには高スループットが重要です。

Tokenization（トークン化）::
テキストを単語やサブワードなど、モデルが数値に変換できる小さな単位（トークン）へ分解するプロセスです。

Training（学習）::
ML モデルを構築する初期フェーズ。データセットから「学習」し、誤差を最小化するよう内部パラメータ（重みとバイアス）を調整します。

Transfer Learning（転移学習）::
大規模データセットで学習済みのモデルを新しい関連タスクの出発点として使う技法。必要なデータ量と学習時間を大幅に削減します。

Weights（重み）::
一つのニューロンが別のニューロンに与える影響の強さと方向を決める、ニューラルネットワークの学習可能パラメータです。

Intermediate Representation (IR)（中間表現）::
高レベルモデルと低レベルマシンコードまたはカーネルの間でコンパイラが使う抽象プログラム表現。ML では、SPIR-V や LLVM のようなターゲット固有コードを生成する前に、融合、レイアウト変換、量子化などの共通最適化を可能にします。

MLIR (Multi‑Level Intermediate Representation)::
ドメイン固有 IR と変換を構築するためのコンパイラ基盤。IREE などのプロジェクトは MLIR を使って ML グラフを Vulkan（SPIR-V 経由）、Metal、CUDA、CPU などのバックエンドへ下げます。

Dialect (MLIR)（方言）::
MLIR では、特定ドメインまたはコンパイル段階をモデル化する演算と型の集合です。本チュートリアルに関連する例には StableHLO（ポータブル ML グラフ演算）、Linalg（線形代数ビルディングブロック）、TOSA（ポータブル NN 演算）、MHLO（StableHLO に密接に関連する旧方言）があります。コンパイラは Vulkan 向け SPIR-V などのターゲットコードへ向けて、方言間を合成・変換します。

StableHLO::
モデルグラフをポータブル形式で捉える標準化 ML IR（OpenXLA の一部）。多くのフレームワーク/ツールが StableHLO へエクスポートでき、下流コンパイラ（IREE や XLA など）がそれを消費します。xref:ML_Inference/Third_Party_Libraries/15_openxla.adoc[OpenXLA] を参照。

XLA (Accelerated Linear Algebra)::
TensorFlow や JAX で一般的に使われる ML コンパイラ（OpenXLA 内）。XLA は StableHLO/HLO を CPU/GPU 向け最適化実行可能コードへ下げます。本チュートリアルの Vulkan ターゲットでは、XLA を直接ではなく IREE を使います。xref:ML_Inference/Third_Party_Libraries/15_openxla.adoc[OpenXLA] を参照。

OpenXLA::
StableHLO、XLA、IREE、関連ツールを含む包括プロジェクト。エクスポート形式（StableHLO）を標準化し、多様なバックエンド向けの効率的な成果物を生成するコンパイラ/ランタイムを提供します。

SPIR‑V::
Vulkan シェーダとコンピュートカーネル向けのポータブルバイナリ中間言語。IREE などの ML コンパイラは Vulkan ターゲット時に SPIR-V を出力し、モデルを Vulkan コンピュートパイプライン経由で実行します。
