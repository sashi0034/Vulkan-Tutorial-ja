= ML 向け Vulkan コンピュート: はじめに

== 導入

機械学習推論とは何か、ニューラルネットワークがどう動くか、サードパーティ ML ライブラリを Vulkan アプリにどう統合するかを学びました。次はより深く潜ります: Vulkan コンピュートシェーダを使って ML 推論操作を自分で実装します。

本章は ML の概念と GPU 実装の橋渡しをします。ML に焦点を当てて Vulkan コンピュートパイプラインを復習し、基本操作をコンピュートシェーダとして実装し、完全に動作する例を構築し、チュートリアルの残りで使うパターンを確立します。

== 本章で扱う内容

本章は、Vulkan コンピュートの基礎から完全な ML 操作まで、理解を段階的に積み上げる構成です。

*Vulkan コンピュートパイプラインの復習*: まず Vulkan のコンピュートパイプラインを、ML ワークロードに特化した焦点で復習します。メインチュートリアルでコンピュートパイプラインの作り方はすでに知っていますが、ML 推論には固有のパターンと要件があります。テンソルデータ用のディスクリプタセットレイアウト、演算ごとのワークグループサイズ、ML で重要なメモリアクセスパターンを議論します。

*ML 操作向けコンピュートシェーダ*: 本章の中心は、ML 操作をコンピュートシェーダとして実装することです。単純なもの——ReLU 活性化のような要素ごと演算——から始め、行列乗算や畳み込みのようなより複雑な操作へ積み上げます。各操作を数学的に（何をするか）と計算的に（GPU 上でどう効率的に実装するか）説明します。

*データ管理*: ML 推論は大量のデータ——入力テンソル、重み、中間活性化、出力——を動かします。このデータを GPU メモリにどう整理するか、どう効率的に転送するか、フットプリントを最小化するためにどうメモリを再利用するかを扱います。ここが理論と実践の接点です: テンソルレイアウト、アラインメント要件、バッファ管理の理解。

*同期*: ニューラルネットワークは依存する操作の列です。レイヤ 2 はレイヤ 1 が終わるまで始められません。パイプラインバリア、メモリバリア、実行依存を使い、可能な並列性を最大化しつつ正しい順序を保証する方法を扱います。

*完全に動作する例*: 実践のない理論は空虚です。完全に実行可能な例——ベクトル加算——を構築します。ニューラルネットワークと比べると些細に見えるかもしれませんが、データ読み込み、コンピュート作業のディスパッチ、同期、結果の読み取りというパイプライン全体を、完全に理解できるほど単純な形で示します。このパターンを掴めば、より複雑な操作への拡張は素直です。

== 前提条件

本章は Vulkan コンピュートパイプラインに慣れていることを前提とします。具体的には次を理解している必要があります。

*コンピュートパイプラインの基本*: コンピュートパイプラインの作成、ディスクリプタセットのバインド、ワークグループのディスパッチ。復習が必要なら、メイン Vulkan チュートリアルのコンピュートシェーダ章を再訪してください。

*Slang コンピュートシェーダ*: Slang でコンピュートシェーダを書くこと、ワークグループ構成の理解、共有メモリの使用。本章ではいくつかのコンピュートシェーダを書くので、基本に慣れている必要があります。

*メモリ管理*: バッファの割り当て、デバイスメモリの管理、ホストとデバイス間のデータ転送。ML 推論はかなりのメモリ管理を伴い、Vulkan のメモリモデルを理解している必要があります。

*同期プリミティブ*: パイプラインバリア、メモリバリア、実行依存。ML 操作を調整するためにこれらを広く使います。

本章全体で、Building a Simple Engine チュートリアルと一貫して、リソース管理に vk::raii（Vulkan HPP RAII ラッパー）を使います。C スタイルの Vulkan API と比べ、自動リソース解放とよりきれいなコードを提供します。

前章からの ML 概念——テンソルとは何か、ニューラルネットワークレイヤがどう動くか、畳み込みや活性化関数のような操作が何をするか——も理解している必要があります。これらの操作を実装するので、何を計算すべきかを知っている必要があります。

== 本章が重要な理由

前章では、サードパーティライブラリ対カスタム実装をいつ使うかを扱いました。再確認: ほとんどの本番アプリケーションでは、ONNX Runtime や TensorFlow Lite のような確立されたライブラリを使ってください。実戦検証され、最適化され、保守されています。

本章が「カスタム実装」経路を取る理由は 2 つです。第一に、ML 操作が GPU コンピュートにどう写るか理解すると、ライブラリをより上手く使える——性能特性を理解し、問題をより効果的にデバッグし、情報に基づく最適化判断ができます。第二に、カスタム実装が正しい選択である特殊シナリオ（密なグラフィックス統合、レンダリング AI、新規操作、組み込み制約）があります。

本番でカスタム実装をデプロイすることがなくても、ここで得る知識は価値があります。技術的詳細に潜りましょう。

== 構築するもの

本章の終わりまでに、次を実装します。

*ML 操作用の完全な Vulkan コンピュート基盤*: ML ワークロードに効くディスクリプタセットレイアウト、パイプライン作成、メモリ管理、同期パターン。

*いくつかの基本 ML 操作をコンピュートシェーダとして*: 要素ごと演算（ReLU、sigmoid）、行列演算（行列–ベクトル乗算、行列–行列乗算）、および畳み込みのようなより複雑な操作の基盤。

*動作するベクトル加算の例*: データ読み込みから結果検証までパイプライン全体を示す、完全に実行可能なアプリケーション。より複雑な操作のテンプレートになります。

*チュートリアル全体で使うパターンと技法*: すべての ML 操作に適用されるメモリ管理戦略、同期パターン、デバッグ手法、最適化技法。

== 本章の使い方

本章はハンズオンです。概念を説明し、それから実装します。コードを自分で書いて実行しながらついてきてください。読むだけにせず——コードを打ち、コンパイルし、実行し、試してください。

知っていると思っていても、Vulkan コンピュートの復習から始めてください。ML 特有の焦点が、考えていなかった側面を際立たせます。それから操作を順に進めてください——それぞれが前の上に積み上がります。

操作を実装するとき、数学的定義とコンピュートシェーダ実装の両方を示します。両方を理解してください。数学は操作が何を計算すべきかを告げ、シェーダは GPU 上でどう効率的に計算するかを告げます。

ベクトル加算の例は重要です。完全に理解できるほど単純ですが、より複雑な操作で使うすべてのパターンを示すほど完全です。注意深く学び、すべての行を理解し、自前実装のテンプレートとして使ってください。

各節が前の上に積み上がり、Vulkan コンピュートシェーダで ML 推論を実装する方法の完全な絵を作ります。本章の終わりまでに、GPU 上でニューラルネットワーク操作を実装する理解と実務コードの両方を持てます。
