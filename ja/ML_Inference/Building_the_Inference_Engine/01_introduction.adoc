= 推論エンジン構築の導入

ML 推論パイプラインについて学び、ONNX Runtime のようなサードパーティライブラリを調べ、基本的な Vulkan コンピュート操作を実装しました。次の問いはこうです: ONNX Runtime のようなライブラリがすでに存在しよく動くのに、なぜ自前の推論エンジンを作るのか？

正直な答えは？ ほとんどの本番アプリケーションでは、おそらく作るべきではありません。ONNX Runtime は何年もの最適化作業を表し、何百もの演算子をサポートし、無数のデプロイで実戦検証されています。使えるときは使ってください。

しかし推論エンジンの作り方を理解すると、より深いものが得られます。内部で何が起きているかが明らかになります。性能ボトルネック、数値精度の問題、デプロイ制約に出会ったとき、なぜ起きるか、どう対処するかを理解できます。どのライブラリをいつ使うか、どの最適化が重要か、どんなトレードオフをしているかについて、より良い判断ができるようになります。

本セクションでは、ゼロからシンプルだが機能する推論エンジンを構築する方法を教えます。本番品質ではありませんが、実際のニューラルネットワークを GPU 上で動かせる程度には本物です。

== 旅路

段階的に構築し、各段階で理解の層を明らかにします。

**段階 1: ハードコードされた順伝播**。まず、動く最も単純なもの——単一のニューラルネットワークをレイヤごとにハードコード——から始めます。単純なバイナリファイルから重みを読み込み、各レイヤのコンピュートシェーダを順に実行します。柔軟性も抽象化もなく、直接実行だけです。これがベースラインです。動き、そして何より完全に理解できます。

**段階 2: パターンの特定**。ハードコード版が動くと、繰り返しに気づきます。どのレイヤも似たことをします: バッファを割り当て、データをアップロードし、ディスクリプタをバインドし、コンピュートシェーダをディスパッチする。違いは詳細——どのシェーダか、どんな次元か、どのパラメータか——にあります。共通点と可変点を特定します。

**段階 3: 抽象化**。パターンが分かったら抽象を作ります。データを表すテンソルクラス。レイヤ用の演算インタフェース。接続を捉えるグラフ構造。これでハードコードした 1 本だけでなく、任意のネットワークを表現できます。

**段階 4: モデル形式**。次に、ネットワーク定義と重みをエンジンに入れる手段が必要です。自前形式を発明することもできますが、業界はすでにこの問題を解いています。ONNX と TensorFlow Lite は分野横断で使われる標準形式です。それらをパースし、グラフ構造と学習済み重みを取り出す方法を学びます。

**段階 5: 統合**。最後にすべての部品をつなぎます。モデルファイルを読み込み、内部表現を構築し、GPU リソースを割り当て、推論を実行します。結果は、実際のネットワークを実行できる動作する推論エンジンです。

== 何を構築するか

本セクションの終わりまでに、次のコードを持てます。

- 学習済み MNIST モデル（手書き数字認識）を読み込む
- ONNX または TensorFlow Lite 形式からモデル構造と重みをパースする
- Vulkan コンピュート上で完全な順伝播を実行する
- 正しい予測を生成する（PyTorch/TensorFlow に対して検証可能）

実装は本番品質ではありません。サポートするレイヤ型は限定的です（dense、convolution、ReLU、max pooling）。すべての最適化は備えません。しかし本物で、完全で、教育的です。

== 教育用コードと本番コード

本セクション全体で、性能より明確さを優先する選択をします。本番コードでは次のような手法を使います。

- 演算子融合（複数演算を 1 つのシェーダに結合。xref:ML_Inference/Advanced_Topics/05_model_optimization.adoc[高度なトピック: モデル最適化] で扱う）
- 高度なメモリプーリング（重ならない寿命にわたってバッファを再利用。xref:ML_Inference/Advanced_Topics/04_memory_management.adoc[高度なトピック: メモリ管理] で扱う）
- FP16 または INT8 量子化（精度と速度のトレードオフ。xref:ML_Inference/Advanced_Topics/02_quantization_strategies.adoc[高度なトピック: 量子化戦略] で扱う）
- マルチスレッドグラフ実行（独立した演算を並列実行）

後のセクションで触れますが、推論エンジンは単純に保ちます。目標は理解であり、最大スループットではありません。

== 前提条件

続ける前に、次に慣れている必要があります。

- Vulkan コンピュートシェーダ（メインチュートリアルの xref:11_Compute_Shader.adoc[コンピュート章] で扱う）
- 基本的なニューラルネットワークの概念（ML 推論パイプライン節で扱う）
- C++ プログラミング、特に RAII とスマートポインタ

全体を通じて Vulkan-Hpp RAII バインディングを使い、メインチュートリアルのパターンを基にします。
