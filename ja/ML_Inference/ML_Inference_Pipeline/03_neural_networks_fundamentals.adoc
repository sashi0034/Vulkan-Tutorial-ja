= ニューラルネットワークの基礎

== はじめに

推論について議論する前に、ニューラルネットワークが実際に何であるかをしっかり理解しましょう。ニューラルネットワークと AI への期待は大きい一方、魔法のブラックボックスとして扱うのではなく基礎を理解することが重要です。ニューラルネットワークは強力ですが、結局は数学的関数——数学のルールによって制限され、また力を与えられています。

本章は第一原理からニューラルネットワークの理解を築き、数学的関数から始め、生物学的ニューロンに触発を得て、ニューラルネットワークが機能するために非線形活性化関数がなぜ不可欠かを示します。

== 関数とは何か？

最も基本的な概念から始めましょう: 数学的関数。*関数* は、ある入力を出力に対応づける方法です。数学内のすべての関数は、いくつかの入力を取り、それらの入力に対応する出力をちょうど一つ生成します。関数の重要な性質は、同じ入力が与えられれば常に同じ出力を返すことです。

出力は一つの数である必要はありません——ベクトルや行列のような構造でも構いません。この単純な概念は驚くほど強力です。実際、コンピュータプログラムは本質的に数学的関数を実行しているため、非常に多様なことができます。

関数は必要に応じて単純にも複雑にもなれます:

- *線形*: `y = f(x) = x + 1`
- *放物線*: `y = f(x) = x² + 1`
- *多入力*: `y = f(a,b,c,d) = 23a + 42b + 51c + d`
- *複雑*: 線形回帰からの勾配降下
- *未知*: `Diagnosis = f(symptoms) = ???`

最後の例が重要です。時には、書き方すら分からない関数を探していることがあります。患者の症状を診断に対応づける関数が存在するが、その関数が何か分からないなら、機械学習——特にニューラルネットワーク——が見つけるのを助けてくれます。

線形回帰で議論したように、機械学習はデータに最もよくフィットする関数を見つけようとします。ニューラルネットワークは未知の関数を見つけるための、最も汎用的な道具の一つです。しかし、非常に汎用的であることは、関数は見つかるが最も効率的な実装とは限らない、という意味でもあります。詳細は後述します。

== 自然から学ぶ: 生物学的ニューロン

人間の脳の働きから触発を得ましょう。これまで考えたこと、形成した記憶、身につけた技能——すべてが数十億のニューロンの協調した活動から生じます。驚くべきことに、個々のニューロンは驚くほど単純です。脳の複雑さと力は、高度な個々の部品ではなく、膨大な数の単純な部品が協調することから来ます。

=== 構造: 通信ネットワーク

自分の神経系について少し考えてみてください。熱いものに触れると、即座に手を引っ込めます。友人の顔を見ると、すぐに認識します。これらの言葉を読むと、意味を理解します。感覚、認識、理解——これらすべてのプロセスは、互いに通信するニューロンのネットワークを通じて起きます。

image::images/ML_Inference/ML_Inference_Pipeline/biological_neuron.png[樹状突起、細胞体、軸索を示す生物学的ニューロン]

生物学的ニューロンには三つの主要部分があります:

1. *樹状突起（Dendrites）* - 他のニューロンからの入力を受け取る、クモの巣のような構造。ニューロンの「耳」と考えてください——他のニューロンが何を言っているかを聞いています。一つのニューロンは数千の樹状突起を持ち、それぞれが異なるニューロンに接続され、多くの情報源から同時に情報を統合できます。

2. *細胞体（Cell Body / Soma）* - 入力を処理し、活性化するかどうかを決めるニューロンの本体。「思考」が起きる場所——細胞体は樹状突起から入るすべての信号を合計し、合算入力が十分強いかどうかを判断します。

3. *軸索（Axon）* - 細胞体から伸びる長い管（軸索起始部から軸索終末まで）で、出力信号を伝達します。ニューロンの「声」——発火を決めると、電気信号が軸索を駆け下り、他のニューロンと通信します。一部の軸索は非常に長く、足趾を制御するニューロンは脊髄から足まで伸びます。

*シナプス* は、一つのニューロンの軸索終末と別のニューロンの樹状突起の間の隙間です。ここで信号がニューロンから次へ渡ります。興味深いことに、ニューロンは実際には互いに触れ合いません——シナプスには微小な隙間があります。電気信号が軸索の端に達すると、化学メッセンジャー（神経伝達物質）の放出が引き起こされ、この隙間を渡って次のニューロンの樹状突起の受容体に結合します。この化学的通信が脳の働きに不可欠です。

重要なパターンに注目してください: 各ニューロンは多数の樹状突起（複数入力）と一つの軸索（単一出力）を持ちます。馴染みがありますか？ それは先に議論した数学的関数の定義そのものです！ 偶然ではありません——人工ニューラルネットワークを設計するとき、直接この生物学的アーキテクチャに触発されています。

=== ニューロンの実際の働き: 発火プロセス

生物学的プロセスは、見事に単純でありながら強力です。コンサートにいて、拍手を始めるかどうか決めていると想像してください。周囲の人の声を聞いています——熱心に拍手する人、静かに座る人、ブーイングする人。これらの入力を無意識に重み付けしています。「近くの三人が拍手しているが、二人は静かで、あの一人は本当に熱心…」受け取る信号の合計が「拍手」と言えば、あなたも拍手を始めます。

ニューロンも同様に働きます。細胞体は複数の樹状突起を通じて信号を受け取ります。一部は *興奮性*——発火方向へ押します。他は *抑制性*——発火に反対します。細胞体は常にこれら競合する信号を合計しています。総興奮が一定の閾値を超えると、ニューロンは軸索を下る電気パルスを送ることで「発火」します。

この発火はすべてか無かのイベントです。ニューロンは「少し」や「たくさん」発火するのではなく、スイッチのように発火するかしないかです。ただし、ニューロンは異なるレートで繰り返し発火できます。強く一貫した興奮を受けるニューロンは秒間何度も発火し、弱い信号のニューロンは時折しか発火しません。この発火率が、信号の強さをエンコードする方法です。

興味深いのは、発火の閾値が固定ではないことです。*シナプス可塑性* というプロセスを通じて、ニューロン間の接続は時間とともに強まったり弱まったりします。新しいことを学ぶとき、文字通り脳内のシナプス接続の強さを変えています。「一緒に発火するニューロンは、一緒に結線する」という言葉がこの考えを捉えています——二つのニューロンが繰り返し同時に活性化されると、間の接続が強まり、将来一方が他方を活性化しやすくなります。

=== ネットワークの力: 複雑さの出現

単一ニューロンは単純ですが、脳には約 860 億のニューロンがあり、それぞれが数千の他と接続しています。それは数兆の接続で、驚くべき複雑さのネットワークを形成します。ここで魔法が起きます。

顔を認識するとき何が起きるか考えてください。光が網膜に当たり、目のニューロンを活性化します。これらのニューロンは視覚野のニューロンへ信号を送り、エッジや色のような基本特徴を検出します。それらは曲線や形状のようなより複雑なパターンを認識するニューロンへ接続します。それらは目や鼻のような顔の特徴を認識するニューロンへ接続します。そしてそれらは特定の顔——文字通り、馴染みのある顔に特異的に応答するニューロン——を認識するニューロンへ接続します。

これは *特徴の階層* です。単純なニューロンは単純なものを検出します。その出力は、より複雑なものを検出するニューロンへ供給されます。そうして、基本感覚入力から高レベル概念へ積み上がります。この階層的組織は、人工ニューラルネットワークを構築するとき使う重要な洞察の一つです。

記憶も同様の原理で働きます。何かを思い出すとき、特定の場所に保存されたファイル（コンピュータのように）にアクセスしているのではありません。代わりに、神経活動のパターンを再活性化しています。幼少期の家の記憶には、視覚的詳細、感情的連想、空間関係などをエンコードするニューロンが含まれ——すべてが協調したパターンで一緒に発火します。だから記憶は予期しない手がかり——匂い、歌、似た場所——で引き起こされることがあります。パターンのどの部分でも全体を再活性化できます。

=== 可塑性と学習: 脳の超能力

おそらく生物学的ニューロネットワークの最も驚くべき性質は、自己再編成能力です。これを *神経可塑性* と呼び、すべての学習の基盤です。

楽器演奏、言語、数学問題解決といった技能を練習するとき、特定の神経経路を強化しています。その技能に関与するニューロンが繰り返し一緒に発火し、接続が強まります。だから練習が効くのです——文字通り、そのタスクをより効率的に実行するよう脳を再配線しています。

脳は損傷への適応にも驚くほど優れています。一領域が損傷されると、他領域が時にその機能を引き継げます。視力を失った人は聴覚と触覚の感度が高まることが多い——耳や指が変わるのではなく、通常視覚を処理する脳領域が他の感覚処理に再利用されるからです。ニューロンは同じ; 変わるのは接続とパターンです。

この適応性は専門化にも及びます。特定の脳領域が特定タスクを担当する傾向はあります（多くの人では言語は左半球、空間推論は右半球）が、ハードコードではありません。脳は経験に基づいて自己組織化します。プロの音楽家の脳は、非音楽家より音楽処理により多くの神経領域を割り当てます。ロンドンのタクシー運転手の脳は、複雑な街路配置を暗記することで海馬（空間ナビゲーションに関与する領域）が拡大します。

=== 人工ニューラルネットワークにとってなぜ重要か

生物学的ニューロンを理解することは、触発と謙虚さの両方を与えます。触発は明確です: 脳のアーキテクチャ——単純な処理ユニット、重み付き接続、階層的組織、接続強度調整による学習——が人工ニューラルネットワークの設計図を提供します。

謙虚さは、除外しているものを認識することから来ます。生物学的ニューロンは人工版よりはるかに複雑です。複雑な内部化学、数十の異なる神経伝達物質による通信、興奮性または抑制性の接続、複数の時間スケールでの同時動作。すべての複雑さを再現しようとしているのではありません——中核の計算原理を抽出し、数学的に扱える形で実装しています。

重要な洞察はこれです: 計算力を捉えるために生物学を完璧に模倣する必要はありません。飛行機は鳥のように羽ばたかなくても飛べるように、人工ニューラルネットワークは生物学的ニューロンのすべての詳細を再現しなくても、印象的な情報処理能力を達成できます。

構築しているものは生物学に触発されていますが、デジタルコンピュータ——特に Vulkan コンピュートシェーダを使った GPU 上での効率的実行——向けに最適化されています。中核のアイデア（重み付き入力、閾値、単純ユニットのネットワーク、調整による学習）を取り、並列計算にうまく写る形で実装します。

== 自然を模倣する: 人工ニューロン

この生物学的理解をもとに、入力を取り、判断し、出力を生成する人工ニューロンを作ることで、脳の働きを再現できます。三入力一出力の人工ニューロンは次のようになります:

image::images/ML_Inference/ML_Inference_Pipeline/linear_perceptron.png[三入力一出力の人工ニューロン（パーセプトロン）]

つまり人工ニューロンには数学的関数があります: `output = f(x, y, z)`。

しかしどの関数を使うべきでしょうか？ 答える前に、ニューラルネットワークを機能させる二つの基本概念——*重み* と *バイアス*——を理解する必要があります。

=== 重みを理解する: 重要度の乗数

*重み* は入力に掛けられる数で、その入力が出力に与える影響の大きさを決めます。大きい重みは「この入力は非常に重要」。小さい重みは「この入力はほとんど重要でない」。負の重みは「この入力は出力を反対方向へ押す」という意味です。

判断を下すとき、私たちは自然に異なる要因を異なる重みで扱います——一部は他より重要です。ニューラルネットワークも数学的に同じことをします。

三入力ニューロンでは、例えば:

- `w₁ = 5.0`（第一入力は非常に重要）
- `w₂ = 0.1`（第二入力はほとんど重要でない）
- `w₃ = -2.0`（第三入力は中程度に重要だが逆方向）

ニューロンが `w₁·x + w₂·y + w₃·z` を計算するとき、重み付き和を作っています——各入力は重みに比例して結果に寄与します。これがニューロンがどの入力が重要かを「学ぶ」方法です。学習中、これらの重みが調整され、入力に正しく応答するようになります。

=== バイアスを理解する: 閾値調整器

*バイアス* は重み付き和に加えられる定数で、ニューロンの活性化閾値を調整します。大きな正のバイアスは活性化を容易にし（より容易に発火）、大きな負のバイアスは活性化を難しくします（より強い入力信号が必要）。

バイアスはニューロンの「デフォルト傾向」を表します。入力がまったくない（x = 0, y = 0, z = 0）場合、重み付き和はゼロになります。しかし例えば `b = -10` のバイアスがあれば、ニューロンは -10 から始まり、正の入力でその負のオフセットを克服する必要があります。`b = +10` のバイアスなら、すでに活性化しやすい状態です。

重みとバイアスを合わせると、各ニューロンに独自の「個性」——何に応答し、どれだけ容易に発火するか——が与えられます。学習中、重みとバイアス両方が調整され、ネットワークが正しい出力を生成します。推論中（ここが焦点）は、学習済みモデルから読み込んだ固定値として計算に使います。

=== まとめる: 線形ニューロン

より意味のある形でニューロンの関数を書けます。実験から始めましょう。関数が線形——つまり直線または平面を記述する——と仮定します:

[source]
----
output = f(x, y, z) = w₁·x + w₂·y + w₃·z + b
----

ここで:

- `w₁`, `w₂`, `w₃` は *重み*（各入力の重要度を決める学習済み値）
- `b` は *バイアス*（活性化閾値を調整する学習済み値）
- `x`, `y`, `z` は入力
- 出力は重み付き和にバイアスを加えたもの

== 線形関数の問題

複数のニューロンをネットワークで接続すると何が起きるか見てみましょう:

image::images/ML_Inference/ML_Inference_Pipeline/neural_network_example.png[入力層、隠れ層、出力層を持つ単純なニューラルネットワーク]

このネットワークでは、出力はノード N6 から来ます。数学をたどってみましょう:

*最終層:*

[source]
----
output = N6_output
N6_output = w_a · N5_output + w_b · N4_output + w_c · N3_output + b_d
----

*隠れ層:*

[source]
----
N3_output = w_e · N1_output + w_f · N2_output + b_g
N4_output = w_h · N1_output + w_i · N2_output + b_j
N5_output = w_k · N1_output + w_l · N2_output + b_m
----

*入力層:*

[source]
----
N1_output = w_n · X + w_o · Y + w_p · Z + b_q
N2_output = w_r · X + w_s · Y + w_t · Z + b_u
----

問題はここです: これらの式をすべて互いに代入して簡約すると:

[source]
----
output = f(x,y,z) = w_final_a · X + w_final_b · Y + w_final_c · Z + b_final
----

ネットワーク全体が単一の線形関数に崩れてしまいました！ これは *線形関数の組み合わせは常に線形* だから起きます。レイヤをいくら追加しても同じです。余分なニューロンはすべて、より単純な形に還元できる不要な計算を作るだけです。生物学的接続の数兆の可能性について語った美しい複雑さは、数学的単純さへ消えます。線形活性化に基づくネットワークでは複雑さを維持できません。

== 解決策: 非線形活性化関数

この問題を解決し、複雑さを取り戻すには、*非線形関数*——入力と出力の間に線形関係を持たない関数——を使う必要があります。これにより驚くべき強力な性質が得られます: *万能近似定理* により、任意の関数は非線形関数のニューラルネットワークで近似できます。

少し考えてみてください: 入力を出力に対応づける関数が *存在できる* なら、非線形関数を使うニューラルネットワークがその関数を近似できます。近似するには入力と出力以上は不要です。これがニューラルネットワークを強力で汎用的にする理由です。

多くの非線形関数が存在しますが、目的に最も直感的な一つを議論しましょう: *シグモイド関数*（ロジスティック関数とも呼ばれます）:

[source]
----
σ(z) = 1 / (1 + e^(-z))
----

シグモイド関数にはいくつかの有用な性質があります:

- 出力は常に 0 から 1 の間（「オフ」から「オン」を表す）
- 滑らかで微分可能（学習に重要だが、推論には不要）
- 自然に閾値挙動を作る

シグモイド出力が 0.5 より大きければ、ニューロンは「活性化された」と見なせます。0.5 の活性化閾値を調整するバイアスや方法も作れます。活性化関数を適用する前にバイアス項を加えることで可能です。

== 汎用性のコスト

この例は重要な洞察を示します: 探している関数が既に分かっている（AND のように）なら、ニューラルネットワークはやりすぎです。単純な AND 演算は定数時間の単一演算です。しかしニューラルネットワークでは:

1. 学習済み重みとバイアス (-30, 20, 20) を保存
2. 浮動小数点の乗算と加算を実行
3. シグモイド関数（指数を含む）を評価
4. 予測のたびにこれらすべてを実行

これは直接実装と比べて *非常に高コスト* です。ニューラルネットワークは未知の関数を見つけられるから強力ですが、既知の関数を実装する最も効率的な方法ではありません。

入力と出力の関係が事前に分かっているなら、ニューラルネットワークより単純なものを使います。しかし、データから関数を発見する必要があり——より単純な機械学習手法（線形回帰、サポートベクターマシンなど）が失敗するとき——ニューラルネットワークは学習を通じて正しい関数をついに見つけられます。

== 現代の活性化関数: シグモイドだけでは足りない理由

シグモイドの直感的性質——任意の入力を 0 から 1 の値へ滑らかに写す、生物学的ニューロンの「オン/オフ」挙動を模倣——に焦点を当ててきましたが、現代のニューラルネットワークは隠れ層でシグモイドをほとんど使いません。なぜでしょうか？ 理解するには、学習プロセスを少し覗き、ニューラルネットワークがどう学ぶかの基本概念を理解する必要があります。

=== ニューラルネットワークの学習: やさしい入門

シグモイドの限界を議論する前に、ニューラルネットワークが実際にどう学ぶかを理解しましょう。本チュートリアルは推論に焦点を当てていますが、学習の基礎を理解すると、モデルが異なる活性化関数を使う理由と、正しく実装する方法を評価するのに役立ちます。

AND ゲートの例を思い出してください。重みが 20, 20, -30 に「選ばれた」と言いました。しかしニューラルネットワークはどうやって自分でこれらの値を発見するのでしょうか？ ここで学習が入ります。

==== 学習問題

ニューラルネットワークの学習は反復プロセスです。ネットワークが予測を行い、どれだけ間違っていたか（*誤差* または *損失*）を測定し、誤差を減らすよう重みを調整します。重要な問いは: 重みを *どう* 調整するか？ 特定の重みを増やすと誤差は上がるか下がるか？ どれだけ？

==== 勾配の登場

ここで *勾配* が入ります。勾配は、入力を変えたとき出力がどれだけ変わるかを測る尺度です。微積分の言葉では、導関数——特定点での関数の傾き——です。

具体化しましょう。単純な関数: `error = (prediction - actual)²`。予測が 0.8 で実際が 1.0 なら、誤差は `(0.8 - 1.0)² = 0.04`。予測をわずかに 0.81 に増やすと、誤差は `(0.81 - 1.0)² = 0.0361`。誤差は下がりました！ 勾配がこの関係を教えてくれます。「予測を増やすと誤差は減る」。

ニューラルネットワークでは、特定の重みを変えると誤差がどう変わるかを知る必要があります。その重みに対する誤差の勾配がまさにそれを教えます。勾配が正なら、重みを増やすと誤差が増える（悪い）。負なら、重みを増やすと誤差が減る（良い）。

==== 勾配降下: 山道の比喩

各重みの勾配が分かったら、*勾配降下* という美しく単純なアルゴリズムを使えます。どう動くか、詳しいメンタルモデルを構築しましょう。

===== ランダムな推測から始める

山道のどこか——峰の近く、途中、尾根——にランダムに降ろされたと想像してください。地図も GPS もなく、足と、どの方向が下り坂か感じる能力だけ。目標は下の谷の最も低い点に到達することです。

これがまさにニューラルネットワーク学習の始まりです。*ランダムな初期重み*——関数がどうあるべきかのランダムな推測——から始めます。これらの初期重みはひどいかもしれず、極端に間違った予測を出すかもしれません。問題ありません。良い重みで始めることを期待しているのではなく、学習プロセスを通じて良い重みを *見つける* ことを期待しています。

===== 損失関数: コンパス

進捗しているかどうか測るには、現在の位置がどれだけ間違っているかを測る方法が必要です。山の比喩では、それが標高——高いほど目標から遠い。

ニューラルネットワークでは、この測定を *損失関数*（*誤差関数* または *コスト関数*——これらの用語は interchangeable に使われます）と呼びます。損失関数はモデルの予測を正解と比較し、「どれだけ間違っているか」を表す単一の数を生成します。

一般的な損失関数:

*平均二乗誤差 (MSE)*: `loss = average((prediction - actual)²)` - 連続値を予測する回帰問題で使用。誤差を二乗するため、大きなミスは小さなミスより重く罰されます。

*交差エントロピー損失*: カテゴリを予測する分類問題で使用。予測確率分布が真の分布とどれだけ異なるかを測ります。

重要な洞察: 損失関数が地形（ランドスケープ）を作ります。重みの可能な組み合わせごとに地形上の点があり、その「高さ」が損失です。良い重みは谷（低損失）、悪い重みは峰（高損失）にあります。学習はこの地形をたどって谷を見つけるプロセスです。

===== 下り坂へ歩む

山道に戻りましょう。足元の地面を感じ、最も急に下る方向を判断します。その方向へ一歩。足を置いた場所で再び傾きを感じ、もう一歩。常に下り坂の方向へ進むプロセスを繰り返します。

これが勾配降下です。勾配は重み空間で「下り坂」の方向——損失を最も速く減らす方向——を教えます。その方向へ重みを調整します:

[source]
----
new_weight = old_weight - learning_rate × gradient
----

マイナス符号が重要です: 勾配が正（重みを増やすと損失が増える）なら、重みを *減らす*。勾配が負（重みを増やすと損失が減る）なら、重みを *増やす*。常に勾配の反対方向——つまり下り坂——へ動きます。

`learning_rate` は一歩の大きさです。重要なパラメータ:

- *大きすぎる*: 谷全体を飛び越し、反対側から反対側へ跳ね返り、底に落ち着かない。さらに悪いことに、山の全く別の部分へ跳ぶかもしれない。
- *小さすぎる*: 最終的には底に到達するが、数千〜数百万ステップかかる。学習が非常に遅くなる。
- *ちょうど良い*: オーバーシュートせず最小値へ着実に進む。

実務では、学習率は多くの場合大きく始め（素早い進捗）、徐々に減らします（オーバーシュートせず最小値に落ち着く）。これを *学習率スケジューリング* と呼びます。

===== いつ止めるか: 収束

底に到達したかどうかは？ 山の比喩では、どの方向へ歩いても上り坂になることに気づくかもしれません。局所的な低点にいます。

ニューラルネットワーク学習では *収束*——損失が有意に減らなくなる点——を探します。収束したことを示す指標:

*損失のプラトー*: 学習ステップ間で損失値があまり変わらない。損失が 0.523、0.522、0.521 と、多くのステップで 0.001 未満の変化なら、おそらく収束しています。

*勾配の大きさが極小に*: 勾配がゼロに近づくと、地形の平坦な地点——最小値またはプラトー——にいます。

*検証性能が改善しなくなる*: 最も実用的な指標。学習中にモデルが見ていないデータ（検証セット）を取り分け、定期的に性能を確認します。検証性能が改善しなくなったら、学習を止める時です。

===== 課題: 局所最小と大域最小

ここで山の比喩が重要な課題を明らかにします。地形が単純なボウル型の谷ではないと想像してください。代わりに、多くの谷、尾根、くぼみを持つ複雑な山脈です。谷には深いもの（低損失）と浅いもの（高損失）があります。

山の一部から始め、勾配に従って下りると、小さなくぼみ——*局所最小*——に終わるかもしれません。そこからすべての方向が上り坂なので、勾配降下は止まります。しかし尾根の向こうに、はるかに深い谷（*大域最小*）があるかもしれません。尾根を越えるには一度上り坂に行く必要があるため、見つけられません。

これはニューラルネットワーク学習の根本的課題です:

*大域最小*: 損失地形全体の絶対的最低点。学習データに対するモデルの最良可能な重みを表します。

*局所最小*: より高い点に囲まれた低点だが、絶対的最低ではない。勾配降下はここで停滞し得ます。

実務では、現代の深いニューラルネットワークでは、期待より問題になりにくいことが分かっています。高次元空間（数百万の重み）の損失地形には興味深い性質があり、すべて同様に良く機能する多くの「十分良い」最小値があります。絶対的大域最小を見つける必要はなく、十分深い最小値を見つければ十分です。

さらに、次のような技法:

- *モメンタム*: 勾配降下に「慣性」を加え、小さな bump やくぼみを転がり越えられるようにする
- *ランダム再起動*: 異なるランダム初期化から複数回学習を開始
- *確率的勾配降下*: 学習プロセスにランダム性を入れ、浅い局所最小から脱出を助ける

が、複雑な損失地形をたどり、大域最小が見つからなくても良い解を見つけるのを助けます。

===== モデルの評価: モデルがより良いかどうか

学習の基本が分かったところで、重要な問いが生じます: モデルが本当に良いかどうかは？ 二つの異なるモデルを比較して、どちらがより良いか決めるには？

損失関数はモデルが学習データにどれだけフィットするかを教えますが、それだけではありません。本当に気にするのは、*新しい未見データ*——学習していないデータ——での性能です。これを *汎化* と呼びます。推論モデルが意味をなすよう、学習プロセスについて簡単に触れましょう。

====== 学習 vs 検証 vs テストデータ

モデルを適切に評価するには、データを三つに分割します:

*学習データ*（通常データの 70〜80%）: モデルを学習する——勾配降下で重みを調整する——ために使用。モデルは学習中にこのデータを見て学びます。

*検証データ*（通常 10〜15%）: 学習中のモデル評価とハイパーパラメータ（学習率、ネットワークアーキテクチャなど）の調整に使用。モデルはこのデータで学習しませんが、学習プロセスの判断に使います。

*テストデータ*（通常 10〜15%）: 最終的で偏りのない性能見積もりのためにのみ使用。モデルは学習中もハイパーパラメータ調整中もこのデータを見ません。

なぜ三つ？ 学習データだけでは、モデルが学習例を暗記している（過学習）のか、一般パターンを本当に学んでいるのか分かりません。学習とテストだけでは、テスト性能に基づいてモデルを調整したくなり、テストセットの情報がモデル設計に漏れます。三分割が最終評価を公正に保ちます。

====== 主要評価指標

タスクによって異なる指標が必要です:

*分類（カテゴリ予測）向け*:

- *正解率 (Accuracy)*: 予測の何パーセントが正しいか？ 単純で直感的だが、クラスが不均衡だと misleading（例: 95% が「スパムでない」なら、常に「スパムでない」と予測するモデルは 95% 正解率だが無用）。

- *適合率と再現率*: 適合率は「陽性と予測した例のうち、実際に陽性だったのは何割？」再現率は「実際の陽性例のうち、何割を見つけた？」不均衡問題で重要です。

- *F1 スコア*: 適合率と再現率の調和平均。両方の懸念をバランスする単一数。

*回帰（連続値予測）向け*:

- *平均絶対誤差 (MAE)*: 予測と実際値の絶対差の平均。解釈しやすい——MAE が 5 なら、平均して 5 単位ずれている。

- *平均二乗誤差 (MSE)*: 二乗差の平均。大きな誤差を小さな誤差より重く罰する。

- *R² スコア*: モデルがデータの分散のどれだけを説明するか。R² 1.0 は完璧な予測、0.0 は平均を予測するのと同程度。

====== モデルの比較

二つのモデルを比較するとき、検証セット（開発中）またはテストセット（最終評価）での性能を見ます:

*モデル A の検証損失がモデル B より低い*: 両方の複雑さが似ていれば、モデル A の方が良い可能性が高い。低い損失はデータへのより良いフィット。

*モデル A の検証正解率がモデル B より高い*: 分類タスクでは、モデル A がより多く正しく予測している。

*モデル A は検証誤差が低いが学習誤差がモデル B より高い*: モデル A の方が汎化が良い——モデル B ほど学習データに過学習していない、という示唆かもしれない。

重要な洞察: 良いモデルは学習データで良く性能するだけのものではなく、見ていないデータでも良く性能するものです。だから常に hold-out 検証/テストデータで評価し、学習データだけでは評価しません。

====== 過学習 vs 未学習

二つの一般的な失敗モードがモデル品質の理解を助けます:

*過学習*: モデルは学習データで非常に良いが検証データで poor。本質的に学習例を暗記し、一般パターンを学んでいない。利用可能な学習データに対してモデルが複雑すぎるとき起きます。対策: より多くの学習データ、より単純なモデル、正則化技法。

*未学習*: モデルは学習・検証両方で poor。データのパターンを全く学んでいない——モデルが根底の関係を捉えるには単純すぎる。対策: より複雑なモデル、より長い学習、特徴/データ表現の改善。

最適な点は、学習・検証両方で良く性能し、検証性能が（通常わずかに下だが）学習性能に近いモデルです。新しいデータへ転移する一般パターンを学んだことを示します。

==== 逆伝播: 勾配を効率的に計算

課題はこうです: ニューラルネットワークは数百万の重みを持つかもしれません。*すべての重み* に対する誤差の勾配が必要です。素朴に計算すると実用的ではないほど遅い。

*逆伝播* はこれを効率的にする巧妙なアルゴリズムです。*連鎖律* と呼ばれる数学的原理を使い、ネットワークを一度後方に通してすべての勾配を計算します。

==== 連鎖律の理解

連鎖律は微積分の基本概念で、*合成関数*——他の関数の中にネストされた関数——の変化率を見つける方法を教えます。

重要な原理: A が B に影響し、B が C に影響するとき、A の C への効果は個々の効果を掛け合わせて見つかります。

数学的に、`y = f(u)` と `u = g(x)` なら、y の x に対する変化率は:

[source]
----
dy/dx = (dy/du) × (du/dx)
----

外側関数の導関数に内側関数の導関数を掛けます。より長い連鎖にも拡張: 四つの関数がネストされていれば、四つの導関数を掛けます。

==== ニューラルネットワークが連鎖律に最適な理由

美しい洞察: *ニューラルネットワークは本質的に合成関数* です。各レイヤは前レイヤの出力を入力として取る関数です。レイヤを積み重ねると、関数を合成しています:

[source]
----
output = layer_n(layer_{n-1}(...layer_2(layer_1(input))...))
----

これはまさに連鎖律が設計された構造です！ 単純な例をたどりましょう。三層ネットワーク:

1. 入力層: `h₁ = activation(W₁ × input + b₁)`
2. 隠れ層: `h₂ = activation(W₂ × h₁ + b₂)`
3. 出力層: `output = activation(W₃ × h₂ + b₃)`
4. 誤差: `error = (output - target)²`

第一層の重み（W₁ の一要素など）を変えると最終誤差がどう変わるか知りたいなら、この依存関係の連鎖をたどる必要があります:

- W₁ を変えると h₁ に影響
- h₁ を変えると h₂ に影響
- h₂ を変えると output に影響
- output を変えると error に影響

連鎖律により、error の W₁ に対する勾配は:

[source]
----
∂error/∂W₁ = (∂error/∂output) × (∂output/∂h₂) × (∂h₂/∂h₁) × (∂h₁/∂W₁)
----

中間の導関数をすべて掛け合わせます。だから「逆伝播」——誤差（終端）から始め、各レイヤの導関数を掛けながら各レイヤを後方へ伝播します。

==== 活性化関数への重要な洞察

ここで活性化関数が重要になります: 各レイヤの導関数には、そのレイヤの活性化関数の導関数が含まれます。`∂h₁/∂W₁` を計算するとき、そのレイヤに適用された活性化関数の導関数が必要です。

活性化関数の導関数が極小（大きな入力でのシグモイドのように）なら、その小さな数が連鎖に掛けられます。深いネットワークでは多くの小さな数を掛け合わせ、勾配が消失——学習が事実上止まるほど小さく——できます。だから活性化関数の選択が学習にとって非常に重要で、ReLU のような勾配特性がより良い関数へ至ったのです。

=== 学習の課題: 勾配消失

シグモイドの問題が理解できます。逆伝播中、各レイヤを後方に流れる勾配は、そのレイヤの活性化関数の導関数で掛けられます。

シグモイドの問題: 大きな正または負の入力では勾配が非常に小さい。ゼロから遠い（x = 10 や x = -10）と、シグモイド曲線はほぼ平坦で、導関数はゼロに近づきます。逆伝播中、これらの極小勾配がネットワークを後方に流れるにつれ掛け合わされ、深いネットワークでは多くの小さな数を掛け、勾配が事実上消失——学習が止まる——します。

これを *勾配消失問題* と呼び、初期の深いニューラルネットワークを悩ませました。入力に近いレイヤは、出力からの勾配信号が到達する頃にはほぼゼロに減衰するため、ほとんど学習しませんでした。

=== ReLU の登場: 単純な解決策

2011 年、研究者は驚くほど単純な活性化関数がこの問題を大きく解決できることを発見しました:

*ReLU (Rectified Linear Unit)*: `f(x) = max(0, x)`

ReLU は驚くほど単純です: 入力が正ならそのまま出力、負ならゼロ。しかしこの単純さがいくつかの利点をもたらします:

1. *正の値では勾配消失なし* - 正の入力では勾配は正確に 1 なので、逆伝播中に減衰しない
2. *計算が安い* - 比較と選択だけ。シグモイドのような指数は不要
3. *スパース活性化* - 約半分のニューロンがゼロ出力。より効率的なスパース表現を作れる

ReLU は深いネットワークの隠れ層のデフォルト活性化関数となり、以前は不可能だったはるかに深いアーキテクチャの学習を可能にしました。深層学習革命を後押しした重要なブレークスルーでした。

ただし ReLU には独自の問題があります: *死んだ ReLU 問題*。学習中にニューロンの出力が負になると、ReLU はゼロを出力し、勾配もゼロ。ニューロンは学習を完全に止め、学習の残り「死んだ」ままです。一部のネットワークでは、かなりの割合のニューロンがこのように死にます。

=== バリエーションと代替

ReLU は勾配消失問題を解決しましたが、死んだ ReLU 問題を導入しました。ReLU の利点を保ちつつ限界に対処する活性化関数の研究が始まりました。しかし物語はそこで終わりません——異なるニューラルネットワークアーキテクチャには異なるニーズがあり、単一の活性化関数がすべての状況に最適ではないことが研究者によって分かりました。

==== Leaky ReLU: 死んだニューロン問題の修正

*Leaky ReLU*: `f(x) = max(0.01x, x)`

死んだ ReLU への最も直接的な修正は単純: 勾配を正確にゼロにしない。負の入力でゼロを出力する代わりに、Leaky ReLU は小さな負の値（通常 0.01x）を出力します。ニューロンが完全に死なない——勾配が正確にゼロにならないため、学習中に常に回復可能——ことを保証します。

Leaky ReLU は ReLU の計算上の利点（依然として比較と乗算だけ）を保ちつつ、死んだニューロン問題を排除します。多くのアプリケーションでは、ReLU のそのまま置き換えでわずかに良く動作します。

==== Tanh: 対称性が必要なとき

しかし——Leaky ReLU が ReLU の問題を修正するなら、なぜ他の活性化関数が必要？ 答えは、異なるアーキテクチャが異なる要件を持つことを理解することにあります。

*Tanh (Hyperbolic Tangent)*: `f(x) = tanh(x)`

Tanh は別の問題——ReLU とその変種の非対称性——に対処します。ReLU 系関数は常に非負（Leaky ReLU ではほぼ非負）値を出力します。微妙な問題を作ります: レイヤ内のすべての活性化が正なら、次レイヤの重みの勾配はすべて同符号になります。最適化の効率が下がり——ネットワークは重み空間の特定方向にしか動けません。

Tanh は -1 から 1 の値を出力し、ゼロ中心です。*ゼロ中心出力* により、活性化は正負どちらも取れ、勾配は任意方向を指せます。一部のアーキテクチャ、特に系列を処理する RNN では、この対称性が安定した学習に重要です。

トレードオフ？ Tanh は依然として勾配消失に苦しみますが、シグモイドより程度は軽いです。ReLU より計算が高コスト（指数が必要）。万能置換ではありません——特定の性質（ゼロ中心、有界出力、滑らかな勾配）がアーキテクチャに有益なとき使われます。

==== その他の現代バリアント: 特化した解

Leaky ReLU と Tanh を超えて、研究者は特定シナリオ向けに最適化された活性化関数を開発しました:

*PReLU (Parametric ReLU)*: Leaky ReLU に似るが、負の傾きが固定ではなく学習中に学習される。柔軟性が増す——ネットワークが各レイヤでどれだけ負の情報を保持するか決められる。

*ELU (Exponential Linear Unit)*: 負の入力で負の値を出力（Leaky ReLU のように）するが、線形傾きの代わりに滑らかな指数曲線を使う。非常に深いネットワークで学習安定性に役立つことがある。

*GELU (Gaussian Error Linear Unit)*: GPT や BERT のような現代 transformer アーキテクチャで使用。確率的正則化のアイデアに基づき、巨大言語モデルの学習に役立つと思われる滑らかで非単調な性質を持つ。

これらの各バリアントは、特定アーキテクチャの特定学習課題から生まれました。任意の選択ではなく——異なる文脈で異なる活性化特性が学習ダイナミクスにどう影響するかの深い理解を反映しています。

=== 推論にとってなぜ重要か

「これらの活性化関数は学習問題を解くために設計されたのに、推論でなぜ重要？」と思うかもしれません。

良い質問です。推論で理解が重要な理由:

1. *実モデルでこれらの関数に遭遇する* - 事前学習済みモデルを読み込むとき、使われている活性化関数を実装する必要があります。なぜ存在し、なぜ必要かを理解すると、いつ何を最適化するかに目を向けて正しく実装できます。

2. *性能特性が異なる* - ReLU はシグモイドよりはるかに速い（指数不要）。秒間数千回推論するとき重要。計算コストを知ると Vulkan シェーダを最適化できます。

3. *数値安定性* - 一部の活性化関数は他より数値問題（オーバーフロー、アンダーフロー）を起こしやすい。挙動を理解すると堅牢な推論コードを書けます。

4. *アーキテクチャ理解* - 異なるモデルアーキテクチャは良い理由で異なる活性化関数を使います。一部レイヤで tanh、他で ReLU を使うモデルは任意ではない——学習ダイナミクスとモデルが学んだ関数を反映しています。

Vulkan 実装では、活性化関数の選択がコンピュートシェーダで実装すべき演算に影響します。ReLU は自明（単一 max 演算）、シグモイドは指数計算が必要。後の章で実装詳細を扱いますが、今はモデルが異なる活性化関数を使う *理由* と、それが表すトレードオフを理解しました。

== まとめ

ニューラルネットワークは任意の他の関数を近似できる数学的関数であり、データ内の未知の関係を発見する強力な道具です。生物学的ニューロン——複数入力を取り、閾値に達したかどうかに基づいて単一出力を生成——に触発を得ています。

重要な洞察は、線形関数だけでは不十分——ネットワークの深さに関わらずより単純な形に崩れる——ということです。シグモイド、ReLU、tanh のような非線形活性化関数が、ニューラルネットワークが複雑な関数を近似するために不可欠です。

ニューラルネットワークは汎用的で強力ですが、コストがあります: 既知関数の直接実装と比べて計算コストが高い。この計算コストが、リアルタイム推論に Vulkan による GPU 加速が不可欠になる理由です。

この根本レベルでのニューラルネットワークの働きの理解ができたので、学習と推論の区別を探り、GPU 上で推論を実行することが何を意味するか理解する準備ができました。
