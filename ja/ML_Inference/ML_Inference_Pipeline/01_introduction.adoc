= ML 推論パイプライン: はじめに

[NOTE]
====
本章は、機械学習の概念を第一原理から教える背景の「教科書」節です。非常に密度が高く完全な読み物を意図しています。すでに機械学習に詳しい学生は、次章 xref:ML_Inference/Third_Party_Libraries/01_introduction.adoc[サードパーティ ML ライブラリの統合] へ進んでも構いません。
====

== 導入

Vulkan チュートリアルを完了しました。グラフィックスパイプラインを作り、メモリを管理し、コンピュートシェーダをディスパッチできます。GPU に望むことをさせる方法を理解しています。今、機械学習を見てこう考えています: どうつながるのか？ 学習済みニューラルネットワークを取り、Vulkan を使って GPU 上で実際にどう実行するのか？

それが本章の主題です。ニューラルネットワークの学習の数学ではありません——それはまったく別の獣です。焦点は *推論*: すでに誰かが学習したモデルを取り、GPU 上で効率的に動かすことです。こう考えてください: 誰かがエンジンを作り、調整し、鍵を渡してくれた。あなたの仕事は速く走ることです。

しかしこうです: ニューラルネットワーク向けコンピュートシェーダの執筆にすぐ飛び込めません。ニューラルネットワークが実際に *何であるか*——どんな操作を行い、なぜそれが重要で、データがどう流れるか——を理解していなければ、盲目でコードを書くことになります。何か動くものは得られるかもしれませんが、なぜ動くか、壊れたときにどう直すかが分かりません。

本章は必要な基盤を築きます。機械学習とは何か、ニューラルネットワークが第一原理からどう動くか、推論が何を意味するか、ニューラルネットワークを GPU コンピュート操作としてどう考えるかを探ります。終わりまでに、高レベルの ML 概念と書く低レベル Vulkan コードをつなぐ明確なメンタルモデルを持てます。

== 本章で扱う内容

「機械学習とは何か？」から「GPU 実行にどう写すか？」への旅をします。経路は次のとおりです。

まず、*機械学習が実際に何か* を話します。誇張でもバズワードでもなく、本物です。コンピュータはどうデータからパターンを学ぶのか？ なぜ有用なのか？ 実アプリケーション、特にグラフィックスとリアルタイムシステムではどこに現れるのか？ この文脈は、GPU 加速がなぜ重要か、Vulkan ベース推論がどこで意味を持つかを理解するのに役立ちます。

次に、*ニューラルネットワークをゼロから* 掘り下げます。単純な数学関数から始め、複雑なネットワークへ積み上げます。生物学的ニューロンが人工ニューロンをどう触発したか、線形関数だけではなぜ足りないか、活性化関数がなぜ不可欠かを見ます。単一ニューロンで AND ゲートを作るような具体例を通して、各ステップで何が起きているかを正確に理解します。抽象理論ではなく、書くコンピュートシェーダの基盤です。

次に、*学習と推論* の決定的な区別を明確にします。学習は学びが起きる場所——勾配降下と逆伝播で数百万の重みを調整する場所です。推論は、それらの学習済み重みを使って予測する場所です。焦点は推論ですが、学習を少し理解すると、何を実装しているか、なぜモデルがそのような構造かをよりよく評価できます。

*転移学習* を探ります——ほぼゼロからモデルを学習することはないという実務的現実です。代わりに事前学習済みモデルを取り、ニーズに合わせて適応します。これがどう動きなぜ強力かが分かると、Vulkan 推論プロジェクトへの取り組み方が形作られます。

次に *モデル形式*: TFLite と ONNX を扱います。これらはデプロイ用に学習済みモデルをパッケージするファイル形式です。これらのファイルに何が入っているか、どう違うか、どう変換するかを理解する必要があります。Vulkan アプリにモデルを読み込むときすぐに使う実務知識です。

最後に、*GPU 実行のメンタルモデル* を構築します。ニューラルネットワーク——この抽象的な数学的もの——はどうバッファ、コンピュートシェーダ、パイプラインバリアに写るのか？ メモリ管理、同期、性能をどう考えるか？ このメンタルモデルが ML 概念と Vulkan 実装の隙間を埋めます。

== 対象読者

本章は、メインの Vulkan チュートリアルを完了し、コンピュートシェーダに慣れていることを前提とします。コマンドバッファの作成、コンピュート作業のディスパッチ、GPU メモリの管理、操作の同期ができる必要があります。「パイプラインバリア」や「ディスクリプタセット」といった用語が意味をなすなら、準備はできています。

機械学習の経験は不要です。ゼロから築きます。ただし PyTorch や TensorFlow のような ML フレームワークを使ったことがあれば、それらのフレームワークがすることと Vulkan で実装することの点をつなぎやすいでしょう。

数学的には、基本的な線形代数——ベクトル、行列、行列乗算——に慣れている必要があります。関数とその性質の理解が役立ちます。微積分は必須ではありませんが、学習の説明で勾配のような概念に触れます（推論の理解に役立ちます）。

== なぜ理論に手間をかけるのか？

こう考えているかもしれません: 「推論を動かしたいだけなのに、なぜこの背景をすべて理解する必要があるのか？」妥当な問いです。重要な理由は次のとおりです。

*デバッグ*: 推論結果が間違うとき——いつか必ず間違います——どこを見るか知る必要があります。前処理の問題か？ レイヤ実装のバグか？ 同期の問題か？ 各レイヤが数学的に何をすべきかを理解すると、どこでおかしくなったかを特定しやすくなります。

*最適化*: ニューラルネットワークにはボトルネックがあり、必ずしも予想どおりの場所にはありません。操作を理解すると、何が実際に高コストか、どの最適化が効くかを特定できます。操作を融合すべきか？ メモリレイアウトを変えるべきか？ 異なるワークグループサイズを使うべきか？ 操作が何をするかを理解せずには答えられません。

*適応*: 新しいモデルアーキテクチャは絶えず現れます。基礎——畳み込みが何をするか、注意がどう動くか、正規化がなぜ重要か——を理解していれば、新しいレイヤ型が現れたときに実装できます。最初に実装した操作にロックされません。

*意思決定*: カスタム Vulkan 推論が正しい選択のときもあります。そうでないときもあります。トレードオフ——カスタム実装が意味をなすとき対既存ライブラリを使うとき——を理解するには、実際に何を実装しているかを理解する必要があります。

これらの基礎を理解するために投資した時間は、問題をデバッグするたび、性能を最適化するたび、新しいモデルアーキテクチャを実装するたびに報われます。

== 本章の使い方

節を順に読んでください——それぞれが前の上に積み上がります。概念にすでにしっかりしているのでなければ、先へ飛ばさないでください。本章は単純から複雑へ、理解を段階的に築くよう設計されています。

例に時間をかけてください。ニューロンで AND ゲートを作るとき、実際に自分で数学を通してください。畳み込みの仕組みを説明するとき、操作をたどってください。具体例が、抽象概念を現実にする場所です。

概念で行き詰まるのは普通です。ニューラルネットワークは多くの動く部品を伴います。戻り、再読し、例をもう一度通してください。理解への投資は、実装を始めるときに報われます。

そして覚えておいてください: 本章はメンタルモデルの構築についてです。まだ何も実装しません——それは後の章です。今は、実装が意味をなす基盤を築いています。

== 内容

続く節で扱う内容は次のとおりです。

*xref:ML_Inference/ML_Inference_Pipeline/02_what_is_machine_learning.adoc[機械学習とは何か？]* - 基本から始めます: 機械学習とは何か、従来のプログラミングとどう違うか、実アプリケーションのどこで使われるか。例から学ぶことをコンピュータに教える力がなぜ強力か、リアルタイム推論に GPU 加速がなぜ重要かを理解します。

*xref:ML_Inference/ML_Inference_Pipeline/03_neural_networks_fundamentals.adoc[ニューラルネットワークの基礎]* - ゼロから理解を築く場所です。数学関数と生物学的触発から始め、人工ニューロンを構成し、非線形活性化関数がなぜ不可欠かを探り、具体例を通します。ニューラルネットワークが実際に何を計算し、なぜそのような構造かを理解します。

*xref:ML_Inference/ML_Inference_Pipeline/04_inference_explained.adoc[推論の説明]* - 学習と推論を区別し、計算プロセスとしての推論が何を意味するかを探ります。推論中にデータがネットワークをどう流れるか、前処理と後処理が何を伴うか、ニューラルネットワークを GPU コンピュート操作の列としてどう考えるかを学びます。

*xref:ML_Inference/ML_Inference_Pipeline/05_transfer_learning.adoc[転移学習]* - 実務的現実として、ゼロからモデルを学習することはまれです。本節は事前学習済みモデルの活用、ニーズへの適応、このアプローチがなぜ強力かを説明します。転移学習を理解すると、実世界の推論プロジェクトへの取り組み方が形作られます。

*xref:ML_Inference/ML_Inference_Pipeline/06_model_formats.adoc[モデル形式]* - ニューラルネットワークはデプロイ用に様々な形式でパッケージされます: TFLite と ONNX。これらの形式に何が含まれるか、どう違うか、いつどれを使うか、どう変換するかを学びます。学習済みモデルを扱うときすぐに使う実務知識です。

*xref:ML_Inference/ML_Inference_Pipeline/07_mental_model_gpu_execution.adoc[メンタルモデル: GPU 実行]* - 最後に、ニューラルネットワークの概念を GPU 実行に写すメンタルモデルですべてをつなぎます。レイヤはどうコンピュートシェーダになるか？ 中間結果のメモリをどう管理するか？ 操作をどう同期するか？ このメンタルモデルが ML 理論と Vulkan 実装の隙間を埋めます。

各節が前の上に積み上がり、何を実装しなぜかを完全に描きます。本章の終わりまでに、続く章の実務実装の詳細に潜る準備ができます。

機械学習が実際に何か、GPU コンピュートになぜ重要かから理解を始めましょう。
