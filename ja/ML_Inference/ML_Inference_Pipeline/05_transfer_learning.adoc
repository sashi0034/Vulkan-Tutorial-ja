= 転移学習

ギターを何年も弾いた後にピアノを学ぶと想像してください。ゼロから始めるわけではありません——リズム、楽譜の読み方、両手の協調は既に理解しています。一つの楽器から別の楽器へ知識を転移しています。技法は違いますが、その基礎知識が大きなアドバンテージを与えます。

これがまさに転移学習がニューラルネットワークにすることです。新しいタスクごとにゼロからモデルを学習する代わりに、有用なパターンを既に学んだモデルを取り、特定のニーズに適応させます。機械学習を身近にする最も実用的なブレークスルーの一つです。

== 問題: 学習は高コスト

ニューラルネットワークをゼロから学習するために実際に必要なことを正直に見ましょう。時間、お金、データ、専門知識のコストは相当なもので、多くの個人や組織にとってゼロからの学習は手の届かないものにします。

=== 計算コスト: 時間とハードウェア

コンピュータビジョンの標準ベンチマークの一つ ImageNet を例に取ります。2 万カテゴリ以上、1400 万枚超の画像を含みます。ResNet-50 のような現代のニューラルネットワークをこのデータセットでゼロから学習するには:

*学習時間*: 単一のハイエンド GPU（NVIDIA V100 など）で ImageNet 上の ResNet-50 学習はおおよそ 2〜3 週間。336〜504 時間の連続計算。コンシューマ GPU（RTX 3080 など）では 4〜6 週間かかることも。

しかも最初からすべてうまくいく前提です。実務では異なるハイパーパラメータ、アーキテクチャ、学習戦略を試します。各実験がまた数週間の学習実行。最先端モデルを学習する研究者は、最良構成を見つけるまで数百の実験を走らせることもあります。

*ハードウェア要件*: 十分なメモリを持つ GPU が必要。ResNet-50 の学習には合理的なバッチサイズで少なくとも 11GB の GPU メモリ。ResNet-152 や EfficientNet-B7 のような大モデルは 16GB 以上。コンシューマ GPU はメモリ不足で、小さいバッチサイズ（学習が遅くなり最終精度を損なう）かクラウド GPU レンタルを強いられます。

クラウド GPU について: AWS の NVIDIA V100 は約 3 ドル/時間。3 週間の ResNet-50 学習で 1,512 ドル。それは一回の学習実行だけ。異なる構成を試すなら、実験数でコストを掛け算。

*エネルギーコスト*: フル負荷のハイエンド GPU は 250〜350 ワット消費。3 週間連続稼働でおおよそ 125〜175 kWh。典型的な電気料金で 15〜25 ドル——単体では大きくないが、複数実験で積み上がる。大規模学習（GPT-3 のように数千 GPU を数週間）ではエネルギーコストは数十万ドルに達することも。

=== データ要件: 数百万の例

ゼロからの学習には大規模データセットが必要。ImageNet は 1400 万画像。1400 万のランダム写真ではなく——各画像が正しいカテゴリで慎重にラベル付けされています。このようなデータセット作成には:

*収集*: 数百万画像の収集。ImageNet では Web からのスクレイピングが含まれ、著作権・倫理的懸念を生んだ。独自データセットなら、タスクに関連する数百万例の収集または生成が必要。

*ラベル付け*: 各画像にラベルが必要。ImageNet ではクラウドソーシング（Amazon Mechanical Turk）で、数十万ドル規模。ラベル 0.01 ドルでも 1400 万ラベルで 14 万ドル。単純分類だけ——物体検出（バウンディングボックス）やセグメンテーション（全ピクセルラベル）は画像あたりはるかに高コスト。

*品質管理*: ラベルは正確である必要。画像ごとに複数ラベラー、検証、誤り修正。高品質ラベルの実コストは生ラベルコストの 10〜100 倍になることも。

ゲームの武器認識システムに、数百万枚のラベル付き武器画像はありますか？ 収集とラベル付けの余裕は？ おそらくありません。

=== 専門知識要件: 「学習」ボタンを押すだけではない

ニューラルネットワークの学習は科学であると同時に芸術です。数十の判断が必要:

*アーキテクチャ*: レイヤ数？ レイヤ型？ レイヤあたりフィルタ数？ これらは学習時間と最終精度の両方に劇的に影響。良い選択にはトレードオフ理解としばしば多くの試行。

*ハイパーパラメータ*: 学習率、バッチサイズ、重み減衰、モメンタム、学習率スケジュール、データ拡張戦略、オプティマイザ選択（SGD、Adam、RMSprop?）、初期化戦略など。それぞれ学習に大きな影響があり、複雑に相互作用。良いハイパーパラメータは広範な実験がしばしば必要。

*学習ダイナミクス*: 損失が遅すぎて減る？ 学習率が低すぎるかも。速く減って頭打ち？ 高すぎるかも。損失が激しく振動？ バッチサイズが小さすぎるかも。過学習？ 正則化またはデータ拡張を増やす。未学習？ モデルが単純すぎるか、もっと長く学習。

これらの問題の診断と修正には経験が必要。初心者はしばしば数週間〜数ヶ月を効果的な学習の習得に費やす。専門家でも特に難しいデータセットやアーキテクチャで苦労することあり。

=== 機会コスト

ハードウェア、データ、専門知識があっても機会コストがあります。ゼロからモデルを学習する数週間〜数ヶ月は、ゲームデザイン、グラフィックス最適化、ゲームプレイ、UX など他の側面に費やせない時間です。

多くのアプリケーションでは、事前学習済みモデルが利用可能なときゼロから学習する価値は単にありません。ここで転移学習が入ります。

=== 具体例: ゲームの武器認識

具体化しましょう。ゲームを作り、ゲーム内スクリーンショットから 20 種類の武器を認識するモデルが欲しいとします。

*ゼロから学習*: 武器タイプあたり数千画像（合計 4 万枚以上）、数週間の GPU 時間、学習の専門知識、良好な精度のための複数反復。総コスト: GPU 時間とラベル付けで 5,000〜10,000 ドル、加えて数ヶ月の時間。

*転移学習*: 事前学習済み ImageNet モデル（無料、既に学習済み）から開始。武器タイプあたり 50〜100 画像（合計 1,000〜2,000——ゲームエンジンから生成可能）。コンシューマ GPU で数時間ファインチューニング。総コスト: 数日の時間、最小限の GPU コスト。

差は劇的。転移学習が不可能を実用的にします。

== 洞察: 特徴は再利用可能

転移学習を機能させる鍵の洞察: ニューラルネットワークが学ぶ特徴はしばしばタスク間で転移可能。最初は自明ではない——猫認識で学習したネットワークがなぜ車認識に役立つ？ しかしニューラルネットワークが実際に何を学ぶか理解すると、転移可能性が明確になります。

=== 階層的特徴学習: 単純から複雑へ

ニューラルネットワークは一枚岩の「猫検出器」や「犬検出器」を学ぶわけではありません。代わりに特徴の階層を学び、ネットワークが深くなるにつれ段階的により複雑で抽象的になります。

典型的な画像分類ネットワークがレイヤごとに学ぶものをたどりましょう:

*レイヤ 1（入力に最も近い）*: 最初の畳み込み層は非常に基本的な視覚プリミティブを検出。フィルタの応答を可視化すると:

- 水平エッジ（暗から明、明から暗への遷移）
- 垂直エッジ
- 様々な角度の対角エッジ
- 色ブロブ（赤、緑、青、特定色組み合わせの領域）
- 単純テクスチャ（点、線、グラデーション）

これらはすべての視覚知覚の構成要素。内容に関わらずすべての画像はエッジと色領域で構成。猫の写真にもエッジ。車にも。医療 X 線にも。これらの特徴は普遍的。

*レイヤ 2-3（初期中間層）*: 第二・第三層は基本エッジをややより複雑なパターンに合成:

- コーナー（二エッジの交点）
- 曲線と円弧
- 単純形状（円、矩形、三角）
- 基本テクスチャ（縞、格子、斑点）
- 色組み合わせとグラデーション

再び汎用特徴。コーナーは建物、顔、車両、ゲームオブジェクトに現れる。曲線は有機形状、車輪、建築要素に。特定物体カテゴリに固有ではない。

*レイヤ 4-6（中間層）*: ここから興味深い。中間層は単純形状をより複雑なパターンに合成:

- 反復テクスチャ（毛、鱗、布地、木目）
- 物体部分（目、車輪、窓、脚）
- 幾何パターン（対称、反復、整列）
- 材質特性（光沢 vs マット、粗 vs 滑らか）

ドメイン固有になり始めるが依然として広く有用。「毛のテクスチャ」は猫、犬、熊、毛むくじゃら生物全般の認識に役立つ。「車輪状円形」は車、自転車、機械の認識に。「目様パターン」は人間、動物、漫画キャラの顔に。

*レイヤ 7-10（後期中間層）*: より抽象的なパターン:

- 複雑物体部分（顔、車体、建物ファサード）
- 空間関係（目は鼻の上、車輪は車体の下）
- 文脈パターン（草は屋外、タイルは屋内を示唆）
- スタイル・外見バリエーション（犬種、車モデル）

よりタスク固有だがドメイン内では依然転移可能。ImageNet（多くの動物カテゴリ）で学習していれば、様々な動物部位、毛パターン、ポーズを認識するよう学んでいる。この知識は ImageNet にない新種動物の認識にもうまく転移。

*最終層（出力に最も近い）*: 真にタスク固有なのは最後の数層だけ。これらがすべての以前の特徴を合成して最終判断:「この特徴組み合わせ（毛テクスチャ + 尖った耳 + ひげ + 猫顔）= 猫」。

転移学習ではこれら最終層を置き換え。それ以前——すべての汎用特徴——は保持。

=== なぜこの階層が出現するか

なぜニューラルネットワークはこの特定の階層を学ぶ？ 最初からタスク固有特徴を学ばない？

答えは学習方法にあります。前章の勾配降下を思い出してください。ネットワークは誤差を減らすよう重みを調整。早期層は後続すべてに影響するため、多くの異なるパターンに有用な特徴を学ぶよう最適化。「エッジ検出器」を一度学んで全物体で再利用する方が、猫、犬、車ごとに別エッジ検出器を学ぶより効率的。

人間知覚と類似。視覚野にはエッジを検出するニューロンがあり、どの物体のエッジかに関わらない。高レベルニューロンがエッジ検出を合成して特定物体を認識。進化が階層的特徴学習が視覚情報処理の効率的な方法と発見し、ニューラルネットワークも学習を通じて同じ原理を発見。

=== 特徴再利用性の可視化

研究者はニューラルネットワークレイヤが何を学ぶか可視化する技法を開発。一手法は特定ニューロンを最大限活性化する画像を見つけること。ImageNet 学習ネットワークで:

*レイヤ 1 ニューロン* は単純パターン——水平エッジのある任意画像で強く発火、猫、車、建物を問わず。

*レイヤ 5 ニューロン* はテクスチャ——毛（猫、犬、熊）、金属表面（車、飛行機、家電）など。

*レイヤ 10 ニューロン* は特定物体部分——顔（人、猫、犬）、車輪（車、自転車、バイク）、文字（看板、本、画面）。

重要な観察: レイヤ 1-10 は多くの物体カテゴリに現れる特徴を学ぶ。ImageNet の正確なカテゴリに固有なのは最終層（12 層ネットワークなら 11-12）だけ。

=== 実務的含意

この階層構造が転移学習がうまく機能する理由。ImageNet 学習ネットワークをゲーム武器認識に適応するとき:

*レイヤ 1-3*（エッジ、コーナー、形状）は既に完璧。武器も他物体と同様エッジとコーナーを持つ。

*レイヤ 4-7*（テクスチャ、材質、物体部分）は大部分が有用。金属、木、幾何パターン——武器に関連。

*レイヤ 8-10*（複雑パターン、空間関係）は多少適応が必要かも。動物部位、車両コンポーネントを学んだが、複雑パターンと空間関係を認識する一般的な能力は転移。

*最終層*（特定分類）は完全に置き換え。「猫」「犬」ではなく「剣」「ライフル」「弓」を出力。

レイヤ 1-10 を保持し新しい最終層だけ学習すれば、数百万ドル相当の ImageNet 学習計算を活用し、最小限の追加学習でタスクに適応。

=== クロスドメイン転移: どこまで行けるか

タスクが元学習タスクに類似するほど、特徴はより転移。しかしかなり異なるドメイン間でも転移学習は機能:

*自然画像 → ゲームグラフィックス*: 非常にうまく機能。スタイル化やレンダリングでもエッジ、テクスチャ、物体構造は残る。特徴はほぼ完全に転移。

*自然画像 → 医療画像*: 驚くほどうまく機能。X 線・CT は写真と大きく異なるがエッジ、テクスチャ、解剖構造は残る。低レベル特徴は完全に転移、中レベル（塊状、線状構造）も有用。

*自然画像 → 衛星画像*: うまく機能。視点とスケールは異なるが建物、道路、植生は認識可能なテクスチャとパターン。

*英語テキスト → 他言語*: 構造が類似な言語（西語、仏語、独語）でうまく機能。大きく異なる（中国語、アラビア語）では劣るがゼロからより良い。

一般則: タスクが類似低レベルパターン（画像のエッジ、テキストの語パターン）を含めば、転移学習は大きく助ける。ドメインがより異なるほど、特徴抽出だけでなくファインチューニングがより必要。

== 転移学習の仕組み

基本アプローチは驚くほど単純:

大規模汎用データセットで事前学習済みモデルから開始。コンピュータビジョンではしばしば ImageNet。自然言語では書籍・Web から数十億語。これらのモデルはドメインの基本特徴を既に学んでいる。

次に事前学習済みモデルを特定タスクに適応。正確なアプローチは状況次第——データ量、元タスクとの差、使える計算資源。

主な戦略を単純から複雑へ探ります。

== 特徴抽出: 高速アプローチ

最も単純な戦略は事前学習済みモデルを固定特徴抽出器として使うこと。事前学習済みモデルは生画像を豊かな特徴表現へ変換するよう学習済み。その表現を使い、上に独自分類器を載せる。

=== 特徴抽出の仕組み

ステップごとに:

*ステップ 1: 事前学習済みモデルから開始*: ImageNet 学習モデル——例 ResNet-50——をダウンロード。50 層、1000 ImageNet カテゴリの確率出力。

*ステップ 2: 最終層を除去*: 最後は 1000 出力の全結合層。これを除去。残るのは画像を 2048 次元特徴ベクトル（ResNet-50 では倒数第二層 2048 ニューロン）へ写す特徴抽出器。

*ステップ 3: 残り全層を凍結*: 残り全層を「凍結」「非学習可能」に。重みは学習中変わらない。固定特徴抽出器。

*ステップ 4: 独自分類器を追加*: 上に新しい全結合層。20 武器タイプなら 20 出力。ランダム重みから開始——まだ何も知らない。

*ステップ 5: 新層だけ学習*: 凍結特徴抽出器に学習データを通し、2048 次元特徴ベクトルを得、新最終層だけを 20 武器カテゴリへ写すよう学習。

=== なぜ速いか

学習が劇的に速い理由:

*パラメータが少ない*: ResNet-50 全体 2500 万ではなく、おそらく 4 万（2048×20）。600 倍少ない。

*データが少なくてよい*: パラメータが少ないほど過学習回避に必要データも少ない。クラスあたり数百例で足りることも、数千ではなく。

*反復が速い*: 50 層全部ではなく 1 層の勾配だけ。全日かかる学習が最終層だけなら時間〜分。

*GPU 不要も*: CPU でも可能。凍結ネットワークで一度特徴抽出（バッチ可）、ディスク保存、保存特徴で最終層学習。特徴抽出は GPU、学習ステップは軽量で CPU 可。

=== 特徴抽出が最適な場合

*類似ドメイン*: 事前学習モデルと同ドメイン。自然写真で学習、自然写真で作業（物体が違っても）なら特徴はうまく転移。

*データが限られる*: クラスあたり数百例、数千ではない。ファインチューニングより少ないデータで機能。

*計算資源が限られる*: 学習用 GPU なし、または速い結果が必要。控えめなハードウェアで可。

*迅速なプロトタイピング*: ML がタスクで機能するか素早くテスト。数時間で動くモデル。

=== 例: ゲーム武器認識

ゲームを作り、スクリーンショットから 20 武器タイプを認識。

*データ収集*: ゲームエンジンから武器タイプあたり 100 スクリーンショット（合計 2000）。数時間。

*特徴抽出*: 事前学習 ResNet-50 を読み込み、最終層除去、2000 画像を通す。各画像 2048 次元特徴ベクトル。GPU で数分。

*学習*: 単純分類器（ロジスティック回帰または小 NN）で 2048 次元ベクトルを 20 カテゴリへ。4 万パラメータだけ、CPU で数分。

*結果*: 合計数時間で動く武器認識モデル。最小 GPU、深層学習専門知識不要。

=== 限界

*特徴の不一致*: ドメインが大きく異なるなら特徴が無関係かも。自然写真モデルは医療 X 線や抽象的ゲームグラフィックスから有用特徴を抽出しないかも。

*適応なし*: 特徴は固定。テクスチャ重視、異なる空間スケールなど微妙に異なる特徴が有益なら、特徴抽出は適応できない。

*精度の上限*: 事前学習特徴がタスクをどれだけよく表現するかに限定。ファインチューニングはしばしば 5〜10% 精度向上。

限界があっても、特徴抽出はしばしば正しい出発点。速く、単純、多くのタスクで驚くほどうまく機能。

== ファインチューニング: バランス型アプローチ

ファインチューニングは一歩進む。事前学習モデル全体を凍結する代わり、後半レイヤを特定タスクに適応させる。特徴抽出より柔軟、事前学習知識も活用。

=== ファインチューニングの仕組み

特徴抽出より細やか:

*ステップ 1: 特徴抽出から開始*: 全体凍結、新最終層追加、その層だけ学習。ベースラインと、ファインチューニング前に新層が妥当な重みを持つ保証。

*ステップ 2: 後半レイヤを解凍*: 末端から段階的に解凍。50 中最後 5 層、最後 20%、最後数ブロックなど。データと元タスクとの差次第。

*ステップ 3: 低い学習率*: 極めて重要。事前学習重みは既に良好——大更新で破壊したくない。ゼロから学習率の 1/10〜1/100。0.01 なら 0.001 または 0.0001。

*ステップ 4: データで学習*: 解凍層がタスクに適応、凍結層は汎用特徴を保持。

*ステップ 5: 必要ならさらに解凍*: データと学習が順調なら追加層解凍、さらに低い学習率。段階的解凍が破滅的忘却（事前学習で学んだことを忘れる）防止に役立つ。

=== 低い学習率が重要な理由

*事前学習重みは良好*: 大きい学習率更新は知識を破壊し、事前学習を本質的に「忘却」。

*小さな調整で足りる*: ゼロからではなく、既に良好な特徴への小調整。「毛テクスチャ」検出ニューロンは「金属テクスチャ」向けにわずかに調整すればよく、テクスチャ検出を一から再学習する必要なし。

*破滅的忘却の回避*: 大きい学習率は小データセットに過学習し汎用知識を忘れる。小さい学習率は大部分の事前学習知識を保持しつつ適応。

一般的な戦略は *判別的学習率*: レイヤごとに異なる学習率。早期層（汎用特徴）は非常に小（または凍結）、後半層はやや大。例:

- レイヤ 1-30: 凍結（学習率 = 0）
- レイヤ 31-40: 学習率 = 0.0001
- レイヤ 41-50: 学習率 = 0.001
- 新最終層: 学習率 = 0.01

=== ファインチューニングが最適な場合

*中程度のデータ*: クラスあたり数千例。複数層学習に足り、深刻な過学習せず、ゼロから学習には不足。

*やや異なるドメイン*: 関連するが同一ではない。低レベル特徴は転移、高レベルは適応必要。

*より高い精度が必要*: 特徴抽出 85% だが 90%+ 必要。ファインチューニングが差を埋めることが多い。

*GPU 利用可*: 複数層学習は GPU ではるかに速い。CPU も可能だが長い。

=== 例: 医療画像分類

より困難: 医療画像（X 線）を疾患カテゴリに分類。

*課題*: X 線は自然写真と大きく異なる。ドメインシフトが大きい。

*特徴抽出試行*: ImageNet ResNet-50 で特徴抽出。70% 精度——ランダムより良いが医療には不足。

*ファインチューニング*:
1. 特徴抽出ベースライン（70%）
2. ResNet-50 最後 10 層解凍
3. 学習率 0.0001 で 20 epoch
4. 結果: 82%——大きな改善

*なぜ機能したか*: 早期層（エッジ、テクスチャ）は自然画像から X 線へうまく転移。後半層は「動物部位」「物体形状」から「解剖構造」「疾患パターン」へ適応必要。ファインチューニングが有用低レベル特徴を保ちつつ適応を許した。

=== トレードオフ

*より多くのデータが必要*: 数千例必要、数百ではない。データ少なすぎると過学習。

*より長い学習時間*: 複数層は 1 層より長い。特徴抽出が分ならファインチューニングは時間。

*ハイパーパラメータへの敏感さ*: 学習率、解凍層数、学習期間が結果に大きく影響。正しい組み合わせは実験必要。

*破滅的忘却のリスク*: 学習率高すぎ、データ少なすぎで解凍しすぎると、事前学習知識を忘れ特徴抽出より悪化。

課題があっても、ファインチューニングはしばしば価値がある。精度改善（典型的に特徴抽出比 5〜15%）は本番で重要なことも。

== ドメイン適応: 複雑アプローチ

タスクが事前学習モデルと根本的に異なるドメインのとき。自然写真で学習、医療 X 線で作業。英語テキストで学習、全く異なる構造の言語が必要。実写真と無関係に見える合成ゲームグラフィックス。

ドメイン適応技法はドメインシフトが単純ファインチューニングでは効果的に扱えないほど大きいときギャップを埋めようとする。

=== ドメインシフト問題

*ドメインシフト* はデータが事前学習モデルの学習データと異なる分布から来るとき。現れ方:

*視覚ドメインシフト*:

- 自然写真 → 医療画像（X 線、CT、MRI）
- 実写真 → 合成レンダ（ゲームグラフィックス、3D モデル）
- 昼間写真 → 夜間写真
- 晴天 → 霧・雨

*テキストドメインシフト*:

- 正式文 → 非正式文（SNS、チャット）
- 現代英語 → 歴史的英語
- 一般文 → 専門用語（法律、医療、技術）

ドメインシフトが大きいと、ファインチューニングでも苦戦。一ドメインで学んだ特徴は他へうまく転移しない。

=== ドメイン適応技法

活発な研究分野。主な戦略:

*敵対的ドメイン適応*: ソースドメイン（元学習データ）とターゲットドメイン（自分のデータ）で区別不能な特徴を生成するよう学習。判別器がドメインを当て、特徴抽出器が騙す。ドメイン不変特徴を強制。

*自己教師あり学習*: 事前学習モデルでラベルなしターゲットドメインデータを予測。高信頼度予測を疑似ラベルとして学習セットに追加、再学習。繰り返し段階的適応。

*マルチタスク学習*: 元タスク（ソースドメインデータ）と自分のタスク（ターゲットドメインデータ）を同時学習。共有層が両ドメインに有用特徴を学ぶ。

*スタイル転送*: ターゲットドメインデータをソースドメインに見えるよう変換（または逆）。合成ゲームグラフィックスを自然写真風にして自然写真学習モデルを使用。

=== ドメイン適応が必要な場合

多くの実務、特にグラフィックスとゲームでは、ドメイン適応は不要。一般的タスク向け事前学習モデルはしばしばドメインに十分近く、特徴抽出またはファインチューニングでうまく機能。

必要かもしれない場合:

*極端な視覚的差異*: 自然写真と根本的に異なる見た目。医療画像、衛星画像、高度にスタイル化されたゲームグラフィックス。

*事前学習モデルなし*: ドメイン向け事前学習モデルなし、ゼロから学習不可。

*ファインチューニング失敗*: 数千例でファインチューニング試したがドメインシフト大きすぎて精度が低い。

ゲーム開発ではめったに必要ない。スタイル化ゲームグラフィックスも自然写真と十分低レベル特徴（エッジ、テクスチャ、形状）を共有し、ファインチューニングはうまく機能。

== 推論にとってなぜ重要か

「推論に焦点、学習ではない。なぜ転移学習を知る必要？」

転移学習の理解は、自分で学習しなくても、モデルデプロイ成功に不可欠。

=== モデル選択とアーキテクチャ理解

TensorFlow Hub、PyTorch Hub、Hugging Face などモデル zoo を閲覧すると、アーキテクチャ（ResNet-50、MobileNet、EfficientNet など）と学習/ファインチューニング内容で説明されたモデルが見える。

転移学習理解が情報に基づく選択を助ける:

*アーキテクチャの一貫性*: 特定タスク向けファインチューニング ResNet-50 は元 ResNet-50 と同アーキテクチャ。同じ推論実装を両方に使える。レイヤ型、テンソル形状、演算は同一——重み値だけ異なる。

*モデルサイズ vs 精度トレードオフ*: 事前学習モデルはサイズ/精度トレードオフの異なるファミリー。MobileNet-V2 は 0.35x〜1.4x 幅倍率バリアント。同アーキテクチャで容量だけ異なる理解が、ハードウェア制約向け選択を助ける。

*転移学習の来歴*: 「ResNet-50 fine-tuned on COCO」は ImageNet 事前学習から始め COCO で物体検出ファインチューニングしたことを示す。この履歴が得意分野と期待前処理に影響。

=== 前処理要件: 重要詳細

多くの推論実装がここで誤る。推論前処理は学習（ファインチューニング含む）前処理と *まったく同じ* である必要がある。

*正規化値*: ImageNet 正規化（mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]）でファインチューニングなら推論でも同値。異なる正規化は分布外入力を見せ無意味な予測。

*リサイズ戦略*: 中央クロップでファインチューニングなら推論も中央クロップ。レターボックスならレターボックス。ミスマッチは精度を大きく低下。

*色空間*: RGB 期待に BGR 提供なら赤と青が逆。一部モデルでは予測完全破壊。

*入力範囲*: [0, 1]、[-1, 1]、[0, 255] などモデルにより異なる。誤った範囲は全入力をずらし精度破壊。

ドキュメントに要件があるべきだが常に完全ではない。転移学習経由を理解すると、元事前学習モデル要件とファインチューニング変更を追って正しい前処理を特定しやすい。

=== デバッグと性能期待

転移学習履歴理解が問題デバッグと現実的期待設定を助ける:

*自分データでの精度*: 1,000 例ファインチューニングは 100,000 例より堅牢性が低い。エッジケースで低精度ならこれが理由かも。

*ドメイン不一致*: 昼間屋外でファインチューニング、夜間屋内で使用——低性能は驚くべきではない。学習ドメイン理解が別モデル/追加ファインチューニング必要時期を特定。

*失敗モード*: 学習に基づき予測可能な失敗。正面顔中心ファインチューニングは横顔で失敗。晴天中心は霧で失敗。学習データ分布知識が制限理解・伝達を助ける。

*最適化の機会*: 特徴抽出ファインチューニング（最終層のみ変更）なら、早期層はベースモデルと同一。複数モデル間でそれらの層共有など最適化可能。

== 実務ワークフロー

本チュートリアルシリーズの典型的ワークフロー。各ステップ理解が情報に基づく判断と一般的落とし穴回避を助ける。

=== ステップ 1: 事前学習済みモデル選択

モデル zoo を閲覧し、ニーズに近いモデルを探す:

*探索するモデル zoo*:

- *TensorFlow Hub* (tfhub.dev): 視覚、テキスト、音声向け数千事前学習モデル
- *PyTorch Hub* (pytorch.org/hub): PyTorch エコシステムの人気モデル
- *Hugging Face* (huggingface.co/models): NLP に特に強いが vision モデルも
- *ONNX Model Zoo* (github.com/onnx/models): 既に ONNX 形式
- *Model Zoo* (modelzoo.co): 複数ソース集約

*選択基準*:

*タスク一致*: 類似タスクで学習されたモデル。物体検出は YOLO、SSD、Faster R-CNN。画像分類は ResNet、MobileNet、EfficientNet。セマンティックセグメンテーションは U-Net、DeepLab。

*性能要件*: サイズと速度確認。MobileNet は速いが ResNet より精度低。EfficientNet は良好な精度/速度トレードオフ。model card に標準ハードウェア推論時間。

*ハードウェア制約*: モバイルターゲットなら小モデル優先（MobileNet、SqueezeNet）。デスクトップ/サーバーは大モデル可（ResNet-152、EfficientNet-B7）。

*ライセンス*: ライセンス確認。多くの研究モデルは OSS（MIT、Apache 2.0）だが商用制限ありも。

=== ステップ 2: そのまま性能評価

ファインチューニング投資前に、事前学習モデルを自分データでテスト:

*迅速な評価*: ターゲットドメインから 50〜100 例で実行。既に 80%+ ならファインチューニング不要かも。

*失敗モードの特定*: 何を間違える？ 体系的（特定物体タイプで常に失敗）かランダム？ 体系的ならファインチューニングが助け。ランダムならより多くのデータまたは別アプローチ。

*前処理の確認*: 正しい前処理使用確認。多くの「モデルが動かない」は実際前処理ミスマッチ。

=== ステップ 3: 必要ならファインチューニング

そのまま性能不足ならファインチューニング検討:

*学習データ収集*: ターゲットドメイン例収集。特徴抽出ならクラスあたり 100〜500。ファインチューニングなら 1,000〜5,000 目標。

*戦略選択*: 特徴抽出から（最速、最少データ）。不足なら最後数層ファインチューニング。十分なデータがあるときだけさらに解凍。

*学習フレームワーク使用*: PyTorch/TensorFlow でファインチューニングは単純。多くの事前学習モデルにファインチューニング例コード。データに合わせ調整。

*慎重な検証*: 学習/検証/テスト分割。学習中検証精度監視。頭打ちで停止し過学習回避。

*前処理の文書化*: ファインチューニングで使った前処理を正確に記録。推論で再現必須。

=== ステップ 4: 推論形式へエクスポート

モデル（事前学習またはファインチューニング）を推論向け形式へ:

*形式選択*:

- *ONNX*: フレームワーク横断互換、デスクトップ/サーバー
- *TFLite*: モバイル、特に Android

*エクスポート*:

PyTorch → ONNX:

[source,python]
----
import torch
model.eval()
dummy_input = torch.randn(1, 3, 224, 224)
torch.onnx.export(model, dummy_input, "model.onnx")
----

TensorFlow → TFLite:

[source,python]
----
converter = tf.lite.TFLiteConverter.from_keras_model(model)
tflite_model = converter.convert()
with open('model.tflite', 'wb') as f:
    f.write(tflite_model)
----

*エクスポートの検証*: 元モデルとエクスポートモデルで同入力推論、出力一致（浮動小数点許容内）。エクスポートバグは一般的で静かにモデル破壊。

*必要なら最適化*: 融合、量子化、剪定など最適化パス。サイズ削減・推論速度改善。

=== ステップ 5: Vulkan で推論実装

本チュートリアルシリーズの大部分をここで:

*モデル読み込み*: ONNX/TFLite をパースしアーキテクチャと重み抽出。

*演算実装*: 畳み込み、行列乗算、活性化など各演算タイプの Vulkan コンピュートシェーダ。

*GPU メモリ割り当て*: テンソル用バッファ、スマートメモリ再利用でフットプリント最小化。

*推論実行*: 正しい順序と適切同期でコンピュートシェーダディスパッチ。

*結果検証*: Vulkan 推論出力と元フレームワーク出力比較。浮動小数点許容内一致。

*性能最適化*: プロファイル、メモリアクセスパターン、ワークグループサイズ、GPU 固有機能。

=== このワークフローの利点

関心のきれいな分離:

*学習の専門家* は PyTorch/TensorFlow で学習・ファインチューニングに集中、低レベル GPU プログラミング不要。

*グラフィックスプログラマ* は効率的 Vulkan 実装に集中、学習アルゴリズム深い専門知識不要。

*モデル形式* が橋——任意フレームワーク学習モデルを任意推論エンジンでデプロイ。

Vulkan を理解するグラフィックスプログラマーは学習アルゴリズム専門家になる必要なし。他人が学習したモデルを使い効率実行に集中。逆に ML 研究者はデプロイ詳細を気にせず学習——Vulkan 実装は他者が担当。

この分離が現代 ML デプロイを実用的にします。

== まとめ

転移学習はゼロから学習する代わりに事前学習済みモデルを活用し、時間・データ・計算資源を節約。鍵はニューラルネットワーク特徴がしばしば転移可能——早期層が多くのタスクに有用な汎用パターンを学ぶこと。

主戦略は特徴抽出（新最終層のみ学習）、ファインチューニング（後半数層をドメインに適応）、ドメイン適応（大きな分布シフト対応）。多くの実務では特徴抽出またはファインチューニングが単純さと性能の適切なバランス。

転移学習理解は適切モデル選択、前処理要件一致、現実的性能期待設定を助ける。典型的ワークフローは事前学習モデル選択、必要ならファインチューニング、推論形式エクスポート、Vulkan 効率 GPU 実行。

モデルがどう学習・適応されるかの理解ができたので、遭遇する各種モデル形式とその扱い方を探る準備ができました。
