= Vulkan による機械学習推論: はじめに

== 導入

「Vulkan による機械学習推論」チュートリアルシリーズへようこそ！本シリーズは、Vulkan の強力なコンピュート能力と現代の機械学習推論の交差点を探り、高性能 AI ワークロードのために GPU をどう活かすかを示します。

=== 新しいフロンティア

前のチュートリアルシリーズが Vulkan のグラフィックスとコンピュートの基礎を紹介したのに対し、本シリーズは機械学習推論の世界——Vulkan コンピュートシェーダを使って学習済みニューラルネットワークを GPU 上で効率的に実行する——へ連れていきます。ML フレームワークと低レベル GPU プログラミングの隙間をどう埋めるかを学び、高レベルフレームワークがしばしば提供できない制御と性能を得ます。

本シリーズは、xref:00_Introduction.adoc[Vulkan Tutorial] を完了し、コンピュートシェーダと GPU プログラミングの概念に慣れている読者を対象とします。実務実装に焦点を当てます: モデルの読み込み、効率的なデータ転送、コンピュートシェーダでのニューラルネットワーク操作の実装、リアルタイム性能向けの最適化。まだコンピュートパイプラインを扱ったことがなければ、続ける前にメインチュートリアルのコンピュートシェーダ章を再訪してください。

=== 自前で作るべきか？ 実務的視点

Vulkan で ML 推論を実装する深みに入る前に、最も重要な問いを扱いましょう: *実際にやるべきか？*

アプリケーションの圧倒的多数について、プロとしての推奨は明確です: *既存の ML 推論ライブラリを使ってください*。TensorFlow Lite、ONNX Runtime、PyTorch Mobile、およびプラットフォーム固有ソリューション（iOS の Core ML、Android の ML Kit、Windows の DirectML）のような成熟したフレームワークは次を提供します。

* *実戦検証された実装* - 何年もの最適化とバグ修正
* *広いモデルサポート* - すでに実装された何百もの演算型
* *ハードウェア加速* - GPU、NPU、専用 AI アクセラレータの自動利用
* *活発な保守* - 新しいハードウェアと技法向けの定期更新
* *本番信頼性* - 世界中の何百万ものアプリケーションで使用

これらのライブラリは何千人年のエンジニアリング労力を表します。使用を妨げる固有の制約がない限り、この成果を活用するのが賢い選択です。

==== カスタム Vulkan 実装が意味をなすとき

ではなぜ本チュートリアルは存在するのか？ Vulkan で自前の推論エンジンを構築することが実行可能なだけでなく——最良のアーキテクチャ選択である、実シナリオがあるからです。

*グラフィックス統合*: すでにレンダリングに Vulkan を使うゲームやグラフィックスアプリケーションを構築しているなら、既存の Vulkan パイプラインに ML 推論を統合する方が、別の ML フレームワークへの橋渡しより効率的になり得ます。異なる GPU コンテキスト間のデータ転送を避け、メモリと同期プリミティブを共有し、統一されたコードベースを保てます。

*レンダリング AI 技法*: Gaussian splatting、ニューラルレンダリング、その他のグラフィックス志向 AI 手法は、しばしば軽量な活性化関数と単純なネットワークアーキテクチャを使います。これらの「単純 AI」アプローチは、計算オーバーヘッドが管理可能で、レンダリングパイプラインとの密な統合が大きな性能利益を与えるため、カスタム Vulkan 実装に特に適しています。

*組み込みとリソース制約プラットフォーム*: 一部の組み込みシステムや特殊ハードウェアは標準 ML フレームワークをサポートしませんが、Vulkan はサポートします。カスタム実装は、リソース制約環境で重要なメモリ使用と性能特性への精密な制御を与えます。

*新規モデルアーキテクチャ*: 標準フレームワークがまだサポートしていない演算を使う最先端研究モデルを扱っているなら、Vulkan コンピュートシェーダで直接実装する方が、フレームワークサポートを待つか制限を回避するより速いかもしれません。

*クロスプラットフォーム一貫性*: Vulkan は Windows、Linux、Android、そしてますます他のプラットフォームでも動きます。多様なプラットフォームで同一の推論挙動が必要で、既存フレームワークが微妙な差を示すなら、カスタム Vulkan 実装がその一貫性を提供します。

*学習と理解*: 最終的に本番で既存ライブラリを使うとしても、Vulkan レベルで ML 推論を理解すると、より良いエンジニアになります。性能特性を理解し、問題をより効果的にデバッグし、より良いアーキテクチャ判断ができます。

==== ハイブリッドアプローチ

実務では多くのアプリケーションがハイブリッドアプローチを使います: ほとんどの推論ワークロードに標準 ML フレームワーク、特定の性能重要または密統合コンポーネントにカスタム Vulkan 実装。本チュートリアルは両方のシナリオに備えます——各アプローチをいつ使うかについて情報に基づく判断ができるほど深く基礎を理解します。

サードパーティ ML ライブラリを Vulkan アプリケーションに統合する章をまるごと割き、フレームワークと Vulkan レンダリングパイプラインの橋渡しを示します。これにより、アプリケーションの各部分に適切なツールを選べます。

=== 何を期待するか

「Vulkan による機械学習推論」シリーズは、理解と実務スキルの両方を与えるよう設計されています。扱う内容:

1. *ML 推論の基礎* - 機械学習推論とは何か、どう動くか、ニューラルネットワークをコンピュート操作としてどう考えるか。

2. *サードパーティ ML ライブラリの統合* - 確立された ML 推論フレームワーク（TensorFlow Lite、ONNX Runtime、PyTorch Mobile、DirectML）を Vulkan アプリケーション内で使う包括的ガイド。ほとんどの本番アプリケーションで推奨されるアプローチであり、ML フレームワークと Vulkan レンダリングパイプラインの橋渡しに十分な紙幅を割きます。

3. *モデル形式と読み込み* - TFLite や ONNX のような一般的な形式の扱い、GPU 実行向けにモデルを読み込み準備する方法の理解。

4. *ML 向け Vulkan コンピュート* - コンピュートシェーダでのニューラルネットワーク操作の実装、効率的なメモリ管理、同期の扱い。

5. *カスタム推論エンジンの構築* - サードパーティライブラリがニーズを満たさない特殊用途向けに、ゼロから自前の推論エンジンを構築する方法。

6. *性能最適化* - サードパーティライブラリでもカスタム実装でも、スループット最大化、レイテンシ最小化、GPU ハードウェアの最大限活用の技法。

7. *実務アプリケーション* - ML 推論をグラフィックスレンダリングと対話アプリケーションと組み合わせる実世界の例。サードパーティライブラリ統合とカスタム Vulkan 実装の両方を示します。

本シリーズ全体を通じて、異なるモデルを試し、様々な最適化戦略を試し、技法を固有の用途に適応することを奨励します。機械学習推論は急速に進化する分野であり、Vulkan レベルで理解すると、既存ライブラリの統合でもカスタムソリューションの構築でも、最先端技法を実装する柔軟性が得られます。

=== 三経路の教え方

本チュートリアルは独特なアプローチを取ります: 可能な限り、ML 推論の目標を達成する *3 つの* 方法を教えます。

*経路 1: サードパーティライブラリ* - ほとんどの本番アプリケーションで推奨されるアプローチ。成熟した ML フレームワーク（TensorFlow Lite、ONNX Runtime、PyTorch Mobile、DirectML）を Vulkan アプリケーションに統合し、ML 推論とレンダリングパイプラインの橋渡しを学びます。

*経路 2: カスタム Vulkan 実装* - Vulkan コンピュートシェーダを使って自前の推論エンジンを構築。この経路は、GPU レベルで ML 推論が実際にどう動くかへの深い洞察を与え、サードパーティライブラリの「内部」で何が起きているかを理解するのに役立ちます。

*経路 3: ML コンパイラ* - ML コンパイラ（IREE、Apache TVM、OpenXLA、MAX/Mojo）を使い、モデルを Vulkan および他のバックエンド向け最適化カーネルに lower します。統合の容易さと性能／移植性のバランスを取ります。

なぜ 3 つすべてを教えるのか？ 内部を理解すると、本番でライブラリとコンパイラをより上手く使えるからです。畳み込みが GPU 操作にどう写るか、メモリがどう管理されるか、どこで同期が重要かを知るとき、より良いアーキテクチャ判断ができ、問題をより効果的にデバッグし、より賢く最適化できます。内部の核心概念に詳しい人ほど、本番でこれらのツールを使う方法を理解できます。

チュートリアル全体で、三経路の練習が得られます。基礎と Vulkan コンピュートの章は操作を自分で実装することを教えます。サードパーティライブラリの章は本番向けソリューションを示します。ML コンパイラ節は、しばしば手書きに近い性能を最小ランタイムオーバーヘッドで提供するコンパイラ駆動経路を紹介します。

=== ML ツールの始め方

本チュートリアルシリーズについていくには、標準の Vulkan 開発環境に加え、いくつかの追加ツールが必要です。

*Python と ML フレームワーク*

ほとんどの ML モデルは Python ベースのフレームワークで学習されます。必要なもの:

- *Python 3.8 以降* - モデル変換スクリプトの実行と ML ライブラリの作業用
- *PyTorch または TensorFlow* - モデル形式の理解と変換の実行用に、少なくとも 1 つの主要 ML フレームワーク
- *ONNX Runtime*（任意） - ONNX モデルの作業と変換の検証に有用

*モデル変換ツール*

異なるモデル形式には異なるツールが必要です。

- *TensorFlow Lite Converter* - TFLite モデルの作業用
- *ONNX tools* - ONNX モデルの操作と最適化用

*Vulkan コンピュートのセットアップ*

Vulkan 開発環境がコンピュートをサポートすることを確認してください。

- GPU が Vulkan コンピュートキューをサポートすることを確認
- メインチュートリアルのコンピュートシェーダ例を実行できることを確認
- 性能分析用のプロファイリングツール（RenderDoc、Nsight、または類似）を用意

最初の章で詳細なセットアップ手順を提供しますが、これらのツールを用意しておくとスムーズについていけます。焦点は実務実装なので、最初からコードを書いて実行します。
