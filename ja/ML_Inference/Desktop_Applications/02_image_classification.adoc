= 画像分類: MNIST から CNN へ
:stem: latexmath

== 既知の手法の問題点

MNIST 向けに動作するニューラルネットワークを構築しました。全結合レイヤで手書き数字を 95% 以上の精度で認識します。同じアーキテクチャを実世界の画像分類問題に当てはめてみましょう。

ImageNet には 1000 の物体カテゴリがあります: 犬、猫、車、飛行機、家具、食べ物など。画像は 224×224 ピクセルのカラー（RGB）です。224 × 224 × 3 = 150,528 の入力値です。

MNIST と同様に全結合ネットワークを構築すると:

[source,cpp]
----
Input: 150,528 neurons (224×224×3 image)
Hidden Layer 1: 512 neurons
Hidden Layer 2: 512 neurons
Output: 1000 neurons (classes)
----

重みはいくつ必要でしょうか？

- Input → Hidden1: 150,528 × 512 = **7700 万重み**
- Hidden1 → Hidden2: 512 × 512 = 262,000 重み
- Hidden2 → Output: 512 × 1,000 = 512,000 重み

合計: **約 7700 万パラメータ**、ほとんどが最初のレイヤに集中します。

float 1 つ 4 バイトなら、最初のレイヤの重みだけで 308 MB です。単一画像の順伝播では、そのレイヤだけで 7700 万回の乗算加算です。それでも*小さな*ネットワークです——ResNet-50 は合計 2500 万パラメータで、はるかに効果的に画像を処理します。

しかしパラメータ数が最悪の問題ではありません。本当の問題は、全結合レイヤが画像の構造を無視することです。

== 画像には構造がある

写真を見ると、近傍のピクセルは関連しています。猫の耳を形成するピクセルは互いに隣接しています。200 ピクセル離れたピクセルはおそらく無関係——一方は空、他方は草の一部かもしれません。

全結合レイヤでは、すべての入力がすべての出力に等しく接続されます。ピクセル (0,0) から出力ニューロン 5 への重みは、ピクセル (100,150) から同じニューロンへの重みと同じくらい重要です。ネットワークは空間的近接性が重要だとゼロから学ばねばなりません。

MNIST が教えてくれたことを考えてください: 第 1 レイヤでエッジとコーナー、後続レイヤで曲線とループ、最終で完全な数字。階層構造は、たとえ 28×28 の小さな画像でも空間関係が重要だから機能しました。上の水平線分と右の垂直線分は数字「7」や「1」を示唆します。ネットワークはこれを学びましたが、数百万の重みを調整して自力で理解しなければなりませんでした。

224×224 画像では、そのアプローチは破綻します:

1. **パラメータが多すぎる**: メモリに収まらず、効率的に学習できず、深刻な過学習
2. **平行移動不変性がない**: ネットワークが画像中央で「猫」を学んでも、隅の「猫」には全く異なる重み。同じパターンを異なる位置で何千回も学ぶ必要がある
3. **空間的局所性を無視**: 猫の耳は近傍ピクセルで構成されるが、ネットワークは (50,50→51,50) の関係と (100,100→101,100) の関係を完全に別物として扱う

画像の仕組みを尊重するアーキテクチャが必要です。

== 重要な洞察: 局所パターンは繰り返される

どんな写真でも、同じ種類のパターンが画像全体に現れます:

- **エッジ**: 暗から明への遷移がどこにでも——物体境界、影、テクスチャ変化
- **コーナー**: 2 本のエッジが交わる重要な形状特徴
- **テクスチャ**: 草、レンガ、毛皮、布地のような繰り返しパターン
- **色**: グラデーション、単色領域、遷移

これらのパターンは位置に依存しません。垂直エッジは左端でも右端でも同じように機能します。「毛皮テクスチャ」の検出は、毛皮が現れる位置に関係なく同じロジックを使います。

画像の各位置ごとに 15 万個別の重みを学ぶ代わりに、1 つのエッジ検出器を学び、*どこにでも適用*したらどうでしょうか？ 一度学び、どこでも使う。

それが畳み込みです。

== 畳み込み: パターン検出のスライディングウィンドウ

3×3 の重みグリッド——画像の 3×3 パッチを見る小さなニューラルネットワーク断片——があると想像してください。垂直エッジ検出器の具体例です。

フィルタの重み配置は次のとおり: 左列はすべて -1、中央列は 0、右列は +1。左側が暗く（ピクセル値約 50）、右側が明るい（約 200）パッチに適用すると:

左側寄与: 50 × (-1) + 50 × (-1) + 50 × (-1) = -150
中央寄与: 50 × 0 + 50 × 0 + 50 × 0 = 0
右側寄与: 200 × (+1) + 200 × (+1) + 200 × (+1) = +600

合計応答: -150 + 0 + 600 = +450

この大きな正の値はネットワークに「ここに垂直エッジを見つけた！」と伝えます。エッジが逆（明から暗）なら -450。ピクセルが均一（エッジなし）ならほぼゼロ。

=== 畳み込みの数学

数学的に、単一出力ピクセル latexmath:[y_{i,j}] の 2D 畳み込みは、フィルタ latexmath:[w] と局所入力パッチ latexmath:[x] の要素ごとの積の和です:

[latexmath]
++++
y_{i,j} = \sum_{m=0}^{K-1} \sum_{n=0}^{K-1} w_{m,n} \cdot x_{i+m, j+n} + b
++++

ここで:
*   latexmath:[K] はカーネルサイズ（例: 3）。
*   latexmath:[w_{m,n}] はフィルタ内位置 latexmath:[(m, n)] の重み。
*   latexmath:[x_{i+m, j+n}] は入力ピクセル値。
*   latexmath:[b] はバイアス項（結果に加える単一の数）。

Vulkan コンピュートシェーダでは、まさにこのループを書きます。画像から 9 値を読み、バッファに格納された 9 重みを掛け、和を出力に書き込みます。

.畳み込み演算 - 画像上をスライドするフィルタ
image::images/ML_Inference/Desktop_Applications/2d_convolution_animation.gif[Convolution animation showing filter sliding across image,600]

同じ 3×3 フィルタを画像のすべての位置にスライドします。位置 (0,0) で 1 出力値を計算。1 ピクセル右の (0,1) へ。224×224 入力全体をカバーするまで横・縦にスライドし、各位置のエッジ強度を表す 224×224 出力を得ます。

=== 出力の制御: パディングとストライド

ウィンドウをスライドするとき、出力フィーチャマップの形状を決める 2 つの重要なつまみがあります。

==== 1. パディング
5×5 画像に 3×3 フィルタを適用すると、横 3 位置・縦 3 位置にしかフィルタを置けません。出力は 3×3 になります。画像が 2 ピクセル「縮小」されます。

深いネットワークでこれを繰り返すと、224×224 画像は数レイヤで 1 ピクセルに消えます！ これを防ぐために **パディング** を使います。入力の端の周りに「偽」のピクセル（通常ゼロ）を追加します。

*   **Valid パディング**: パディングなし。画像は縮小。
*   **Same パディング**: 出力が入力と同じサイズになるだけゼロを追加。3×3 フィルタなら各辺に 1 ピクセル。

==== 2. ストライド
**ストライド**はフィルタが各ステップで「飛ぶ」ピクセル数です。
*   **ストライド = 1**: 1 ピクセルずつ移動。高詳細、高計算量。
*   **ストライド = 2**: 1 ピクセルおき。実質的に出力解像度を半分に（224×224 → 112×112）。

現代アーキテクチャでは、**ストライド畳み込み**（ストライド=2）をプーリングの代わりに使い画像サイズを減らすことが多いです。ネットワークが「最大を取る」のような固定ルールではなく、*学習*してダウンサンプルできるからです。

=== 複数チャンネル: 3D 畳み込み

実画像は 2D グリッドだけではなく、深さ（赤、緑、青）があります。CNN 第 1 レイヤの畳み込みフィルタは実際 **3D ボリューム** です。

入力が 224×224×3 なら、3×3 フィルタは実際 **3×3×3** です。3 色チャンネルすべてにわたる 3×3 パッチを同時に見ます。例えば「主に赤い垂直エッジ」を見つける必要がある特定特徴を学ぶかもしれません。

64 個のフィルタがあると、各々が 1 つの 2D フィーチャマップを出力します。64 枚を積み重ね 224×224×64 出力を作ります。次レイヤのフィルタは **3×3×64**——前レイヤで発見されたすべての特徴を見ます。

[latexmath]
++++
\text{Output Channels} = \text{Number of Filters}
++++
[latexmath]
++++
\text{Filter Depth} = \text{Input Channels}
++++

これが CNN が強力な理由です: 2D パターンだけでなく、複数の情報「レイヤ」間の相関を探します。

== 畳み込みレイヤの構築

畳み込みレイヤには複数のフィルタ（初期レイヤでは通常 32–128、後続では 256–512）があります。各フィルタは異なるパターンを検出するよう学習します:

- フィルタ 1: 水平エッジ
- フィルタ 2: 垂直エッジ
- フィルタ 3: 対角エッジ（左上から右下）
- フィルタ 4: 対角エッジ（右上から左下）
- フィルタ 5: コーナー
- フィルタ 6–32: 学習中に学ぶその他のパターン

32 フィルタすべてを 224×224 RGB 入力に適用すると、32 枚の出力画像（「フィーチャマップ」）が得られ、各 224×224 で、特定パターンが現れる位置を強調します。

.典型的な CNN アーキテクチャ
image::images/ML_Inference/Desktop_Applications/cnn_layers.png[CNN layer architecture showing progression,800]

入力: 224×224×3（高さ × 幅 × RGB チャンネル）
↓
32 種類の 3×3×3 フィルタを適用（空間 3×3、3 色チャンネルすべて）
↓
出力: 224×224×32（高さ × 幅 × 32 フィーチャマップ）

32 フィルタ各々に 3×3×3 = 27 重み（+ 1 バイアス）= フィルタあたり 28 パラメータ。
合計: 32 フィルタ × 28 = レイヤ全体で **896 パラメータ**。

同じ仕事をする全結合レイヤの 7700 万パラメータと比較してください。これが畳み込みが機能する理由です。

== なぜこれが問題を解決するか

**平行移動不変性**: 同じフィルタが猫の耳を (50,50) でも (150,100) でも検出。パターンを一度学べばよい。

**パラメータ効率**: 各位置ごとに数百万の別々の重みの代わりに、数百の共有重みをどこにでも適用。

**空間的局所性**: 3×3 フィルタは近傍ピクセルが関連していることを自然に尊重——局所領域だけを見る。

**階層的特徴**: 複数の畳み込みレイヤを積むと階層が得られる。レイヤ 1 はエッジ、レイヤ 2 はエッジを形状に、レイヤ 3 は形状を物体部分に、レイヤ 4 は部分を物体に。

== エッジから物体へ: 階層

犬の写真を畳み込みネットワークに通したときの流れを追いましょう:

**レイヤ 1**（入力: 224×224×3、出力: 224×224×64）:

エッジ検出器、色ブロブ、単純テクスチャを学習。1 フィルタは垂直エッジ、別は水平エッジ、別は赤から茶へのグラデーションに応答。出力はエッジマップのように——抽象的で、まだ物体として認識できない。

.レイヤ 1 フィーチャマップ

**レイヤ 2**（入力: 224×224×64、出力: 112×112×128）:

エッジを形状に合成。1 フィルタは「水平・垂直エッジで形成されたコーナー」に応答。別は「円形境界」に応答。まだ低レベルだが物体の一部のように見え始める。空間解像度も 224→112 に削減——「正確な位置」より「何か」が重要。

.レイヤ 2 フィーチャマップ

**レイヤ 3**（入力: 112×112×128、出力: 56×56×256）:

形状を部分に合成。1 フィルタは「曲線エッジで囲まれた毛皮テクスチャ」（犬、猫、テディベアの一部の可能性）。別は「高コントラストの丸い形状」（目の候補）。別は「平行線の矩形」（窓、ドア、本の背）。

**レイヤ 4**（入力: 56×56×256、出力: 28×28×512）:

部分を物体固有特徴に合成。このレベルでは「犬の顔」「車輪」「鳥の翼」に特化して活性化するフィルタがある。空間解像度は粗い（28×28）——分類には精密な位置決めは不要——「この画像に犬が含まれる」だけ分かればよく、「犬の鼻がピクセル (145, 87) にある」必要はない。

.レイヤ 4 フィーチャマップ（高レベル概念）

**最終レイヤ**（グローバル平均プーリング + 全結合）:

28×28×512 フィーチャマップを 1000 クラススコアに縮約。「犬特徴の強い証拠、猫は弱い、飛行機の証拠なし → 予測: 犬」。

.完全な CNN アーキテクチャ

この階層は手設計ではありません。「レイヤ 1 はエッジ、レイヤ 2 は形状」とネットワークに指示しません。すべてのフィルタをランダム重みで初期化し、MNIST と同様に逆伝播で学習します。畳み込み構造が与えられた問題を解く最適な方法だから、階層が*自然に現れる*のです。

== 非線形性: ReLU が王者である理由

畳み込みは線形演算（乗算と加算）です。10 個の線形レイヤを積むと、結果は依然 1 つの大きな線形演算です。数学的にネットワーク全体を 1 レイヤに畳み込めます！

「犬の耳」のような複雑な非線形パターンを学ぶには、レイヤ間に非線形性を注入する必要があります。**活性化関数**を使います。

=== ReLU（Rectified Linear Unit）

現代 CNN ではほぼ **ReLU** のみを使います:

[latexmath]
++++
f(x) = \max(0, x)
++++

*   **なぜ？** 計算コストがほぼ「無料」——シェーダでは `max(0.0, value)` だけ。
*   **勾配の流れ**: Sigmoid（0–1 に「潰す」ため勾配が消失しやすい）と違い、ReLU は正の値で飽和しない。深いネットワークの学習がはるかに容易。

=== その他の活性化
*   **Leaky ReLU**: `max(0.01x, x)`。「Dead ReLU」（ニューロンがゼロで固定され再学習しない）を防ぐ。
*   **GELU（Gaussian Error Linear Unit）**: Transformer や現代 Vision モデルで使用。ReLU の「滑らかな」版。

== バッチ正規化: 安定化装置

データが深いネットワークを流れると値が drift する——レイヤによっては非常に大きな数、他は極小。**内部共変量シフト**と呼ばれ、学習を非常に不安定にします。

**バッチ正規化**は各畳み込みレイヤの後にデータを「再中心化」してこれを解決します。平均が 0 付近、標準偏差が 1 付近になるよう保証します。

*   **学習時**: 現在のバッチの mean/std を計算。
*   **推論時（Vulkan）**: これらの値は事前計算され畳み込み重みに「畳み込まれる」。**推論時バッチ正規化は無料！** ONNX Runtime でモデルを使うと、バッチ正規化レイヤは見えません——性能のため畳み込みに数学的にマージされています。

== 受容野: 深いパターンがどれだけ見えるか

重要な概念: 受容野。レイヤ 4 のニューロンは元画像のどれだけを「見る」か？

レイヤ 1 では 3×3 フィルタは入力のちょうど 3×3 ピクセルを見ます。

レイヤ 2 では 3×3 フィルタはレイヤ 1 出力の 3×3 位置を見ます。各位置は入力の 3×3 ピクセルから来たので、レイヤ 2 は実質的に:

----
Layer 2 receptive field = 3 + (3-1) = 5×5 input pixels
----

レイヤ 3 では各位置はレイヤ 2 の 3×3 を見、各レイヤ 2 位置は入力の 5×5 を見ます:

----
Layer 3 receptive field = 5 + (3-1) = 7×7 input pixels
----

レイヤ 5 まで行くと、単一ニューロンは正確なアーキテクチャ次第で 13×13 以上の入力ピクセルを見るかもしれません。

.受容野の成長

深いネットワークが単純パターンから複雑パターンを構築する方法です。初期レイヤは小領域を見て単純パターンを検出。後続レイヤは大領域（初期レイヤの出力を見るため）を見て複雑パターンを検出。

プーリングはこれを加速します。224×224 を 112×112 に減らすと、小さいマップの各位置は大きいマップの 2×2 領域に対応。受容野の成長率が倍になります。

== なぜ大きいフィルタだけを使わないのか

階層を飛ばして、例えばレイヤ 1 で 11×11 フィルタを使いより多くの文脈を見ればよいのでは、と思うかもしれません。

パラメータです。11×11 フィルタはチャンネルあたり 121 重み。RGB 入力を処理する 64 フィルタのレイヤでは:

----
11×11×3 channels × 64 filters = 23,232 parameters
----

同じ受容野を持つ 2 つの 3×3 レイヤを積んだ場合（同じ受容野）:

----
Layer 1: 3×3×3 × 64 = 1,728 parameters
Layer 2: 3×3×64 × 64 = 36,864 parameters
Total: 38,592 parameters
----

待って、*より多い*パラメータ！ しかし決定的な違いがあります: **間に ReLU 活性化がある 2 レイヤ**。

ReLU を挟んだ 2 つの 3×3 畳み込みレイヤを積むと、1 つの 5×5 レイヤと同じ受容野（入力で 5×5）だが、より多くの非線形性。余分な ReLU がより複雑な関数を学ばせます。3 つの 3×3 レイヤは 1 つの 7×7 と同じ受容野をカバーするが、非線形性は 1 つではなく 3 つ。

現代ネットワーク（VGG、ResNet、MobileNet など）はほぼ exclusively 3×3 と 1×1 畳み込みを使う理由です。大きいフィルタは表現力を増やさずパラメータを無駄にします。

== プーリングと空間削減

224→112→56→28 と削減した理由は？

**計算効率**: 224×224 画像は 50,176 位置。112×112 は 12,544——4 倍少ない。プーリングは計算を劇的に削減。

**平行移動不変性**: 入力を 1 ピクセルシフトしても、max プーリング（各 2×2 領域の最大）が同じ出力を出すかもしれない。正確な位置の重要度が下がる。

**抽象化**: 分類では犬の目が (45,67) か (47,68) かは気にしない。画像の上中部どこかに目があることが重要。空間解像度を下げると位置不変特徴を学ばせる。

Max プーリングは小領域（通常 2×2）の最大値を取り出力。ストライド 2 の 2×2 max プーリングは 4×4 入力を 2×2 出力に——重ならない各 2×2 ブロックの max を取る。

.Max プーリング演算
image::images/ML_Inference/Desktop_Applications/max_pooling.png[Max pooling with 2x2 filter,500]

現代アーキテクチャではストライド畳み込み（ストライド=2 で 1 位置おき）を使うことも——max プーリングのようなハードコードではなく学習可能。原理は同じ: 空間次元を段階的に減らしチャンネル深さを増やす。

== パターン: Conv → ReLU → Pool → Conv → ReLU → Pool...

ほぼすべての CNN がこのパターンに従います:

----
Input Image (224×224×3)
  ↓
[Conv 3×3 → ReLU → Pool] → (112×112×64)
[Conv 3×3 → ReLU → Pool] → (56×56×128)
[Conv 3×3 → ReLU → Pool] → (28×28×256)
[Conv 3×3 → ReLU → Pool] → (14×14×512)
  ↓
Global Average Pool → (1×1×512)
  ↓
Fully Connected → (1000 class scores)
  ↓
Softmax → (probabilities)
----

空間次元は縮小: 224 → 112 → 56 → 28 → 14 → 1

チャンネル深さは成長: 3 → 64 → 128 → 256 → 512

空間詳細を意味的深度とトレードします。初期レイヤは「エッジはどこ？」後続は「犬がいるか？」

== MobileNetV2: CNN を効率化

標準畳み込みは高コスト。512 入力・512 出力チャンネルの 3×3 畳み込みは:

----
3×3 spatial × 512 input × 512 output = 2,359,296 operations per position
For 28×28 image: 2.36M × 784 = 1.85 billion operations
----

MobileNetV2 は **depthwise separable convolution** という手法を使います。1 つの高コスト演算の代わりに 2 つの安い演算に分割:

**Depthwise 畳み込み**: 3×3 フィルタを*各チャンネルに個別*適用。

----
Input: 28×28×512 (512 separate channels)
Apply 512 different 3×3 filters (one per channel)
Output: 28×28×512
Cost: 3×3 × 512 = 4,608 operations per position
----

各チャンネル内の空間パターンは捉えるが、チャンネル間の情報は混ぜない。

**Pointwise 畳み込み**: 1×1 畳み込みでチャンネルを混合。

----
Input: 28×28×512
Apply 512 different 1×1×512 filters
Output: 28×28×512
Cost: 1×1 × 512 × 512 = 262,144 operations per position
----

チャンネル間の情報は混合するが空間近傍は見ない。

合計コスト: 4,608 + 262,144 = 位置あたり **266,752 演算**

標準畳み込み: 位置あたり 2,359,296 演算と比較。

**約 8.8 倍高速化**、精度低下は最小。

=== MobileNetV2 ブロック: Inverted Residuals

MobileNetV2 は **Inverted Residual Block** でさらに効率化。

ResNet のような従来 Residual ブロックは「Wide → Narrow → Wide」。多チャンネルを取り圧縮して再拡張。

MobileNetV2 は逆: **Narrow → Wide → Narrow**。

1.  **Expansion（1×1 Conv）**: 少ない入力チャンネルを拡張（通常 6 倍）。depthwise 畳み込みにパターン探索の「余地」を与える。
2.  **Depthwise（3×3 Conv）**: 高次元データで空間フィルタリング。
3.  **Projection（1×1 Conv）**: 再び少ないチャンネルに圧縮。

=== Linear Bottlenecks

微妙だが重要な詳細: ブロック内の最終 1×1 Projection レイヤは **ReLU を使わない**。

なぜ？ 低次元空間で ReLU を使うと情報を失いすぎる（データの半分をゼロ化）。ボトルネックで線形活性化（活性化なし）を使うと、発見した複雑特徴を保持できる。

.Inverted Residual Block 構造
[latexmath]
++++
\text{Input} \xrightarrow{1\times1, ReLU} \text{Expanded} \xrightarrow{3\times3, ReLU} \text{Filtered} \xrightarrow{1\times1, Linear} \text{Output}
++++

depthwise separable 畳み込み、inverted residual、linear bottleneck の組み合わせが、MobileNetV2 をスマートフォンで 60 FPS、数千物体を正しく認識させます。

== 本チュートリアルで MobileNetV2 が重要な理由

Vulkan で推論を実装するとき、どの演算が実行時間を支配するか知る必要があります。MobileNetV2 では:

**Pointwise 畳み込み**（チャンネル混合の 1×1 畳み込み）は行列積。計算 bound。最適化は演算スループット最大化——テンソルコア、ベクトル化命令、大きなスレッドブロック。

**Depthwise 畳み込み**（チャンネルごとの 3×3 フィルタ）はメモリ bound。各出力値に 9 入力値と 9 重みの読み込み。最適化はメモリアクセスパターン改善——読み込みの coalescing、共有メモリ、キャッシュミス最小化。

アーキテクチャを理解すると最適化の焦点が分かります。ReLU（`max(0, x)` だけ）に数日かけても、ReLU は実行時間の 1% 未満なので効果は小さい。pointwise 畳み込みを最適化すれば推論は 2–3 倍速くなり得ます。

実装の前に理論を教える理由です。

== ネットワークの学習: 逆伝播の直感

フィルタが*何をするか*は話しましたが、値はどう得るか？ MNIST では数千重み。MobileNetV2 では数百万。手で調整できません。

**逆伝播**と**確率的勾配降下法（SGD）**を使います。

=== 損失曲面
ネットワークの誤差（Loss）を山脈と想像。任意の点の「高さ」は間違えた画像数。「座標」（経度・緯度）は数百万の重みの値。

開始時重みはランダム。ランダムな山頂に立っている。目標は最も低い谷——誤差ゼロの点——を見つけること。

=== 勾配
画像（例: 犬）を入力し、順伝播して結果を見る。ネットワークが「車」と予測すれば Loss は高い。

次に**勾配**を計算。勾配は数学的ベクトルで「重み latexmath:[w_1] をわずかに変えると Loss は上がる。重み latexmath:[w_2] を変えると Loss は下がる」と教えます。

[latexmath]
++++
w_{\text{new}} = w_{\text{old}} - \eta \cdot \frac{\partial \text{Loss}}{\partial w}
++++

latexmath:[\eta]（eta）は**学習率**——山を下るステップの大きさ。

=== 大規模学習
CNN ではレイヤ 10 のフィルタの勾配はその後のすべてのレイヤに依存。逆伝播は出力から入力へ「逆走」し、各フィルタが誤答にどれだけ寄与したか計算。

学習が推論よりはるかに遅い理由です。推論は 1 回の「順伝播」（山を下る）だけ。学習は数千回の順・逆伝播（傾き計算とステップ）。

== 成功の準備: データ拡張

最高のアーキテクチャでもデータが貧弱なら失敗します。犬の写真を草の上だけ見せると「緑背景 = 犬」を学ぶ。歩道の犬では「灰色の車」と予測するかもしれません。

これを防ぐために**データ拡張**を使います。学習中同じ画像を 2 度見せません。先に「いじる」:

*   **ランダム反転**: 犬を左右反転。依然犬。
*   **ランダムクロップ**: 尻尾や耳にズーム。部分も全体と同じくらい重要と学ぶ。
*   **色ジッター**: 明るさ・コントラスト・彩度を変更。日差しの犬も日陰の犬も同じ。
*   **回転**: 数度回転。

学習終了時、犬の*バリエーション*を数百万見ており、実世界のカメラノイズと照明条件に非常に頑健。

== 最終判断: Softmax と Cross-Entropy

数百万回の乗算の最後、ネットワークは 1000 数値（ImageNet クラス各 1 つ）を出力。**Logits** と呼ばれ、例:

*   Dog: 14.2
*   Cat: 8.5
*   Car: -2.1

人間にはあまり有用ではありません。**確率**で合計 100%（1.0）が欲しい。

=== Softmax 関数

Logits を確率にするために **Softmax**:

[latexmath]
++++
\text{Prob}_i = \frac{e^{L_i}}{\sum_{j} e^{L_j}}
++++

latexmath:[L_i] はクラス latexmath:[i] の logit。

*   **latexmath:[e] の力**: 指数関数により最高 logit が確率の「ライオンシェア」を得る。「勝者を報酬し敗者を罰する」。
*   **正規化**: 分母ですべての確率がちょうど 1.0 に。

=== Cross-Entropy Loss

学習中、これらの確率を「Ground Truth」（画像*が*犬なら Dog は 1.0、他は 0.0）と比較。**Cross-Entropy Loss**:

[latexmath]
++++
\text{Loss} = -\sum \text{Truth}_i \cdot \log(\text{Prob}_i)
++++

ネットワークが正解に 100% 確信なら loss は 0。*誤った*答えに確信なら loss はほぼ無限大。勾配降下が従う非常に「急」な山を作り、速い学習につながる。

== 学んだこと

次を理解しました:

- **全結合ネットワークが画像で失敗する理由**: パラメータ過多、平行移動不変性なし、空間構造無視
- **畳み込みとは**: どこにでも適用される共有重み、位置に関係なくパターン検出
- **CNN が階層を構築する方法**: 積み重ねレイヤでエッジ → 形状 → 部分 → 物体
- **3×3 フィルタが支配する理由**: 効率的パラメータ、大きい受容野のため積み重ね可能、非線形性追加
- **プーリングの役割**: 空間次元削減、平行移動不変性増加、計算削減
- **MobileNetV2 の仕組み**: depthwise separable 畳み込みが精度と効率をトレード
- **アーキテクチャが実装に重要な理由**: 支配演算を知ると最適化対象が分かる
