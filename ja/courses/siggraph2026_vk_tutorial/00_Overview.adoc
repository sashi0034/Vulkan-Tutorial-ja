= SIGGRAPH 2026: 2026 年における Vulkan アプリケーションの書き方

== 概要

ようこそ！本シリーズでは、新しい Vulkan 拡張と 2026 年時点の Khronos 推奨ベストプラクティスを使って、Vulkan レンダラーの書き方を説明します。本チュートリアルはシェーダーオブジェクト、ダイナミックレンダリング、ディスクリプタヒープ技法を使い、SIGGRAPH 2026 コース「How to write a Vulkan application in 2026」に基づいています。

スライド: 近日公開。
ソースコード: 近日公開。

本コースの終わりまでに、次のことを学びます。

- 最新の拡張を使って 2026 年の Vulkan アプリケーションを設計・実装する。
- レガシーな Vulkan 1.0 パターンを特定し、より単純な代替に置き換えて既存の Vulkan アプリケーションをモダナイズする。
- アプリケーション設計の単純化と開発者生産性の向上に適した Vulkan 拡張を評価する。
- レガシーな Vulkan 手法と推奨手法を区別し、避けるべき技法とその理由を理解する。
- ディスクリプタヒープ、ダイナミックレンダリング、関連拡張などの最近の Vulkan 機能を使う。

*前提条件*:
本コースは主に、Vulkan 経験がほとんどない、またはまったくない開発者を対象としています。想定する聴衆は、コンピュータグラフィックスの背景があり、OpenGL や DirectX などの API 経験がある可能性のあるプログラマで、Vulkan 入門を求めている方です。

C++ への慣れが期待されます。グラフィックスプログラミングの概念（例: シェーダー、レンダリングパイプライン）へのある程度の慣れがあると役立ちますが必須ではなく、本コースではこれらのトピックを Vulkan の文脈で扱います。
