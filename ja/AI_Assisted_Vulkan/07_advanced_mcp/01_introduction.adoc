= 高度な MCP ツーリングと自動化

== はじめに: 汎用支援の先へ

ここまで **Model Context Protocol（MCP）** を、ドキュメントとソースコードを読む受動的ブリッジとして使ってきました。しかし MCP の真の力は **カスタムツーリング**——AI に Vulkan エコシステムと直接やり取りする「腕と脚」を与える能力——にあります。

**協調エンジニアリング** では、AI を「チャットボット」から **能動的メンテナ** へ変えます。本セクションでは、基本的なファイル読み取りを超え、Vulkan レジストリの照会、ハードウェアリミットの監査、エンジン性能の自律管理を可能にする特化 MCP サーバの構築へ進みます。

== コア概念: AI の「プラグイン」

MCP サーバを、AI アシスタントの推論エンジン向け「プラグイン」と考えてください。デフォルトでは、AI はデータベース照会やパフォーマンスプロファイラ実行の仕方を知りません。MCP サーバを与えることは、**ツールボックス** を与えることです。

1.  **Registry ツール:** ライブ `vk.xml` ファイルを照会できる。
2.  **GPUInfo ツール:** `gpuinfo.org` のようなデータベースからハードウェアリミット（例: `maxDescriptorSetSampledImages`）を照会できる。
3.  **Profiler ツール:** エンジンのタイミングログを読み、ボトルネックを特定できる。

自律エージェント（**Goose** など）にこれらのツールを与えることで、以前は仕様とコードの間で何時間もの手動照合が必要だったタスク——**エージェンティック自動化**——が可能になります。

== なぜ Vulkan に高度な MCP か？

Vulkan の開発サイクルは **データルックアップ** に支配されています。

*   **仕様:** XML で定義される。
*   **ハードウェア:** JSON リミットで定義される。
*   **性能:** マイクロ秒単位のタイミングログで定義される。

AI モデルは、構造化データへ直接アクセスできるとき、それについての推論に非常に効率的です。MCP なしでは、あなたが仲介者となりチャット窓へ手動でデータをコピー＆ペーストします。高度な MCP があれば、目標を伝え（「Adreno 750 向けにシャドウマップ解像度を最適化」）、AI が **自分でデータを取りに行きます。**
