= デスクトップ、モバイル、組み込みデプロイ

== はじめに: マルチプラットフォームチェーン

Vulkan の最大の強みは移植性です。よく書かれたアプリはハイエンド Windows デスクトップ、Samsung スマートフォン、Raspberry Pi で動作できます。しかし各プラットフォームには、メモリ管理、アセット読み込み、性能最適化に関する独自の「法則」があります。

**協調エンジニアリング** では、アシスタントを **プラットフォーム認識スペシャリスト** として使います。AI でプラットフォーム間の「コンテキストギャップ」を埋め、モバイル向けの面倒な JNI ボイラープレートを自動化し、タイル GPU 向け最適化を提案し、組み込みシステムの厳しいリソース制約を管理します。

== コア概念: プラットフォームコンテキスト

「プラットフォーム認識」アシスタントとは、ターゲットの具体的ハードウェア制約が与えられた AI です。**コンテキストブリッジ** により、「ターゲットの現実」を AI に伝えます。

1.  **モバイルコンテキスト:** AI は **Tile-Based Deferred Rendering（TBDR）** を理解します。オンチップタイルメモリ性能を最大化するため、サブパス依存とメモリレスアタッチメントを優先します。
2.  **デスクトップコンテキスト:** AI は異なるベンダー拡張の性能トレードオフを理解します。`VK_NV_ray_tracing` とクロスベンダーの `VK_KHR_ray_tracing_pipeline` の使い分けを知っています。
3.  **組み込みコンテキスト:** AI は厳しい VRAM と CPU 予算を理解します。伝統的なページングを欠くアーキテクチャに収めるため、シェーダとテクスチャの量子化戦略を提案します。

== なぜデプロイに AI を使うか？

デプロイでは「構文ノイズ」が移植性のブロッカーになります。
*   **Android/iOS:** ウィンドウサーフェスとアセットローダを動かすだけでも、何百行もの「グルーコード」（JNI や Objective-C{pp}）を書かなければなりません。
*   **AI ソリューション:** エンジンのアセット構造を記述すると、AI が完全な多言語ブリッジを生成し、C{pp} Vulkan コアをクリーンでプラットフォーム非依存に保ちます。
