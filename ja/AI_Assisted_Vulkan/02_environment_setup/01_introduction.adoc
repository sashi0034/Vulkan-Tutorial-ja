= 現代の AI ツールベルト: 環境セットアップ

== はじめに: 「ラボ」の進化

伝統的なグラフィックスプログラミングでは、環境セットアップは静的なプロセスでした。コンパイラ（GCC/Clang/MSVC）を入れ、デバッガを設定し、IDE を Vulkan SDK に向ける。これらのツールとの関係は厳密に一方向——あなたが命令を与え、ツールが実行する——でした。

**協調エンジニアリング** を真に受け入れるには、「開発ラボ」を生きたエコシステムとして再考しなければなりません。IDE に「チャットプラグイン」を足すだけでは足りません。**コンテキストブリッジ** を構築するのです。このブリッジにより、高推論 AI モデルがあなたが見ているものを見て、アーキテクチャ上の意図を理解し、特定 GPU のハードウェアリミットまで照会できるようになります。

本章はこの変革の基盤を築き、「伝統的セットアップ」から **AI 強化ハブ** へと移行します。

== コア概念: 「コンテキストギャップ」

AI を使う開発者が最もよく感じるフラストレーションは **コンテキストギャップ** です。「新しいイメージバリアを追加して」と頼むと、エンジンが要求する `VkDependencyInfo`（Vulkan 1.3）パターンではなく `VkImageMemoryBarrier`（Vulkan 1.0）を使った汎用スニペットが返ってきます。エンジンのどこに置くべきかも考慮せず、複数ファイル・複数バージョンの Vulkan にまたがる変更を追い回す羽目になり、自分で書くより倍の時間がかかる——実質的に AI の仕事を自分がやっている、という状態です。

AI 統合セットアップは、3 つの明確なコンテキスト層を提供することでこの摩擦を解消することを目指します。第一に、**構文コンテキスト** は、コードベース構造と抽象構文木（AST）に対する IDE の深い理解を活用します。AI があなたの `Texture` クラスが `vk::raii::Image` をラップしていると知れば、互換性のない生ポインタ操作は提案しません。第二に、**意味コンテキスト** は **Model Context Protocol（MCP）** 経由で AI を公式 Vulkan 仕様と `vk.xml` レジストリに直接つなぎ、ビットマスクの意味を正確に「調べ」られるようにします。最後に、**ランタイムコンテキスト** は、あなたの特定ハードウェアの現実を AI に与えます。現代のエージェントは `vulkaninfo` やエンジンのログを照会し、パイプライン作成が失敗した理由を理解し、実際のハードウェアリミットに基づく修正を提案できます。

== 戦略: 「協調エンジニアリング」ループ

「エンジンを代わりに書いてくれる」AI は望んでいません。それは保守不能なコードと「ブラックボックス」アーキテクチャにつながります。代わりに、信頼と速度を積み上げる **段階的な勝利（Incremental Wins）** に焦点を当てます。

=== 「ボイラープレート」の勝利
Vulkan は現代グラフィックスの中でもより冗長な API の一つです。単一の `VkGraphicsPipelineCreateInfo` が 200 行に及ぶこともあります。新しい環境では *意図*（「4x MSAA とデプステスト有効の deferred G-Buffer パス用パイプラインを作成」）を記述し、AI に冗長な構造を生成させます。そして最初から打つのではなく、結果を **監査** します。

=== 仕様アシスタントの勝利
同期問題をデバッグしている場面を想像してください。ブラウザにタブを切り替える代わりに、AI アシスタントに尋ねます。*「現在の `VK_IMAGE_LAYOUT_ATTACHMENT_OPTIMAL` の使い方に基づき、Adreno 750 上で `SHADER_READ_ONLY_OPTIMAL` への遷移に必要なアクセスマスクは何か？」* アシスタントには **コンテキストブリッジ** があるため、ターゲットハードウェアに合わせた仕様どおりの答えが返ります。

== 主要インターフェースの選択: IDE 対エージェント

よくある混乱は、「IDE 内」アシスタントと「外部エージェント」のどちらを使うかです。プロの Vulkan ワークフローでは両方必要です。**統合アシスタント**（JetBrains AI や GitHub Copilot など）は「インナーループ」に住み、リアルタイムオートコンプリートやカーソル下のコードの即時説明に優れます。一方、**タスク指向エージェント**（Goose や Aider など）は「アウターループ」に住みます。ディスクリプタ管理を Descriptor Buffers へ移行するような高レベルのタスクを与えると、複数ファイルを横断し、コンパイラを実行し、ビルドが通るまで反復します。

次世代の IDE ではこの境界は曖昧になり、AI エージェントが IDE のコア機能により深く統合されます。それでも、各ツールの役割と責任を理解するにはこの区別は有用です。

両方のセットアップを示し、AI が人間の同僚と同じくチームの一部となるハイブリッド環境を作ります。

== 最初のステップ: Vulkan MCP サーバの構築

プラットフォーム固有のセットアップに入る前に、この一度きりの前提を完了してください。以降のすべての IDE・エージェント章は、すでに **mcp-Vulkan** サーバをクローンしてビルド済みであることを前提とします。結果のバイナリを、パス `/path/to/mcp-Vulkan/vulkan/build/index.js` で各ツールに指し示します。

**Node.js**（LTS）と **npm** が必要です。まだない場合は link:https://nodejs.org/[nodejs.org] から LTS インストーラをダウンロードしてください。

[source,bash]
----
# Clone the repository
git clone https://github.com/gpx1000/mcp-Vulkan.git

# The package.json lives in the vulkan/ subdirectory
cd mcp-Vulkan/vulkan

# Install dependencies and build
npm install
npm run build
----

`mcp-Vulkan` ディレクトリの絶対パスを控えてください（中で `pwd` を実行）。すべてのプラットフォーム章がサーバ登録時にこのパスを使い、`/path/to/mcp-Vulkan/vulkan/build/index.js` として示します。

== 環境ロードマップ: IDE からエージェントへ

続く章では、グラフィックスエンジニアが使う主要な開発エコシステムをカバーするよう構成しています。すべて読む必要はありません——日々の「ラボ」に合うものに集中してください。

まず **JetBrains CLion & Android Studio**——深い C{pp} インデックスとモバイルファースト Vulkan 開発の標準——から始めます。洗練された「意味的認識」を活用し、コードベース構造の深い理解を AI に与える方法を学びます。

Windows 利用者向けには **Visual Studio** 体験を探り、包括的なデバッグツールと、リアルタイム診断支援のための高推論モデルとの橋渡しに焦点を当てます。

Apple 開発者向けには **Xcode & Apple Silicon** エコシステムの独自能力に触れ、ネイティブ知能と M シリーズの高速メモリをローカル推論に活かす方法を示します。

主要環境のセットアップ後、**共通地点** である **Goose & エージェンティックワークフロー** 章に到達します。**Ollama** 駆動のローカルモデルを使い、複数ファイルを横断し、コンパイラを実行し、エンジンアーキテクチャを反復できる **Goose** のような自律エージェントのセットアップを学びます。

最後に **コンテキストブリッジ** に入ります。**Model Context Protocol（MCP）** を使い、AI をライブな Vulkan 仕様に直接つなぐ方法を学びます。

次は、C{pp} 開発の **標準の担い手** から始めます。
