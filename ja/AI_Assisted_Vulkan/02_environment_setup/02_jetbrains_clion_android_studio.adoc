= JetBrains CLion & Android Studio: 標準の担い手

== はじめに: インデクサの力

C{pp} と Android の世界で、JetBrains は長い間ツーリングの高い基準を示してきました。**CLion** と **Android Studio** は単なるテキストエディタではなく、コードの意味構造を理解するよう設計された巨大で洗練されたエンジンです。Vulkan 開発者にとって、この「理解」は AI に与えられる最も価値ある資産です。

**協調エンジニアリング** を真にマスターするには、これらの IDE を単純なツールから、深いインデックスを活用して AI の幻覚を減らす **AI 強化ハブ** へ変えなければなりません。

== コア概念: 意味基盤としてのインデックス

ほとんどの AI ツールはコードを単純なテキスト文字列として扱います。`VkImageCreateInfo` を見ると、学習中に得たパターンに基づいて認識します。しかしエンジンがカスタムラッパや特定版の Vulkan ヘッダを使う場合、汎用 AI は環境に存在しないメンバやフラグをしばしば提案します。

CLion と Android Studio は **Indexing Engine** でこれを解決します。マクロ、テンプレート展開、Vulkan SDK からインクルードされたすべてのヘッダを含む、プロジェクトの完全な **抽象構文木（AST）** を構築します。

この **構文認識** により、「イメージ作成ロジックをリファクタして」と頼んでも、AI は単に推測しません。IDE のインデックスを照会し、`VkImage` がどこで定義されているか、`CMakeLists.txt` でどの拡張が有効か、`vk::raii` ラッパを使っているかを正確に見つけます。AI を IDE のインデックスに橋渡しすることで、「確率的推測」から「決定論的真実」へ移ります。AI はコンパイラと同程度にコードベースを知る **精密ツール** になります。

== CLion を AI ハブへ変える

標準の CLion インストールは「伝統的ツール」です。AI ハブにするには、ネイティブアシスタント、自律エージェント、ハイブリッドモデル戦略という 3 つの主要統合点に焦点を当てます。

=== ネイティブ統合: JetBrains AI Assistant & MCP

JetBrains は **Model Context Protocol（MCP）** を AI Assistant に直接統合しています。これによりアシスタントは外部「ツール」で事実を検証でき、Vulkan 開発者にとって不可欠です。

**Vulkan MCP を AI Assistant に追加する:**

クラウドベースのアシスタント（**Claude 4.6** や **GPT-5.3** など）に Vulkan 仕様の地上真理を与えるには、次の手順に従ってください。これは xref:AI_Assisted_Vulkan/02_environment_setup/01_introduction.adoc#_first_step_building_the_vulkan_mcp_server[環境セットアップのはじめに] で説明したとおり、すでに `mcp-Vulkan` サーバをビルド済みであることを前提とします。

1.  **Settings** を開く（`Ctrl+Alt+S` または `Cmd+,`）。
2.  **Tools > AI Assistant > MCP Servers** へ移動。
3.  **+**（Add）ボタンをクリックし **Local Stdio** を選択。
4.  **Name:** `Vulkan-Spec`
5.  **Command:** `node`（Node.js が PATH にあることを確認）。
6.  **Arguments:** ビルド済み MCP サーバへの絶対パスを指定: `/path/to/mcp-Vulkan/vulkan/build/index.js`。
7.  **Environment Variables:** （任意）他サーバが必要とするキーがあれば追加。

設定後、AI チャット窓の「Tools」アイコンにアクティブな MCP が表示されます。これで `VkVideoProfileInfoKHR` のような特定構造について質問すると、アシスタントは MCP を呼び出し、学習データになくても最新レジストリから正確な定義を取得できます。

**実世界シナリオ: 仕様チェック**
**Dynamic Rendering** を実装しているところを想像してください。`vkCmdBeginRendering` がインスタンス作成で特定拡張の有効化を必要とするか確信がありません。IDE を離れる代わりに、**Vulkan Spec MCP** 付きの AI Assistant を使います。*「現在のインスタンスセットアップはダイナミックレンダリングをサポートしているか？」* と尋ねます。Assistant は MCP ツールでライブ Vulkan レジストリとプロジェクトの `init.cpp` を照会し、実際のコードに基づく決定的な「Yes」または「No」を返します。

=== 自律エージェント: Junie

単純なチャットを超えたい方向けに、JetBrains は **Junie** を導入しました。サイドパネルに住み質問を待つ標準アシスタントと異なり、Junie は真の **自律エージェント** です。

デフォルトで Junie は主要推論エンジンとして **Gemini 3.1 Flash** を使います。速度と巨大なコンテキストウィンドウのために選ばれており、プロジェクト全体と Vulkan 仕様を同時に「読む」ことができます。

**モデルの管理: いつ切り替えるか**

**Gemini 3.1 Flash** は優れたデフォルトですが、特に難しい問題に遭遇したとき、Junie はさらに強力な推論重視モデルへ切り替えられます。例えば複雑なマルチスレッドフレームグラフの設計を頼むなら、Junie 設定で **Gemini 3.1 Pro**、**Claude 4.6**、または **GPT-5.3** へ切り替えたくなるかもしれません。これらのモデルは「高レベル設計」タスク向けに深い推論能力を持ちますが、コストは高くなります。

Junie ツールウィンドウのドロップダウン、または `Settings > Tools > Junie` メニューから好みのモデルを選べます。これにより「推論コスト」と「知能深度」を手元のタスクに合わせられます。多くの開発者はコーディングの 90%——シェーダインタフェース実装やユニットテスト記述など——に Gemini 3.1 Flash を使い、自律エージェントが深い論理バグを解くか新しいエンジンサブシステムを設計する必要があるときだけタスクを Claude 4.6 へ「昇格」させます。

Vulkan プロジェクトでは、IDE の意味モデルとの Junie 統合が大きな利点です。IDE に直接組み込まれているため、インデクサ、デバッガ、ビルドシステムへのファーストクラスアクセスがあります。これにより、*「VMA（Vulkan Memory Allocator）を使いステージングとデバイスローカルメモリの両方を扱う基本 `Buffer` クラスを実装して」* のような高レベルタスクを Junie に割り当てる **自律ワークフロー** が可能になります。Junie はスニペットを返すだけでなく、ヘッダとソースファイルを作成し、`CMakeLists.txt` に追加し、結果のコンパイルを試み、遭遇した構文エラーを修正します。

この **「コンテキストブリッジ」の利点** により、Junie はコードを「歩ける」ためプロジェクトアーキテクチャを理解します。すでに `Device` ラッパを定義していれば、生の `VkDevice` をインスタンス化しようとせず、それを見つけて使います。

=== ブリッジ: 外部エージェントへのコンテキスト提供

大規模な複数ファイルリファクタに **Goose** のような外部エージェントが必要なこともあります。効果的に行うには、エージェントが CLion の目を通してプロジェクトを「見る」必要があります。これは **JetBrains IDE MCP Server** 経由で達成されます。

**JetBrains IDE MCP** プラグインをインストールすると、IDE の内部 API を外部エージェントに公開します。*「`VkFramebuffer` のすべての使用箇所を見つけ、新しい `AttachmentBuilder` クラスへ移行して」* と伝えると、単なるテキスト `grep` ではありません。CLion のインデクサに実際のシンボル使用を尋ね、複雑なテンプレートコードでも一箇所も見逃しません。

== モデル戦略: クラウドとローカルのシームレスなブレンド

**協調エンジニアリング** パラダイムの重要な部分は、タスクに適したモデルを選ぶ能力です。単純な `VkImageView` 作成ループを書くのに、常に数十億パラメータのクラウドモデルが必要なわけではありません。

=== 「ハイブリッドハブ」戦略

JetBrains には独自の AI Assistant があり、多くの異なる外部 AI ツールにネイティブ接続できます。加えて、多くの開発者は **Continue** プラグインや外部エージェントを使い、真に柔軟な環境を作ります。これにより複数の「バックエンド」を設定し、タスクの複雑さに応じて切り替えられます。

新しい同期認識レンダーグラフの設計のような複雑なアーキテクチャタスクには、高推論の **クラウド戦略** モデル——**Claude 4.6** や **GPT-5.3**——を使えます。高価ですが、複雑な C{pp} 抽象化の深い理解があります。逆に、反復的ボイラープレート、ユニットテスト記述、単純関数のリファクタなどの **ローカル戦略** では、**Ollama** 経由の **Qwen 3-Coder（30B）** や **Llama 4** のようなローカルモデルへ切り替えられます。無料、ゼロレイテンシで、プロプライエタリなエンジンコードがマシンを離れないことを保証します。

両方の戦略は JetBrains AI タスクウィンドウ内でネイティブに使え、タスクの複雑さに応じて簡単に切り替えられます。このハイブリッドアプローチは効率とコスト効率を最大化し、Vulkan 開発ニーズに両方の世界の最善を活かせます。

=== 「共通地点」への到達

本チュートリアルの残りに必要な **共通地点** に到達するには、少なくとも 2 つのエンドポイントで IDE を設定すべきです。アーキテクチャ計画と複雑なデバッグ向けの高推論 **クラウドモデル**、および実装とコスト効率の良い反復向けの高速 **ローカルモデル**。

これらが IDE のインデクサ（AI Assistant、Junie、または Continue 経由）にリンクされたら、AI 支援開発の「ベースキャンプ」に到達しています。環境を完成させるには、次に **共通地点** へ進み、自律エージェントとローカル推論エンジンをセットアップしてください: xref:AI_Assisted_Vulkan/02_environment_setup/05_goose_native_agent.adoc[Goose とローカル知能]。

== Android Studio: Gemini の利点

モバイル開発者向けに、**Android Studio** は独自の利点——**Gemini**——を提供します。このモデルは Android NDK とモバイル GPU アーキテクチャ（Qualcomm の Adreno や ARM の Mali など）の機微向けに特に調整されています。加えて、Android Studio は JetBrains ベースのため、CLion 向けに述べた Junie とツールのほとんどは Android Studio でもネイティブに動作します。

=== 「コンテキスト感度」ワークフロー

モバイル Vulkan レンダラで作業するとき、しばしば `VK_ANDROID_external_memory_android_hardware_buffer` のような Android 固有拡張を扱います。これは API の悪名高いほど挑戦的な領域です。

Android Studio では、Gemini は `AndroidManifest.xml` と C{pp} ソースへの深いアクセスを持ちます。*「`AHardwareBuffer` を Vulkan レンダラへどうインポートするか？」* と尋ねると、Gemini はスニペットを返すだけではありません。マニフェストで正しい権限が有効か確認し、NDK バージョンを検証し、Java/Kotlin 側と JNI/Vulkan 側の両方を含む複数ファイル計画を提供します。

== 欠点と段階的な勝利

=== 「重いインデクサ」税
JetBrains の理解の深さにはハードウェアリソースのコストが伴います。巨大エンジンでは、初期インデックスと AI Assistant のオーバーヘッドが相当な CPU と RAM を消費し得ます。大規模リファクタ中にノート PC のファンが回るかもしれませんが、この「税」は前払いです。インデックス後、AI の精度は劇的に上がり、後の状態不一致デバッグの何時間もを節約します。

=== 段階的な勝利: 意味的リファクタリング
この環境での最大の勝利は **意味的リファクタリング** を実行できることです。コア Vulkan 抽象化の変更が必要なら（例: カスタム `Buffer` クラスからより現代的なものへ）、クラスをハイライトし *「意味的移行を実行して」* と AI に頼めます。AST があるため、生テキスト AI モデルでは達成しにくい安全性で、プロジェクト全体にわたるリネーム、シグネチャ更新、呼び出しサイト修正を安全に行えます。

== まとめ: 統合 AI 開発

AI 支援 Vulkan に CLion と Android Studio を使うことは、**意味分析** の活用です。AI がコンパイラが見るものを「見る」ようにすることで「コンテキストギャップ」を減らし、AI が自身のエンジニアリング専門性の深く情報に基づく拡張として働く強力な協調ラボへ IDE を変えます。
