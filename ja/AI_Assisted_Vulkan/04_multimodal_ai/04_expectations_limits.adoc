= 現実的な期待の設定: ビジョンの限界

== はじめに: 視覚分析の限界

ここまでで、「見る」ことができるアシスタントを構築しました。Shadow Acne、Z-fighting、バンディングを高い速度で識別できます。しかし、過信の罠——AI があらゆるピクセル異常を自動で「直す」と期待すること——に落ちやすいです。

**協調エンジニアリング** では、AI は **診断アシスタントであり、コンパイラではない** ことを忘れないでください。コードの結果は見ますが、根底の GPU 状態は見えません。これらの限界を理解することが、初心者ユーザーと熟練エンジニアを分けます。

== コア概念: ピクセルは状態ではない

マルチモーダル AI の最も根本的な限界は、スクリーンショットからの 2D ピクセル配列しか見ないことです。**コンテキストブリッジ** でそのコンテキストを与えない限り、実際の Vulkan オブジェクトにはアクセスしません。

例えば、ちらつく画面を AI に見せたとします。AI はパターンを見て **Z-fighting** を提案します。近平面を 1 時間調整してもちらつきが残る、ということもあり得ます。実際の原因は **同期ハザード**——前の書き込みが終わる前にイメージを読んでいた競合条件——だった、というケースです。ピクセルだけを見る AI にとって、デプスファイトのアーティファクトと同期ちらつきはほぼ同じに見えることがあります。だからこそ、AI の視覚診断は常に **仮説** として扱い、**バリデーションレイヤ** と **RenderDoc** で検証すべきです。

== 「解像度ギャップ」

ほとんどのマルチモーダルモデルは処理中に画像をダウンスケールし、「解像度ギャップ」につながり得ます。シーンの 4K スクリーンショットが、Vision Encoder がスキャンする前にはるかに小さいサイズに圧縮されることがあります。この圧縮により、**サブピクセルエイリアシング**、**マイクロフリッカ**、**テクスチャ継ぎ目** のような微妙な高周波アーティファクトが完全に失われることがあります。全画面ショットを送り、目に見えるシマーがあるのに AI が「クリアだ」と言う場合は、**クロップとズーム** で緩和してください。アーティファクト領域に絞ってクロップすれば、関連ピクセルが AI の限られたビジョンバッファの大半を占め、正しい診断に必要な細部が保たれます。

== 「ハードウェアギャップ」

Vulkan は明示的 API であり、ハードウェアごとに挙動が異なります。NVIDIA カードで見える視覚バグは、AMD カードやモバイル Adreno GPU ではまったく違って見えることがあります。

**ハードウェアコンテキスト** を明示的に与えない限り、AI は「汎用デスクトップ GPU」を仮定します。モバイル向けならプロンプトに次を含めてください。*「Tile-Based Deferred Renderer（TBDR）アーキテクチャ上でデバッグしている。」* この小さなコンテキストで、AI の推論は「バリアを確認」から「サブパス依存と load/store ops を確認」へと変わります。

== まとめ: 情報に基づくグラフィックスエンジニア

マルチモーダル AI はグラフィックスアーティファクトの迅速な識別に強力なツールですが、エンジニアリング判断の代替ではありません。**迅速なパターン認識** と「よくある容疑者」の特定に使い、同期、メモリ安全性、ハードウェア固有ドライバ異常の最終確認には頼らないでください。
