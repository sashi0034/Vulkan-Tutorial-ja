= モデル選択と特化: はじめに

== コア決定: AI モデル戦略の選択

前のセクションでは「コンテキストブリッジ」——AI がコード、ビルドシステム、Vulkan 仕様を見るためのインフラ——を構築しました。次に、AI 支援の旅で最も重要な決定に取り組みます。**適切なタスクに適切なモデルを選ぶこと。**

すべての大規模言語モデル（LLM）が同等ではありません。深いアーキテクチャ能力を持つがレイテンシが高い推論重視のものもあれば、C{pp}23 コード生成に優れるが高レベルシステム設計に苦戦しうる構文特化型もあります。本セクションでは、単に「AI を持つ」段階を超え、Vulkan という独自のハイパフォーマンス世界向けに **特化** します。

== 概念: ベースモデルの重み対ライブコンテキスト

モデル選択をマスターするには、モデルが「知っている」こと（学習データ）と「見られる」こと（ライブコンテキスト）の違いを理解しなければなりません。

1.  **ベースモデル（事前学習済み重み）:** 「箱から出した」状態のモデルです。何百万行もの C{pp} と、学習時点の Vulkan Spec 全体をすでに読んでいます。
2.  **特化（ライブコンテキスト）:** エンジン固有の「味」をモデルに与える方法です。次で実現します。
    *   **MCP（Model Context Protocol）:** Vulkan Spec の正確な版とローカルプロジェクトファイルへのリアルタイムアクセス。
    *   **RAG（Retrieval-Augmented Generation）:** エンジン内部 wiki、コーディング標準、過去のアーキテクチャ決定の検索可能な「記憶」。
    *   **ファインチューニング（LoRA）:** 「このエンジンでは常に `vk::raii` を使い、生ポインタは使わない」といった命名規約を教える小さな「パッチ」の学習。

== なぜ Vulkan に特化が重要か

Vulkan は「生きている」API です。新しい拡張は毎月追加され、ベストプラクティスも進化します（例: Render Pass から Dynamic Rendering へ、Descriptor Sets から Descriptor Buffers へ）。

*   **幻覚のリスク:** 汎用 AI は、学習データでより一般的なため Vulkan 1.0 パターンを提案することがあります。
*   **精度要件:** グラフィックスプログラミングはメモリオフセット、アラインメント、同期において極度の精度を求めます。95% 正確なモデルは危険です。エンジン固有のアラインメントラッパを理解する特化モデルこそ資産です。

== モデル選択ロードマップ: 汎用から特化へ

続く章では、AI アシスタントの選択と特化の技術的・戦略的プロセスを案内します。

まず **ベースモデルの選択** から始めます。**Qwen 3-Coder**、**Mistral-Nemo**、**Llama 4** など、Vulkan 開発向けの主要モデルを比較します。アーキテクチャ計画に必要な複雑・高レベル推論に優れるモデルと、リアルタイムオートコンプリートやシェーダ実装の「インナーループ」に適した小さく速いモデルの違いを学びます。

次に、重要な **ハードウェアと VRAM 予算** に取り組みます。これらのモデルをローカルで動かすには精密な戦略が必要です。VRAM 需要の計算式、「KV Cache Tax」、そして AI が Vulkan アプリから最も貴重なリソースを奪わないための **量子化** の役割を詳しく示します。

続いて **Ground Truth: MCP & RAG 特化** に深く入ります。**MCP** で AI をライブ Vulkan レジストリ（`vk.xml`）に向けて「グローバルコンテキストギャップ」を埋め、**RAG** でエンジン自身の「ローカルコンテキスト」をインデックスする方法を学びます。これらにより、アシスタントがプロジェクト固有のアーキテクチャスタイルに合う有効な API 呼び出しを提案することを保証します。

最後に、高度に特化したアシスタントを求める方向けに **エンジン向けファインチューニング** を探ります。**Teacher-Student** パラダイムを学び、高推論クラウドモデルでローカルアシスタントを「教え」再訓練して **特化 SLM** にします。**LoRA（Low-Rank Adaptation）** でエンジン固有の抽象化とコーディング標準をモデルの重みに「焼き込み」、常時プロンプトなしでもシニアエンジニアリングのパターンをデフォルトにします。

本セクションの終わりには、もはや「汎用」AI は使いません。グラフィックスプログラミングのグローバル標準と、特定コードベースのローカルな「法則」の両方を理解する、特化した Vulkan 対応アシスタントを手にします。
