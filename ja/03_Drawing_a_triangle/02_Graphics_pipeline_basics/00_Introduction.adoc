= はじめに

これからの数章では、最初の三角形を描画するように構成したグラフィックスパイプラインをセットアップしていきます。グラフィックスパイプラインとは、メッシュの頂点とテクスチャを受け取り、レンダーターゲット上のピクセルに至るまでの一連の処理です。簡略化した概要を以下に示します。

image::/images/vulkan_simplified_pipeline.svg[]

*入力アセンブラ*は、指定したバッファから生の頂点データを集めます。また、頂点データ自体を複製せずに特定の要素を繰り返すためにインデックスバッファを使うこともあります。

*頂点シェーダ*は各頂点に対して実行され、一般にモデル空間からスクリーン空間への頂点位置の変換を行います。また、頂点ごとのデータをパイプラインの下流へ渡します。

*テッセレーションシェーダ*は、特定のルールに基づいてジオメトリを細分化し、メッシュの品質を高めます。近くにあるレンガの壁や階段のような表面が平坦に見えにくくするために使われることが多いです。

*ジオメトリシェーダ*は各プリミティブ（三角形、線、点）に対して実行され、それを破棄したり、入力より多くのプリミティブを出力したりできます。テッセレーションシェーダに似ていますが、はるかに柔軟です。ただし、Intelの統合GPUを除くほとんどのグラフィックスカードでは性能があまり良くないため、今日のアプリケーションではあまり使われません。また、ジオメトリシェーダのほぼすべてのユースケースは、より新しいMeshシェーダパイプラインで代替できます。Meshシェーダはレイトレーシングと同様にまったく新しいパイプライン解法であり、標準のグラフィックスパイプライン構成の外に存在します。そのパイプラインについては、将来のチュートリアル節で取り上げます。

*ラスタライゼーション*段階は、プリミティブを*フラグメント*に分解します。これらはフレームバッファ上で塗りつぶされるピクセル要素です。画面外に落ちるフラグメントは破棄され、頂点シェーダが出力した属性は図に示すようにフラグメント間で補間されます。通常、深度テストにより、他のプリミティブのフラグメントの後ろにあるフラグメントもここで破棄されます。

*フラグメントシェーダ*は生き残った各フラグメントに対して呼び出され、どのフレームバッファに、どの色と深度値で書き込むかを決めます。頂点シェーダからの補間データ（テクスチャ座標やライティング用の法線など）を使ってこれを行えます。

*カラーブレンディング*段階は、フレームバッファ上の同じピクセルに対応する異なるフラグメントを混ぜる操作を適用します。フラグメントは単純に上書きしたり、加算したり、透明度に基づいて混ぜたりできます。

緑色の段階は*固定機能*段階と呼ばれます。これらの段階はパラメータで動作を調整できますが、仕組みそのものはあらかじめ定義されています。

一方、橙色の段階は`programmable`（プログラム可能）であり、望む操作を正確に適用する独自のコードをグラフィックスカードにアップロードできます。たとえばフラグメントシェーダを使い、テクスチャリングやライティングからレイトレーサまで実装できます。これらのプログラムは多数のGPUコア上で同時に実行され、頂点やフラグメントなどの多くのオブジェクトを並列に処理します。

OpenGLやDirect3Dのような古いAPIを使ったことがあるなら、`glBlendFunc`や`OMSetBlendState`のような呼び出しでパイプライン設定を自由に変えられたはずです。Vulkanのグラフィックスパイプラインはほぼ完全に不変なので、シェーダを変えたり、別のフレームバッファをバインドしたり、ブレンド関数を変えたりしたい場合は、パイプラインを最初から再作成する必要があります。欠点は、レンダリング操作で使いたい状態の組み合わせすべてを表す、複数のパイプラインを作成しなければならないことです。しかし、パイプライン内で行うすべての操作があらかじめ分かっているため、ドライバははるかに良く最適化できます。

プログラム可能な段階の一部は、やりたいことに応じて省略可能です。たとえば単純なジオメトリだけを描画するなら、テッセレーションとジオメトリ段階は無効にできます。深度値だけに関心があるなら、フラグメントシェーダ段階を無効にできます。これはlink:https://en.wikipedia.org/wiki/Shadow_mapping[シャドウマップ]生成に役立ちます。

次の章では、まず画面に三角形を出すために必要な2つのプログラム可能な段階、すなわち頂点シェーダとフラグメントシェーダを作成します。ブレンドモード、ビューポート、ラスタライゼーションなどの固定機能の構成は、その次の章でセットアップします。Vulkanでグラフィックスパイプラインをセットアップする最後の部分は、入力と出力のフレームバッファの指定です。
