= メモリ一貫性: Slang バリアとパイプラインストール

これまでのセクションでは、同期の「何を」と「どこで」を探りました。ここでは「どのように」に焦点を当てます——具体的には、GPU のパイプラインを満杯に保つために、安全性と性能のバランスをどう取るかです。

== オールインワンバリア: GroupMemoryBarrierWithGroupSync

多くの開発者は Slang の `GroupMemoryBarrierWithGroupSync()` から始めます。これは次の 2 つの重要な操作を組み合わせる高レベル関数です。

1.  **実行同期**: 現在のワークグループ内のすべてのスレッドをこの行で待たせます。隣もバリアに到達するまで、どのスレッドも進めません。
2.  **メモリバリア**: ワークグループが行ったすべてのメモリ書き込み（共有とグローバルの両方）が利用可能かつ可視になることを保証します。

この関数は実質的に同期の「セーフモード」です。アルゴリズムの次のステップのためにすべてのデータが準備できていることを 100% 確信したいときに使います。

=== GLSL: 明示的なバリア

GLSL にはすべてを行う単一の「魔法」関数はありません。代わりに、何を同期するかを明示する必要があります。多くのバグが入り込む場所でもありますが、性能上の勝ち筋が見つかる場所でもあります。

[source,glsl]
----
// Slang の GroupMemoryBarrierWithGroupSync() に相当する GLSL
memoryBarrierShared(); // 共有メモリ書き込みを利用可能／可視にする
barrier();             // 全スレッドがこの点に到達するのを待つ
----

**グローバルメモリ**（SSBO）を扱う場合、`barrier()` だけでは不十分です。他のスレッドがバリアを過ぎて進む前にバッファへの書き込みが実際に可視になるよう、`memoryBarrierBuffer()` も呼ばなければなりません。

[source,glsl]
----
// ワークグループ内の他スレッド向けにグローバルメモリの準備を保証
memoryBarrierBuffer();
barrier();
----

Vulkan 1.4 は **Memory Semantics** でこれをさらに洗練し、同期する「ドメイン」（Uniform、Buffer、Image、Shared）を正確に指定でき、一般バリアの「すべてを同期する」ペナルティを避けられます。

== 同期のコスト

同期は無料ではありません。バリアを呼ぶたびに、実質的に GPU に「今やっていることを止めて待て」と言っています。

*   **ワークグループバリア**は、多くのスレッド（例: 256 や 1024）を巻き込むため高価です。ハードウェアはこれらすべてのスレッドを追跡し、同じ点に到達したことを保証しなければなりません。
*   **パイプラインストール**: 一部のスレッドが作業を速く終えても、他が遅いメモリフェッチで遅れると、速いスレッドはアイドルし、潜在的な **TFLOPS**（毎秒数兆回の浮動小数点演算）を浪費します。

=== 影響の削減

同期の性能ペナルティを最小化するには、次の戦略を検討してください。

1.  **作業のバッチ化**: バリアの間でできるだけ多くの作業を行う。バリアを 2 つ持つ 1 つの大きなカーネルは、それぞれバリアを 1 つ持つ 2 つの小さなカーネルより速いことが多い。
2.  **共有メモリのダブルバッファリング**: 同じ共有メモリ配列への読み書き（バリアが必要）の代わりに、2 つの配列を使う。`B` から読みながら `A` に書き、その後スワップする。
3.  **アトミック演算**: グローバルカウンタのインクリメントのような単純な作業には、`InterlockedAdd`（Slang が HLSL から継承）を使う。**アトミック演算**はハードウェアレベルで同期を処理し、他スレッドから「割り込み不能」であるため、手動バリアよりはるかに速いことが多い。

== Vulkan 1.4 での細粒度制御

現代の Vulkan（1.3+）と Synchronization 2 は、さらに粒度の細かい制御を許します。シェーダ内で、言語がサポートしていればより具体的なバリア型を使えます。

*   **`GroupMemoryBarrier()`**: 実行同期を強制せず、メモリだけを同期する。
*   **サブグループバリア**: 32/64 スレッドの束（**サブグループ**）内の同期は、GPU のグローバルスケジューラを巻き込む必要がないため、ワークグループ全体の同期より大幅に高速です。
