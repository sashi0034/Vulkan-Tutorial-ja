= 命令スループットの分析: コンピュートバウンド対メモリバウンド

すべての GPU カーネルには「心拍」——命令を処理しメモリにアクセスする速度——があります。この心拍を理解することが効果的な最適化の鍵です。そのためには、2 つの主要なボトルネック型を区別する必要があります。**コンピュートバウンド** と **メモリバウンド** です。

== Roofline モデル

これを可視化する強力な方法が **Roofline モデル** です。x 軸が **算術強度**（メモリバイトアクセスに対する数学演算の比率）、y 軸が **性能**（GFLOPS）のグラフを想像してください。

このモデルの「屋根」は、ハードウェアのピーク理論性能によって決まります。

* カーネルの算術強度が低い（メモリアクセスが多く数学が少ない）場合、屋根の「斜面」に閉じ込められます——**メモリバウンド** です。
* カーネルの算術強度が高い場合、屋根の平らな部分に当たります——**コンピュートバウンド** です。

== ボトルネックの特定

これらのボトルネックを特定するには、NVIDIA Nsight、AMD Radeon GPU Profiler（RGP）、Intel VTune のようなプロファイリングツールで **ハードウェアメトリクス** を見る必要があります。モバイルと組み込みターゲットでは、異なるが類似のツールセット——Arm Performance Studio（Streamline）、Qualcomm Snapdragon Profiler、Imagination PVRTune、またはベンダー中立の Android GPU Inspector——を使い、*実際のデバイス* 上で実行しなければなりません。デスクトップ GPU はスマートフォンの振る舞いについて誤解を招きます。モバイルプロファイリングワークフローは xref:Advanced_Vulkan_Compute/12_Mobile_and_Embedded_Compute/03_mobile_optimization.adoc[モバイル GPU 向け最適化] で詳しく扱います。

=== コンピュートバウンドカーネル
**コンピュートバウンド** カーネルは、ハードウェアの算術ユニット（**ALU**）が完全に占有されているものです。これらのカーネルの特徴は次のとおりです。

* **高い ALU 利用率**: ALU が時間の大部分でアクティブ。
* **低いメモリスループット**: メモリバスは比較的アイドル。
* **修正**: 数学を単純化し、**混合精度** を使うか、**コオペラティブ行列** のような専用ユニットを活用する。

=== メモリバウンドカーネル
**メモリバウンド** カーネルは、ALU がしばしばアイドルし、VRAM からのデータを待っているものです。これらのカーネルは次を示します。

* **低い ALU 利用率**: 算術ユニットがメモリ待ちで「ストール」している。
* **高い VRAM スループット**: ハードウェアの帯域限界に達している。
* **修正**: **メモリ合体（Memory Coalescing）** を改善し、データ再利用に **共有メモリ（LDS）** を使うか、VRAM に触れずにデータ共有する **サブグループ演算** を使う。

== ストール理由の理解

現代のプロファイラは、ウェーブフロントが*なぜ*ストールしたかを教えてくれます。一般的な理由は次のとおりです。

* **命令フェッチストール**: ハードウェアが次の命令を十分速くフェッチできない（コンピュートでは稀）。
* **実行ストール**: ALU が長時間命令（複雑な超越関数など）で忙しい。
* **メモリ依存ストール**: 最も一般的なストール——ウェーブフロントが VRAM からの `load` 完了を待っている。

== レイテンシ隠蔽と占有率

第 2 章で議論したように、GPU はアクティブなウェーブフロント間を切り替えることでメモリレイテンシを隠蔽します。これが **占有率** がこれほど重要な理由です。占有率が低いと、メモリ待ちの間に GPU が「仕事」を使い果たし、ALU がアイドルして性能が悪化するかもしれません。

ただし注意してください。高い占有率が常に良いわけではありません。占有率が高すぎると **キャッシュ競合** が増え、異なるウェーブフロントが互いのデータを L1 や L2 キャッシュから絶えず追い出すかもしれません。占有率の「スイートスポット」を見つけることは、チューニングプロセスの重要な部分です。

最適化は反復プロセスです。プロファイルし、ボトルネックを特定し、狙いを定めた修正を適用し、再びプロファイルします。これが、GPU の巨大な並列潜在能力を最大限に活かす、真に最適化された解に最終的にたどり着く方法です。
