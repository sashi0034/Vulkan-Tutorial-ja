= サブグループシャッフル、ブロードキャスト、算術演算

== メモリなしのデータ交換

前のセクションでは、**サブグループ** をハードウェアのネイティブ実行幅（例: 32 または 64 スレッド）として紹介しました。サブグループを本当に強力にするのは、メモリに一切書かずに呼び出し間でデータを共有できることです。VRAM も LDS もなく——レジスタ間通信だけです。

これは主に 3 カテゴリの演算で行われます。**ブロードキャスト**（1 つの値を全員へ送る）、**シャッフル**（特定スレッド間で値を入れ替える）、**算術（リダクション／スキャン）**（サブグループ全体で数学を行う）。

== サブグループブロードキャスト

サブグループ通信の最も単純な形は **ブロードキャスト** です。サブグループ内の 1 スレッドが、同じサブグループ内の他の全スレッドとローカル値を共有できます。

[source,slang]
----
// サブグループブロードキャストの Slang 例
float localValue = computeSomeData();
float sharedValue = WaveReadLaneAt(localValue, 0); // 全員がスレッド 0 の値を得る
----

上の例では、サブグループ内のすべてのスレッドが同じ `sharedValue` を持ち、それはもともとスレッド 0 だけのものでした。「アンカー」値や、1 スレッドだけが計算・ロードすればよい設定データの共有に非常に有用です。

=== GLSL: サブグループのやり方

GLSL では `subgroup` 組み込み関数を使います。適切な拡張（演算に応じて通常 `GL_KHR_shader_subgroup_basic` または `GL_KHR_shader_subgroup_ballot`）を有効にする必要があります。

[source,glsl]
----
#extension GL_KHR_shader_subgroup_basic : enable

// サブグループブロードキャストの GLSL 相当
float localValue = computeSomeData();
float sharedValue = subgroupBroadcast(localValue, 0);
----

== サブグループシャッフル

ブロードキャストが 1 つの値を全員に送るのに対し、**シャッフル** はより複雑なパターンを許します。サブグループ全体のレジスタの **順列**（並べ替え）と考えることができます。

Slang では、一般的なインデックス付けに `WaveReadLaneAt` を使うか、相対移動向けのより具体的な関数を使えます。

[source,slang]
----
// 各スレッドが隣と値を「入れ替え」（32 スレッドを仮定）
uint neighborIdx = (WaveGetLaneIndex() + 1) % 32;
float neighborValue = WaveReadLaneAt(localValue, neighborIdx);
----

現代の GPU は `WaveReadLaneFirst` やビット単位シャッフルのような、より専門的なシャッフルもサポートします。これらはハードウェアのデータパスに直接対応するため、一般シャッフルより効率的なことが多いです。

== サブグループ算術（リダクションとスキャン）

データを動かすだけでなく、サブグループは単一命令で全スレッドにわたる数学を実行できます。これらを **リダクション** と **スキャン** と呼びます。

*   **サブグループリダクション**: 全スレッドの値を 1 つの結果にまとめる（例: `WaveActiveSum`、`WaveActiveMin`、`WaveActiveMax`）。
*   **サブグループスキャン（Inclusive／Exclusive）**: 各スレッドは、自身のインデックスまでの全スレッドの部分和（または min/max）を受け取ります。**inclusive** スキャンでは現在スレッドの値が含まれ、**exclusive** スキャンでは含まれません。

[source,slang]
----
// サブグループ内の全ローカル値の和を計算
float subgroupTotal = WaveActiveSum(localValue);

// 各スレッドが、より低い ID を持つスレッドの値の和を得る
float runningSum = WavePrefixSum(localValue);
----

これらの演算は、高性能プレフィックス和、**ストリーム圧縮**（配列をアクティブ要素だけにフィルタする）、並列リダクションの構成要素です。共有メモリとバリアを使う複雑なマルチパスカーネルを書く代わりに、同じ作業を数サイクルで単一サブグループ内でできることがよくあります。

== 適切な演算の選択

どこでもサブグループ演算を使いたくなりますが、単一サブグループ内でしか動かないことを覚えておいてください。1024 スレッドのワークグループ全体でデータを共有する必要がある場合は、サブグループ間の橋渡しに依然として **共有メモリ（LDS）** が必要です。

ただし、「サブグループファースト」のアプローチがしばしば最速です。可能な限りサブグループ内で作業を行い、別のサブグループと通信する絶対必要なときだけ LDS を使います。
