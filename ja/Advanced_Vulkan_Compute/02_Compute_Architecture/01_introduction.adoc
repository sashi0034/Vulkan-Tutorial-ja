= コンピュートアーキテクチャと実行モデル: はじめに

== 概要

効率的なコンピュートカーネルを書くには、「ワークグループ」や「呼び出し」という抽象的な実行モデルを超え、これらが物理ハードウェアへどう対応するかを理解する必要があります。Vulkan はベンダー横断の API を提供しますが、その下にある AMD、NVIDIA、Intel のシリコンは、それぞれ独自のデータ扱い方を持ちます。Vulkan デバイスの最大の導入基盤をなす Arm（Mali/Immortalis）、Qualcomm（Adreno）、Imagination（PowerVR）の**モバイル** GPU も同様で、タイルベース設計は同じ概念をまったく異なるシリコンへマッピングします。詳細は xref:Advanced_Vulkan_Compute/12_Mobile_and_Embedded_Compute/01_introduction.adoc[モバイルと組み込みコンピュート] で扱います。

本章では、シェーダコードとシリコンの間のギャップを埋めます。`vkCmdDispatch` で定義する 3D グリッドシステムが、GPU の **Compute Unit（CU）** や **Streaming Multiprocessor（SM）** へどのように分割・配布されるかを探ります。

=== シリコンの言葉

本題に入る前に、用語を揃えましょう。ベンダーごとに同じ概念に異なる名前を使います。

*   **ワークグループ**（Vulkan/OpenCL）は、しばしば **スレッドブロック**（CUDA）に対応します。
*   **呼び出し**（Vulkan）は、単に **スレッド** です。
*   **サブグループ**（Vulkan）は、**ウェーブフロント**（AMD）や **ワープ**（NVIDIA）と呼ばれます。モバイルでは Arm も同等のものを **warp** と呼びますが、実用上のサイズは大きく異なり（4〜128）、実行時に問い合わせる必要があります。
*   **Compute Unit**（AMD）は **Streaming Multiprocessor**（NVIDIA）に相当し、Arm Mali や Qualcomm Adreno では **shader core** に相当します。

これらの対応を理解すれば、ターゲットがデスクトップでもモバイルでも、ハードウェア固有のドキュメントやパフォーマンスガイドを読めます。

== ハードウェアマッピング

ワークロードをディスパッチすると、GPU のハードウェアコマンドプロセッサがグローバルグリッドを個々のワークグループに分割します。これらのワークグループがスケジューリングの基本単位です。

GPU 実行モデルの重要な規則は**ワークグループ原子性**です。ワークグループが物理コンピュートユニットに割り当てられると、そのすべての呼び出しはそのユニット上に留まり、完了まで離れません。複数ユニットに分割できません。この局所性こそが **共有メモリ（LDS - Local Data Store）** を可能にします。ワークグループ内の全スレッドが物理的に同じハードウェアブロック上にあるため、専用の超高速キャッシュを共有できるのです。

=== 呼び出しと SIMD

ワークグループがスケジューリング単位である一方、**呼び出し**は実行の最小単位です。ただし GPU は **SIMD（Single Instruction, Multiple Data）** マシンです。呼び出しを一つずつ実行するのではなく、小さな束（サブグループ）にまとめます。

この束では、すべての呼び出しが同じ時刻にまったく同じ命令を実行しますが、データは異なります。数学には非常に効率的ですが、大きな落とし穴があります。**分岐発散（Branch Divergence）** です。コードに `if` があり、一部のスレッドが左、他が右へ進むと、ハードウェアは*両方*のパスを実行し、各パスで非アクティブスレッドをマスクアウトしなければなりません。

== パフォーマンス指標

本セクションでは、ハードウェア活用度を決める 2 つの主要指標に焦点を当てます。

1.  **占有率（Occupancy）**: 「並行性」の指標です。コンピュートユニット上のアクティブワークグループ数が理論最大に対してどれだけあるかを表します。高い占有率は**レイテンシ隠蔽**に役立ちます。ある束が遅い VRAM からのメモリフェッチを待っているとき、スケジューラは数学の準備ができている別の束に即座に切り替えられます。
2.  **帯域効率**: 「スループット」の指標です。現代の GPU は巨大なメモリ帯域を持ちますが、データ整列が悪いと簡単に無駄になります。Vulkan 1.4 の **スカラーレイアウト** がデータを密に詰め、シェーダが VRAM からフェッチしたすべてのバイトを実際に使えるようにする方法を見ていきます。
