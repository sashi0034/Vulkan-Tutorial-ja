= モバイルと組み込みコンピュート: はじめに

== 概要

本シリーズではこれまで、主に「大型」環境——ギガバイト級の専用 VRAM、広いメモリバス、余裕のある電力予算を持つディスクリートデスクトップ／ワークステーション GPU——を想定してきました。しかし世界中の Vulkan 対応デバイスの大多数はデスクトップ GPU ではなく、スマートフォン、タブレット、自動車ダッシュボード、セットトップボックス、ドローン、シングルボードコンピュータに搭載された **モバイル** および **組み込み** プロセッサです。

Vulkan は当初からクロスプラットフォーム API として設計されており、これには電力と面積に制約のあるこれらのデバイスも含まれます。実際、今日の Vulkan ドライバの最大の導入基盤は **Android** スマートフォン上で動いています。コンピュートワークロードを何十億ものデバイスに届けたいなら、モバイルを後回しにはできません。

本章では、開発環境のセットアップから最適化と移植性まで、モバイルと組み込みプラットフォームでのコンピュート開発を紹介します。Android Studio と NDK の設定、OpenCL-on-Vulkan ツーリング（`clvk`/`clspv`）の Android および組み込み Linux プロジェクトへの統合、Raspberry Pi や NVIDIA Jetson のようなデバイスでの SSH リモート開発向け CLion セットアップをカバーします。高レベルの Android パッケージングについては、メインチュートリアルの xref:14_Android.adoc[Android] 章が主参照のままです。ここでは *コンピュート固有* の関心と、下のハードウェアがデスクトップではなくタイルベースのモバイル GPU であるときに何が変わるかに焦点を当てます。

== なぜモバイルコンピュートが重要か

モバイル GPU はもはやおもちゃではありません。Arm（Mali、Immortalis）、Qualcomm（Adreno）、Imagination（PowerVR）、Apple（iOS では MoltenVK 経由）の現代モバイル部品は次をサポートします。

*   完全なワークグループと共有メモリセマンティクスを持つ **コンピュートシェーダ**。
*   **サブグループ演算**（第 4 章の「Wave」組み込み）、しばしば固定または問い合わせ可能なサブグループサイズ付き。
*   **16 ビットストレージと算術**（FP16/Int16）。これらのデバイスでは 32 ビットより*速く電力効率も良い*ことが多い。
*   **Buffer Device Address**、**Descriptor Indexing**、そして `clvk` のようなレイヤードエコシステムを動かすために必要な機能の増加。

これによりモバイルは、オンデバイス機械学習推論、コンピュテーショナルフォトグラフィ、リアルタイム動画処理、物理——これまで築いてきたすべてのワークロード——の優れたターゲットになります。

== なぜ組み込みコンピュートが重要か

ポケットの中のスマートフォンを超え、Vulkan コンピュートはますます長い尾の **組み込み** システム上で動きます。自動車インフォテインメントと運転支援ユニット、産業ビジョンシステム、ロボティクスコントローラ、デジタルサイネージ、Raspberry Pi（`v3dv` ドライバ経由）や NVIDIA Jetson モジュールのようなシングルボードコンピュータです。これらのデバイスはモバイルの多くの制約——限られた電力、共有メモリ、熱制限——を共有しますが、独自のものも追加します。リアルタイム要件、安全認証、ヘッドレス動作、異例に長い製品寿命です。

== 章のロードマップ

1.  **Android 上のコンピュート**: Android Studio、NDK、Gradle シェーダコンパイルのセットアップ。ディスパッチと結果の消費。Android での OpenCL 移植性のための `clvk`。タイルベースモバイル GPU のアーキテクチャ上の癖。
2.  **モバイル GPU 向け最適化**: 電力、帯域、精度の戦略——VRAM トラフィックの最小化、FP16 の活用、デバイスを涼しく保つ——ベンダープロファイラへのリンク付き。
3.  **モバイルを超えて: 組み込みとヘッドレス Vulkan コンピュート**: CLion、Raspberry Pi、NVIDIA Jetson 向けの開発環境セットアップ。組み込み Linux 上での `clvk`。ディスプレイなしのヘッドレス Vulkan コンピュート。完全なツール参照表。

本章の終わりまでに、「高度なコンピュート」がデスクトップの専有領域ではないことが分かるでしょう——同じ並列のマインドセットが当てはまりますが、*制約*は鋭くなり、そこで慎重なエンジニアリングが最も報われます。
