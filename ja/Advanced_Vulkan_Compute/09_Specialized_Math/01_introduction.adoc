= コオペラティブ行列と専用数学

これまでの章では、データを効率的に動かす方法と、複雑なワークロードをオーケストレーションする方法を見てきました。では、ワークロード自体が計算集約的な場合はどうでしょうか。長い間、GPU コンピュートは浮動小数点演算（FP32）と同義でした。しかし現代のハードウェアは、非常に特定の種類の仕事——高速線形代数——向けに設計された専用ユニットを含むよう進化しています。

これが **コオペラティブ行列（Cooperative Matrices）** の世界です。NVIDIA の「Tensor Core」や AMD の「Matrix Core」を聞いたことがあるかもしれませんが、Vulkan は `VK_KHR_cooperative_matrix` 拡張（現在は Vulkan 1.4 の一部）を通じて、これらの専用ユニットへのベンダー中立な抽象を提供します。これらのユニットは機械学習専用ではありません。重い行列乗算と累積（**GEMM**——General Matrix-Matrix Multiplication——演算）を伴うあらゆるタスクに極めて強力です。

大きな連立一次方程式を解く必要がある流体シミュレーションや、複雑な変換に依存する信号処理パイプラインを構築している場合でも、コオペラティブ行列は巨大なスループット向上を提供できます。ハードウェアの専用ユニットで小さな行列乗算を直接行うことで、標準のコンピュートシェーダループが届けられるものをはるかに超える性能を達成できます。

本章では、これらの専用数学ユニットがどう動くかを掘り下げます。Slang と GLSL で `cooperative_matrix` 型をどう使うかを探り、重要なところで精度を保ちつつ計算に FP16 や Int8 を使う **混合精度** の活用法を見ます。これは単なる速度以上の話です。高性能コンピューティングタスク向けに、現代 GPU シリコンの潜在能力をフルに活用することです。
