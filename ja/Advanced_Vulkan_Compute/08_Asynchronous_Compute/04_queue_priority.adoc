= キュー優先度: レイテンシクリティカルなワークロードの管理

複数キューを扱うとき、実質的に GPU に複数の作業ストリームを渡し、ハードウェアスケジューラに優先順位の決定を任せています。デフォルトではスケジューラは公平であろうとしますが、高性能エンジンでは「公平」が常に望むものではありません。次フレームまでに*必ず*終わらなければならない重要な物理更新がある一方、背景のパストレースデノイザは数ミリ秒余分にかかっても構わないかもしれません。

Vulkan は **キュー優先度** を通じてこれに影響を与える方法を提供します。論理デバイス（`VkDeviceCreateInfo`）を作成するとき、キューファミリー内の各キューに `priority` を指定します。この値は 0.0 から 1.0 の浮動小数点数で、1.0 が最高優先度を表します。

[source,cpp]
----
// 高優先度コンピュートキューのセットアップ
float priorities[] = { 1.0f, 0.1f }; // 物理は高優先度、デノイズは低優先度
vk::DeviceQueueCreateInfo queueCreateInfo {
    .queueFamilyIndex = computeQueueFamilyIndex,
    .queueCount = 2,
    .pQueuePriorities = priorities
};
----

== スケジューラが優先度をどう使うか
キュー優先度は**ヒント**であり保証ではないことを理解することが重要です。正確な振る舞いはハードウェアの内部スケジューラに大きく依存します。ほとんどの現代 GPU は次の 2 つの主な戦略のいずれかを使います。

1.  **厳格優先度**: 準備ができている高優先度キューのタスクを常に選びます。応答性には優れますが、高優先度キューが常に忙しいと低優先度タスクの **飢餓（starvation）** につながり得ます。
2.  **重み付きラウンドロビン**: スケジューラは優先度に基づき各キューに一定割合の実行時間を割り当てます。たとえば優先度 1.0 のキューは優先度 0.5 のキューの 2 倍の「スケジューリングスロット」を得るかもしれません。

ハイエンドデスクトップ GPU はしばしば、低優先度タスクを **プリエンプト**（例: 長時間動くコンピュートシェーダを止める）して高優先度のものに席を譲れる洗練されたハードウェアを持ちます。ただしプリエンプションは無料ではなく、GPU 状態の保存と復元を伴い、数マイクロ秒かかることがあります。

== グローバルキュー優先度
VR（Virtual Reality）コンポジタや AR（Augmented Reality）空間トラッカーのように、レイテンシが本当に唯一重要なシステムの場合、さらに多くの制御が必要かもしれません。そこで `VK_EXT_global_priority` が登場します。この拡張により、キューに **リアルタイム** 優先度を要求できます。

デバイス上の他キューに対してのみ相対的に働く標準キュー優先度と異なり、グローバル優先度はドライバ（および OS）に、同じ GPU 上で動く他アプリケーションよりさえワークロードが重要だと伝えます。誤用するとシステムの残りの部分でスタッターを起こし得るため、控えめに使ってください。

== 「単一キューの罠」を避ける
よくある誤りは、同じキューファミリー内に複数の高優先度キューを作ることです。こうすると、実質的に「先着順」モデルに戻ります。ハードウェアスケジューラが仕事をするには、明確で区別された優先度を提供しなければなりません。

もう一つの重要な考慮は **キューファミリー**（類似の能力を持つキューのグループ）の選択です。一部の Vulkan 実装は、異なる能力を持つ複数のキューファミリーを提供します。たとえば「Dedicated Compute」キューファミリーは、グラフィックスエンジンとリソースを共有しないコンピュートディスパッチ専用ハードウェアを持ち、より効率的で **パイプラインバブル**（GPU 実行タイムラインの隙間）を起こしにくいかもしれません。各キューファミリーが何を提供するかを理解するため、常に `VkQueueFamilyProperties` を確認してください。

実際には、キュー優先度の管理はバランスの技です。正しく使えば、エンジンの応答性を保ち、最も重要なタスクが常に必要な緊急度で扱われることを保証する強力な道具です。このオーケストレーションこそ、真に高度な Vulkan エンジンの特徴です——単に「仕事をする」ことを超え、「正しい順序で正しい時刻に仕事をする」ことへ進むことです。
