= ストールの可視化: パイプラインバブルを見つける

== ハードウェアプロファイラ

同期コードを最適化する最も効果的な方法は、GPU が実際に何をしているかを見ることです。パイプラインを可視化するために、**NVIDIA Nsight Graphics** や **AMD Radeon GPU Profiler** などのハードウェアプロファイラを使います。

これらのツールでは、GPU の各部分（グラフィックスコア、コンピュートコア、転送エンジン）がいつ忙しいかを正確に示す「タイムライン」ビューが見られます。**パイプラインバブル** はこのタイムライン上の隙間——到達していない依存を待っているためにハードウェアがアイドルになる期間——です。

== 原因の特定

バブルを見つけたら、その原因を特定しなければなりません。本物の依存（例: G-Buffer 完了を待つライティングパス）ですか？それとも過度に保守的なバリアによる人工的なストールですか？

よくある誤りは、すべてのバリアに `vk::PipelineStageFlagBits2::eAllCommands` を使うことです。これは GPU に「先行するすべてのコマンドが終わるまですべて止めよ」と伝えます。巨大な「金づち」であり、大きなバブルを作れます。代わりに、常に可能な限り最も具体的なステージマスク（例: `eColorAttachmentOutput`）を使うべきです。

== 実践的な洗練

マスクを洗練するには、次のプロセスに従います。

1.  **バブルを見つける**: プロファイラのタイムラインで隙間を見つける。
2.  **依存を特定する**: 隙間の直前のバリアを見る。
3.  **ステージマスクを洗練する**: より早いステージで依存を満たせないか確認する。例えば、シャドウパスは前フレームの頂点作業が終わり次第開始できますか？
4.  **修正を検証する**: プロファイラを再実行し、バブルが縮んだか消えたかを確認する。
