= 転送キューとアセットストリーミング同期: ノンブロッキングアップロード

== はじめに

モダンなオープンワールドゲームや複雑な建築ビジュアライゼーションでは、すべてのアセットを事前に読み込む余裕はありません。プレイヤーが世界を移動するにつれ、テクスチャ、メッシュ、アニメーションデータをバックグラウンドでストリームする必要があります。これをメインのグラフィックスキューで行うと、大きなアップロードをサブミットするたびに「スタッター」（フレーム落ち）を招く危険があります。

解決策は **専用転送キュー** を使うことです。ほとんどの現代 GPU には、CPU 可視のステージングバッファから GPU 最適メモリへデータを移すことに特化したエンジンがあります。このエンジンはグラフィックスとコンピュートユニットから完全に独立して走れ、フレームレートに影響を与えずにギガバイト単位のデータをストリームできます。

== ステージングパイプライン

アセットストリーミングは多段階のプロセスです。まず CPU が **ステージングバッファ** をマップし、生データ（PNG やメッシュファイルなど）を書き込みます。次に転送キューを使い、そのデータを **GPU 最適バッファまたはイメージ** へコピーします。最後に、グラフィックスキューにデータが準備できたことを通知し、シェーダで使い始められるようにします。

課題はいつものように同期です。次を保証しなければなりません。

1.  **CPU から転送へ**: CPU がステージングバッファへの書き込みを終えるまで、転送キューがコピーを始めないこと。
2.  **転送から GPU へ**: 転送操作が完了し、データが GPU メモリから見えること。
3.  **転送からグラフィックスへ**: 転送キューが作業を終え、必要ならリソースの所有権をリリースするまで、グラフィックスキューがテクスチャをサンプリングしようとしないこと。

== 構築するもの

この章では、堅牢でノンブロッキングなアセットストリーミングシステムを実装します。次を探ります。

1.  **ノンブロッキングデータアップロード**: バックグラウンドのテクスチャ／バッファストリーミングに専用転送キューをどう活用するか。
2.  **ステージング同期**: グラフィックスキューが新しいデータをサンプリングする前に転送完了を待つよう、**タイムラインセマフォ** で協調する。
3.  **所有権ハンドシェイク**: 第 3 章で学んだキューファミリー所有権転送を、バックグラウンドストリーミングシステムの文脈で実装する。

この章の終わりには、完璧に滑らかでスタッターのないフレームレートを保ちながら、エンジンがバックグラウンドで大量のデータを読み込めるストリーミングアーキテクチャを手にしているはずです。
