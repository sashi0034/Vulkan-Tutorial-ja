= パイプラインバリアとレイアウト遷移: コアループ

== はじめに

前章が GPU ステージ間の理論的な「ハンドシェイク」の理解だったとすれば、この章では実際の実装に手を染めます。モダンな Vulkan 1.3+ の世界では、`vk::ImageMemoryBarrier2` が同期ツールボックスで最もよく使う道具です。イメージのレイアウト遷移、異なるハードウェアキャッシュ間でのデータの可視性の確保、高性能レンダリングに必要な複雑な状態変化の管理に使います。

イメージを単なるピクセルの格子と考えがちですが、GPU にとっては、異なる種類のアクセス向けに最適化できる洗練されたリソースです。カラーアタッチメントとしての書き込みに適したレイアウトは、フラグメントシェーダでのサンプリングにはひどい場合があります。これらの遷移を効率的に——そして本当に必要なときだけ——管理することが、スタッタするレンダラと滑らかな 60 FPS 体験を分けます。

この章では、これらのバリアの仕組みを深く掘り下げます。まず、レガシーな「レンダーパス」システムをほぼ置き換えた **ダイナミックレンダリング** の文脈におけるイメージバリアそのものの解剖から始めます。次に、Vulkan で最も誤解されがちな話題の一つである **キューファミリー所有権** に取り組みます。これは、テクスチャやバッファなどのリソースをエンジンの Graphics・Compute・Transfer キュー間で移したいときに必要な明示的な「引き渡し」です。

最後に、選択が性能に与える影響を見ます。Vulkan は **グローバルメモリバリア** と、より具体的なリソース束縛バリアの選択肢を与えます。どちらをいつ使うかを学び、不要なストールを入れずにハードウェアを全力で動かすために、ドライバにちょうど適切な量の情報を与えられるようにします。

モダン Vulkan 同期の主力である Image Memory Barrier 2 から始めましょう。
