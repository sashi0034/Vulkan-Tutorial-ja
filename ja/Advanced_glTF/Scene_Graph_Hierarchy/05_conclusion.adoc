= まとめ: キャラクターのための基盤

本章では、高度なキャラクターパイプラインの要求を支えるよう、エンジンアーキテクチャを大きく進化させました。「シンプルなエンジンの構築」シリーズの単純なハイブリッドシーン管理を超え、**統一されたグローバルシーングラフ** を実装しました。

本章の主な成果:

*   **統一階層**: ワールドエンティティと内部モデルノードの境界を溶かし、エンティティ横断の親子付けと相互作用をシームレスにしました。
*   **Dirty Flag 最適化**: 冗長な行列乗算を排除する状態追跡システムを実装し、数百ボーンの複雑なスケルトンを追加してもエンジンが性能を保てるようにしました。
*   **データ指向設計**: インデックスベースのリンクを持つノードのフラット配列へ移ることで、CPU キャッシュ効率を最大化し、大規模キャラクターシーンにエンジンを備えました。
*   **双方向物理同期**: シーングラフと物理ソルバの間の数学的橋を確立し、キネマティックなアニメーション制御とダイナミックなラグドールシミュレーションの両方を可能にしました。
*   **メタデータ駆動の自動化**: glTF の "extras" を抽出してアセットパイプラインを近代化し、アーティストが 3D ツール内で物理プロキシと相互作用フックを自動定義できるようにしました。

この最適化されたシーングラフは、以降のすべてに不可欠な「神経系」です。この堅牢な基盤がなければ、スケルタルアニメーション、IK、物理駆動ラグドールといった機能の実装は、断片的で最適化されていないコードベースにつながります。

世界が複雑な階層を効率的に管理できるようになった今、キャラクターに命を吹き込む準備が整いました。次章では **高度なスケルタル＆コンピュートスキニング** を実装し、頂点変形の重い処理を GPU へ移し、表現豊かで高性能なアニメーションの舞台を整えます。

== 検証: 確認すべき点

シーングラフ実装を検証するには:

1.  **トポロジカルソート**: 親ノードが常に子より先に `nodes` ベクトルへ追加されることを確認します。
2.  **Dirty Flag 伝播**: `update_transforms()` にブレークポイントを置き、親が変更されたときすべての子に `WorldDirty` が正しくセットされることを確認します。
3.  **行列の正しさ**: 単純な二ノード階層について、`world_matrix` の結果を既知の良いソース（glTF-Validator のノードトランスフォームなど）と比較します。
