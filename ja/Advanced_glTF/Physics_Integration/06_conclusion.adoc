= 物理統合: まとめ

== 構築したもの

本章はキャラクターシステムの物理層を構築しました。解いた個別の問題がどれほど多いかを一歩下がって味わう価値があります。

まず、物理エンジンにメッシュ幾何をそのまま渡して良い結果を期待できない *理由* を確立するところから始めました。実用的な答え—プロキシコライダー—は妥協のように聞こえますが、そうではありません。カプセルとボックスはメッシュコライダーより安定したシミュレーションを生み、速く動き、メモリも少なく、しかも高忠実度近似と視覚的に区別できないラグドール挙動を生み出します。各身体部位に正しい形状を選ぶ技術は経験で育つクラフトスキルであり、用意した glTF extras システムにより、アーティストはコード変更なしにそれらの形状を直接制御できます。

物理拘束を使ってプロキシコライダーを機能するスケルトンへつなぎました。ここでの概念的飛躍—Ball-and-Socket 拘束が肩を表し、Hinge が膝を表すと理解すること—は、コードのいずれよりも重要です。そのメンタルモデルがあれば、角度制限値は調整すべき数値にすぎません。提供したヒューマノイド参照範囲は出発点であり仕様ではありません。すべてのゲームがキャラクターに求める感触は異なります。

次に **ラグドールハンドオフ問題** を正しく解きました。それはつまり、キャラクターの現在の動きから物理ボディ速度を初期化し、ステートマシンで遷移を管理し、ブレンド中も拘束位置が常にほぼ有効になるよう物理エンジンへキネマティック更新を送り続け、物理結果をシーングラフへ読み戻してコンピュートスキニングパイプラインが正しいアニメーション状態を見るようにすることです。

最後に、ビットマスク設計で自己衝突フィルタリングに対処しました。隣接して拘束された身体部位同士がコンタクトを生成しないようにして古典的な「爆発するラグドール」アーティファクトを防ぎつつ、キャラクターがワールドや他キャラクターと正しく相互作用できるようにしました。

== 理解しておくべきトレードオフ

構築したシステムは固定トポロジを仮定します。キャラクターあたり一つのラグドール、一組のボーン、一組の拘束です。予測可能なスケルタル構造を持つヒューマノイドにはよく働きます。四足動物、珍しい解剖学のクリーチャー、アセット作成時にスケルトンが分からないプロシージャル生成キャラクターにはあまり向きません。それらの場合、extras 駆動アプローチがより価値を持ちます—拘束を汎用的に定義し実行時に解決できます—が、拘束グラフのオーサリングと検証にはより洗練されたツールが必要です。

ハンドオフに使ったブレンドウェイト方式（固定時間にわたって 0 から 1 へ線形増加）は単純で信頼できますが、限界があります。滑らかに倒れるキャラクターには 150ms のブレンドが良く見えます。突然の大きなインパルスを受けたキャラクターには間違って見え得ます—その場合、ブレンドではなく即座にラグドールへスナップしたいかもしれません。構築したステートマシン設計はこれを支えます。入力インパルスが閾値を超えたら BLENDED 状態を完全にバイパスし、直接 RAGDOLL へ遷移できます。

構築した衝突フィルタリングシステムは、身体領域ごとに 1 ビットのフラットなビットマスクを使います。32 ビットマスクで 32 の異なる衝突レイヤが得られ、単一キャラクターには十分すぎますが、大規模群衆システムでは制約になります。Jolt Physics と PhysX はどちらも、数百から数千のキャラクターへフィルタリングシステムをスケールできる、より洗練されたフィルタリング機構（ブロードフェーズレイヤ、ナローフェーズコールバック）を提供します。

== 次に来ること

視覚レンダリングと物理システムの両方が同じシーングラフから動作し、正しく物理的に正確な挙動を生むようになった今、洗練の最終層を追加する強い立場にあります。**プロシージャルアニメーション** です。

プロシージャルアニメーションは、アーティストがオーサリングしたキーフレームではなく、アルゴリズムでランタイムにジョイントトランスフォームを生成または修正する規律です。ゲームで最も重要な応用が **インバースキネマティクス（IK）** です。足の位置を直接アニメートする代わりに、足を着地させたい場所から必要なジョイント角を計算します。これによりキャラクターは凹凸地形に正しく足を置き、アニメー夕が想定した正確な位置にないオブジェクトに手を伸ばし、缶詰のように見えるのではなく自然に見える形で物理イベントに応答できます。

第 5 章では IK システムを一から構築し、フォワードキネマティクスが足置き問題を解けない理由と IK が解ける理由の理論から始めます。単純な CCD（Cyclic Coordinate Descent）アルゴリズムと、より正確な FABRIK（Forward And Backward Reaching IK）アルゴリズムの両方を実装し、本シリーズ全体で構築してきた同じシーングラフとアニメーションブレンド基盤へ配線します。

== 検証: 確認すべき点

物理統合を検証するには:
1.  **キネマティック同期**: キャラクターが `ANIMATED` 状態のとき、デバッグ可視化を有効にします。衝突カプセルは動きに合わせて描画メッシュを完璧に追従すべきです。
2.  **ラグドールハンドオフ**: ラグドールをトリガーしたとき、キャラクターが新しいポーズへ「ポップ」または「スナップ」してはいけません。遷移は滑らかで、キャラクターの運動量は保たれるべきです。
3.  **自己衝突**: 移動中にキャラクターの肢同士が衝突しないことを確認します。肢が互いに「押し離して」いる場合、衝突フィルタリングのビットマスクを確認します。
