= 拘束とジョイント制限

== 拘束なしラグドールが見られない理由

前の節で作成した物理ボディを今すぐすべてダイナミックモードに切り替え、物理ソルバを走らせれば、ラグドールは得られます—しかしひどいものです。身体部位はそれぞれ独立に暴れます。腕は肩につながりません。膝は横・後ろ・前、そして完全な円を描くように曲がります。脊椎は混沌とした切り離された身体区画の塊に潰れます。キャラクターは倒れる人というより、さっきまで同じ空間を占めていたばらばらの物体の袋のように見えます。

問題は、物理ボディはデフォルトでは互いについて何も知らないことです。個別のシミュレートオブジェクトです。それらをつなぎ—膝を *膝* に、肩を *肩* にするもの—が **拘束** です。物理拘束はソルバが従わなければならないルールです。「この二つのボディは一定距離を保つ」「このボディはあのボディに対して回転できるが、この角度範囲内のみ」「このボディはあのボディに取り付けられ、あらゆる方向に自由に回転できる」といったことを言えます。

拘束がなければ、別々のレンガの山です。拘束があれば、スケルトンです。

== ヒューマノイドに必要な二つの拘束

ヒューマノイドラグドールでは、二つの拘束タイプが全身をカバーします。**ボールアンドソケット** 拘束と **ヒンジ** 拘束です。

ボールアンドソケット拘束（スフェリカル拘束と呼ばれることもあります）は、あらゆる方向への回転を許しつつ、ボディの分離を防ぎます。人間の肩や股関節の動きを考えてください。腕や脚は広い円錐の方向に回転できますが、ジョイント自体は並進しません—ボールはソケットに留まります。肩・股関節・首・足首に適した拘束です。物理エンジンの用語では、三つの並進自由度をすべて除去し、角度制限の下で三つの回転自由度を残します。

ヒンジ拘束は、ちょうど一つの軸周りの回転のみを許します。膝を考えてください。一平面で曲がり、横方向や軸方向には（少なくとも有意には）回転しません。ヒンジ拘束はこの単一軸回転を強制し、膝・肘、そしてラグドールに必要な詳細度次第では手首にも適切です。三つの並進自由度と三つの回転自由度のうち二つを除去し、一つだけを残します。

この区別は重要です。ラグドールが目にどう映るかを直接決めます。横に曲がる膝は即座に、本能的に間違って見えます。円錐状に動く肩は、正確な円錐角が生物学的に精密でなくても正しく見えます。拘束タイプを解剖学的機能に合わせることは、説得力のあるラグドールにするための最も重要なことです。

== 角度制限: 人間の可動域

拘束タイプだけでは不十分です。角度制限のないボールアンドソケットジョイントは腕が 360 度回転することを許します—腕が肩の後ろに真っ直ぐ向いたり、ヘリコプターのブレードのように回転したりします。実際の肩はボールアンドソケットですが、軟組織によって定義された有限の可動域があり、それを超えるとラグドールを笑い物にする「不可能な回転の腕」が生まれます。

角度制限は、各軸の有効な回転範囲を物理ソルバに伝えます。ヒンジ拘束では単純で、単一のヒンジ軸周りの最小・最大角度です。ボールアンドソケットではより複雑—通常 **スイング制限**（骨が中立軸からどれだけ逸れるかを記述する円錐半角）と **ツイスト制限**（骨が自身の主軸周りにどれだけ回転できるか）を指定します。

これらの制限を glTF extras に格納するのは、コライダー定義と同じパターンに従います。肩ボーンの extras は次のようになります。

[source,json]
----
{
  "physics": {
    "collider": "capsule",
    "radius": 0.06,
    "half_height": 0.14,
    "mass": 2.5,
    "constraint": {
      "type": "ball_socket",
      "swing_limit_deg": 80,
      "twist_limit_deg": 45,
      "parent_bone": "spine_upper"
    },
    "collision_group": "arm",
    "collision_mask": "world,props"
  }
}
----

膝ボーンの extras は次のとおりです。

[source,json]
----
{
  "physics": {
    "collider": "capsule",
    "radius": 0.055,
    "half_height": 0.20,
    "mass": 3.0,
    "constraint": {
      "type": "hinge",
      "hinge_axis": [0, 0, 1],
      "limit_min_deg": -140,
      "limit_max_deg": 0,
      "parent_bone": "thigh"
    },
    "collision_group": "leg",
    "collision_mask": "world,props"
  }
}
----

`parent_bone` フィールドは、この拘束がどのボディに接続するかを示します。拘束は親ボーンの物理ボディとこのボーンの物理ボディの間に作成されます。`hinge_axis` はボーンのローカル空間での回転軸を指定します。

== 拘束定義の解析

extras 解析を拡張して拘束定義も取得します。

[source,cpp]
----
struct ConstraintDef {
    enum class Type { NONE, BALL_SOCKET, HINGE };

    Type        type              = Type::NONE;
    float       swing_limit_deg   = 180.0f;   // Ball-socket: cone half-angle
    float       twist_limit_deg   = 180.0f;   // Ball-socket: twist range
    float       hinge_min_deg     = -180.0f;  // Hinge: minimum angle
    float       hinge_max_deg     =  180.0f;  // Hinge: maximum angle
    glm::vec3   hinge_axis        = {0,0,1};  // Hinge: rotation axis in local space
    std::string parent_bone;                  // Name of the parent node
};

ConstraintDef parse_constraint_def(const tinygltf::Value& phys_extras)
{
    ConstraintDef def;
    if (!phys_extras.Has("constraint")) return def;

    const auto& con = phys_extras.Get("constraint");

    const std::string type_str = con.Get("type").Get<std::string>();
    if      (type_str == "ball_socket") def.type = ConstraintDef::Type::BALL_SOCKET;
    else if (type_str == "hinge")       def.type = ConstraintDef::Type::HINGE;
    else                                return def;

    if (con.Has("swing_limit_deg")) def.swing_limit_deg = static_cast<float>(con.Get("swing_limit_deg").GetNumberAsDouble());
    if (con.Has("twist_limit_deg")) def.twist_limit_deg = static_cast<float>(con.Get("twist_limit_deg").GetNumberAsDouble());
    if (con.Has("limit_min_deg"))   def.hinge_min_deg   = static_cast<float>(con.Get("limit_min_deg").GetNumberAsDouble());
    if (con.Has("limit_max_deg"))   def.hinge_max_deg   = static_cast<float>(con.Get("limit_max_deg").GetNumberAsDouble());

    if (con.Has("hinge_axis")) {
        const auto& ax = con.Get("hinge_axis");
        def.hinge_axis = {
            static_cast<float>(ax.Get(0).GetNumberAsDouble()),
            static_cast<float>(ax.Get(1).GetNumberAsDouble()),
            static_cast<float>(ax.Get(2).GetNumberAsDouble())
        };
    }

    if (con.Has("parent_bone")) def.parent_bone = con.Get("parent_bone").Get<std::string>();

    return def;
}
----

== 拘束の作成

定義を解析したら、実際の物理拘束を作成します。拘束は二つのボディを参照する必要があるため、すべてのボーンボディ作成後に行います—拘束を物理エンジンに登録する前に両方が存在している必要があります。

[source,cpp]
----
void create_ragdoll_constraints(
    const std::vector<Node>& nodes,
    const std::map<std::string, uint32_t>& name_to_node,
    std::vector<BoneBody>& bodies,
    std::map<uint32_t, JPH::BodyID>& node_to_body,
    PhysicsWorld& physics_world)
{
    for (auto& bone_body : bodies) {
        const Node& node        = nodes[bone_body.node_index];
        const ConstraintDef& con = node.constraint_def;

        if (con.type == ConstraintDef::Type::NONE) continue;
        if (con.parent_bone.empty()) continue;

        // Look up the parent body by name
        auto parent_node_it = name_to_node.find(con.parent_bone);
        if (parent_node_it == name_to_node.end()) continue;

        auto parent_body_it = node_to_body.find(parent_node_it->second);
        if (parent_body_it == node_to_body.end()) continue;

        JPH::BodyID parent_body = parent_body_it->second;
        JPH::BodyID child_body  = bone_body.physics_body;

        if (con.type == ConstraintDef::Type::BALL_SOCKET) {
            physics_world.create_ball_socket_constraint(
                parent_body,
                child_body,
                glm::radians(con.swing_limit_deg),
                glm::radians(con.twist_limit_deg));

        } else if (con.type == ConstraintDef::Type::HINGE) {
            physics_world.create_hinge_constraint(
                parent_body,
                child_body,
                con.hinge_axis,
                glm::radians(con.hinge_min_deg),
                glm::radians(con.hinge_max_deg));
        }
    }
}
----

ここに重要な順序の考慮があります。拘束はすべてのボディが物理ワールドに追加された後に作成すべきです。一部のエンジン（例: Jolt Physics）では、それらを参照する拘束を安全に登録する前に、シミュレーションのブロードフェーズにすべてのボディが存在している必要があります。

== ヒューマノイドの参考制限値

出発点として、汎用ヒューマノイドラグドールの妥当な角度制限を示します。スポーツ医学文献のおおよその生体力学的値で、精密な調整を要さず説得力のあるラグドールにするためやや緩めています。

肩（ボールアンドソケット）は中立からおおよそ 90〜100 度スイングし、各方向に約 45 度ツイストできます。肘（ヒンジ）は 0 から約 145 度まで屈曲し、他の軸では有意な角度の自由度はありません。手首には横方向の偏位（約 ±20 度）と屈伸（屈曲約 70 度、伸展約 60 度）があります—ラグドール目的ではしばしば単一ヒンジに簡略化されます。

股関節（ボールアンドソケット）は前方に約 120 度、後方に 15 度、横に 45 度スイングし、軸方向ツイストは約 45 度です。膝（ヒンジ）は 0 から約 135 度まで屈曲します。足首の屈伸は合計おおよそ 50 度で、しばしばヒンジに簡略化されます。

脊椎が最も複雑です。現実には小さなジョイントの連続で、それぞれ数度の自由度を持ちます。ラグドールでは、脊椎を二つまたは三つのボールアンドソケット拘束（下部脊椎—中部脊椎、中部脊椎—上部脊椎）としてモデル化するのが一般的で、それぞれスイング制限約 15〜20 度—脊椎が曲がり潰れるのに十分だが、グロテスクな折り畳みは許さない程度です。

これらの値は出発点であり、仕様ではありません。特定のアートスタイルとゲームプレイ感に合わせて調整が必要です。様式化されたアクションゲームでは誇張された可動域が望まれるかもしれません。現実的なシミュレーションでは生体力学的測定に基づくより厳しい制限が必要かもしれません。正解は、プレイして正しく感じるものです。
