= プロシージャルアニメーションとインバースキネマティクス

== キーフレームアニメーションの限界

これまでに構築してきたもの—シーングラフ、コンピュートスキニングパイプライン、物理ラグドールシステム—はすべて、アーティストが Blender で作成したアニメーションの再生とブレンドに奉仕してきました。キーフレームアニメーションは強力なツールであり、キャラクター動作の大部分ではまさに正しいアプローチです。アーティストはウォークサイクル、戦闘の一撃、アイドルの呼吸アニメーションを何時間も磨き上げられ、その投資は結果に表れます。

しかしキーフレームアニメーションには根本的な限界があります。特定の環境向けにオーサリングされていることです。ウォークサイクルは平らな地面を前提とします。手を伸ばすアニメーションは、手が届く対象がアニメー夕が置いた場所に正確にあることを前提とします。死亡アニメーションは、キャラクターが開けた特徴のない床に倒れることを前提とします。現実の世界—あるいは現実のゲーム世界—は、そのどれでもありません。

岩だらけの丘を渡るキャラクターは、各足が遭遇する表面に正確に着地する必要があり、地面の半メートル上に浮いたり斜面に食い込んだりしてはいけません。動的に置かれたピックアップに手を伸ばすキャラクターは、毎フレーム変わる実際のオブジェクト位置に手が到達する必要があります。遮蔽物の後ろに飛び込む兵士は、身体が事前オーサリングされたしゃがみアニメーションにコミットしていても、頭が敵を追う必要があります。全力で鋭いコーナーを回るキャラクターは、レール上をスケートするように直立したままでなく、カーブにリーンすべきです。

これらの問題は、キーフレームアニメーションを増やすだけでは解けません。地形構成・オブジェクト位置・動的シナリオは無限にあります。解決策は **プロシージャルアニメーション**です。アーティストがオーサリングしたものではなく、現在のワールド状態に応じてエンジンが計算する、ジョイントトランスフォームのリアルタイム数学的計算です。

== プロシージャルアニメーションとは何か、何かではないか

業界ではこの用語が緩く使われるため、ここで言うプロシージャルアニメーションを正確にしておく価値があります。本章では、シーンの現在状態に応答するアルゴリズムによる、スケルトンのジョイントトランスフォームのランタイム修正を意味します。プロシージャルコンテンツ生成、パーティクルシステム、ジオメトリを生成するものの話ではありません。ジョイント回転を計算する話です。

プロシージャルアニメーションは通常、キーフレームアニメーションの *代わり* ではなく *上に* レイヤーされます。ベースポーズはアニメー夕のクリップから来て、プロシージャルレイヤーが環境に応答するよう特定ジョイントを調整します。これは加算的なプロセスです。キーフレームシステムが生物学的に妥当なベース動作を提供し、プロシージャルレイヤーがアーティストが予測できなかった小さく的を絞った補正を行います。

位置補正の中心技法が **インバースキネマティクス**（IK）です。標準のキーフレームアニメーションは *フォワードキネマティック* に働きます。ジョイント回転を設定すると、エンドエフェクタ（手、足）は数学が置く場所に行きます。IK は逆です。エンドエフェクタが *いなければならない* 場所を指定し、アルゴリズムがそこに置くために必要なジョイント回転を計算します。これは自明な逆変換ではありません。複数ジョイントのチェーンでは通常、有効解が無限にあり、アルゴリズムは最も自然に見えるものを選ばなければなりません。

二つの IK アルゴリズムを詳しく扱います。**Cyclic Coordinate Descent（CCD）** と **FABRIK**（Forward And Backward Reaching IK）です。どちらもゲームエンジンで広く使われる実用的なリアルタイムアルゴリズムです。最も一般的な二つの応用—凹凸地形上の足置きと、頭部追従のルックアットコントローラ—も扱います。最後に、物理シミュレーションの速度データからキャラクター身体の傾きを導く、物理駆動のプロシージャルリーンを実装します。

== 本章がシリーズの他部分とどう適合するか

本章は、ワールド行列が正しく伝播する動作するシーングラフ（第 2 章）があり、アニメーションブレンドシステム（第 3 章）を理解していることを前提とします。実装する IK アルゴリズムは、シーングラフからジョイントのワールド行列を読み、補正したジョイントのローカルトランスフォームを書き戻し、Dirty Flag 伝播システムを起動してすべての子孫ノードを更新します。

第 4 章のラグドールシステムも、特に足置きの節で関連します。キャラクターが凹凸地形に立っているとき、足 IK システムは物理システムと戦ってはならず、協力しなければなりません。制御の階層と、IK 補正と物理フィードバックが矛盾ではなく強化し合うようにする方法を議論します。
