= プロシージャルアニメーション: まとめと次のステップ

== 構築したもの

本章を通じて、アーティストがオーサリングしたアニメーションを忠実に再生するキャラクターから、リアルタイムで環境に適応するキャラクターへと移行しました。この二つの間の距離は—コード行数で測れるとしても—プレイヤーが知覚するものの観点では遥かに重要です。岩の上を正しく踏み、面白いものを見ようと頭を回し、コーナーに自然にリーンするキャラクターは、純粋にキーフレームされたキャラクターにはない形で、ゲーム世界に *存在* していると感じられます。

実装したものを振り返りましょう。まず二つの基本 IK アルゴリズム、CCD と FABRIK から始めました。CCD はジョイントごとに働き、エンドエフェクタをターゲットへ段階的に近づけるよう各ジョイントを回転させ、短いチェーンと厳しいジョイント制限では速く収束します。FABRIK は回転ではなく位置で考え、チェーンをより均等に再配分し、長いチェーンや腕全体のリーチではより自然に見える傾向があります。どちらもシーングラフの Dirty Flag システムと統合されます。ワールド行列を読み、補正を計算し、ローカル回転を書き戻します。

次に CCD を足置き—ヒューマノイドキャラクターで最も普遍的に効果のある IK 応用—に適用しました。足置きでは基本 IK を超える三つの問題を解く必要がありました。身体高さ調整（凹凸地形に合わせて骨盤を下げる）、足の回転（単に正しい高さに届くだけでなく地形法線に揃える）、時間的平滑化（生の IK ターゲットはスナップし、滑らかなターゲットは接地感がある）です。足が接地相かスイング相かに基づいて IK 補正をゲートするプラントウェイトシステムも扱い、IK がベースアニメーションを上書きするのではなく強化するようにしました。

ルックアットコントローラは、純粋に回転のプロシージャル技法を導入しました。頭の現在の前方向とターゲット方向の間の角度を計算し、その角度を脊椎から頭までのジョイントチェーンに分配することで、自然な頭と首の追従を実現します。平滑化されたターゲットシステムにより、追従は適切な慣性感で応答します—警戒中のキャラクターなら速く、リラックスしていれば遅く。

最後に、物理駆動リーンシステムは、キャラクターの移動物理を身体姿勢に接続します。キャラクターの加速度から実効重力ベクトルを計算し、脊椎ジョイント全体に適用するリーン角を導きます。結果は、アーティストがオーサリングしたリーンアニメーションを一つも使わずに、速度・重さ・進行方向を身体で伝えるキャラクターです。

== 覚えておくべきトレードオフ

本章で構築したすべてのシステムにはトレードオフがあり、明示的に名付ける価値があります。

CCD は FABRIK より正しく実装するのが簡単ですが、長いチェーンや極端なリーチでは偏ったポーズになり得ます。FABRIK は視覚的によりバランスが取れますが、解いた位置をジョイント回転へ戻す追加ステップが必要で、座標系まわりの複雑さが増します。ヒューマノイドの足チェーンでは通常 CCD が適切なデフォルトで、脊椎や腕全体のリーチでは FABRIK を好みます。

レイキャストと骨盤調整付きの足置きは、足置きなしより大幅に良く見えますが、地形変化とキャラクター適応の間にレイテンシ（平滑化の時定数）を導入します。積極的な平滑化はレイテンシを減らしますが、見えるスナップの可能性を増やします。保守的な平滑化はより自然に感じられますが、棚から降りた直後に足が一瞬浮くことを意味します。対象地形でプロファイルし、それに応じて調整してください。

直接回転に基づくルックアットコントローラは、中程度の角度範囲—レストポーズからおそらく 60〜70 度まで—でよく働きます。それを超えると、複数ジョイントへの回転分配が助けになりますが、自然に見える限界に当たります。後ろを見る必要があるキャラクターでは通常、ルックアット角を 180 度まで伸ばそうとするのではなく、身体を物理的に回す「振り返る」アニメーションが必要です。

物理駆動リーンは純粋にヒューリスティックです。物理的に正しいリーン角（`atan2(acceleration, gravity)` から導く）は、同時に他の誇張された動作をしているゲームキャラクターには視覚的に極端すぎることがよくあります。`lean_strength` チューニングパラメータがあるのは、物理的正確さと視覚的妥当性が同じではないからです。式より目を信じてください。

== 次に来ること

第 6 章では **モーフターゲットと表情アニメーション** に注目します。これまで扱ってきたのはすべてスケルタルアニメーション—身体を変形する階層ジョイントシステム—でした。モーフターゲットは根本的に異なる表現です。ジョイントを回転する代わりに、表情・リップシンクポーズ・その他の連続的な形状変化の間をブレンドする頂点ごとの変位ベクトルを格納します。

モーフターゲットは別の課題群を提示します。頂点数は多く—高品質な顔メッシュは数万頂点になり得ます—カットシーンで重い表情アニメーションがある場合、アクティブなモーフターゲット数も相当になり得ます。Vulkan 1.4 のディスクリプタインデクシングを使い、ドローごとのディスクリプタ更新なしにモーフターゲット変位バッファを扱い、第 3 章で構築したスキニングコンピュートパイプラインにモーフ結果を統合します。

== 検証: 確認すべき点

IK 実装を検証するには:
1.  **収束**: CCD または FABRIK で、反復ごとの誤差距離を監視します。単調に減少すべきです。振動したり増加したりする場合、ジョイント回転ロジック（特に親からローカル空間への変換）が誤っている可能性が高いです。
2.  **安定性**: 届かないターゲットで IK ソルバをテストします。チェーンはジッターや激しいスピンなしに、ターゲットへ向かって完全に伸びるべきです。
3.  **骨盤調整**: 急な斜面に立つとき、キャラクターの腰が正しく下がることを確認します。下がらないと脚が過度に伸びたり「硬い」ように見えます。
