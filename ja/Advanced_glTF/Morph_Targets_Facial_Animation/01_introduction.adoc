= モーフターゲットと表情アニメーション

== 別種の変形

これまでに扱ってきた技法—シーングラフ、コンピュートスキニング、ラグドール、IK—はすべて同じ基盤の上にあります。ジョイントのスケルトン、それぞれがトランスフォームを持ち、メッシュの変形を駆動します。スケルタルアニメーションは身体・肢・おおよそ回転的な変形をする任意のメッシュ領域に対して強力で効率的です。しかしアキレス腱があり、それが顔です。

人間の顔は、肢とは根本的に異なる仕方で変形します。口周りの皮膚は膝が曲がるように回転するのではなく、部分的に相反する方向に引っ張る何十もの独立した筋肉に駆動され、複雑に滑り、膨らみ、圧縮し、伸びます。これをすべてジョイントで捉えるには非現実的な数が必要で（映画制作のプロ表情リグは顔だけで数百ジョイントになり得ます）、それでもポーズ間の補間はアーティファクトを生み得ます。ジョイント回転の球面線形補間は、皮膚変位の線形ブレンドと同じではないからです。

表情アニメーションに対する業界の解が **モーフターゲット** であり、使うソフトウェアによって **ブレンドシェイプ** または **シェイプキー** とも呼ばれます。考え方は単純です。ジョイント回転をアニメートする代わりに、異なる表情を表す頂点位置の完全なセットを複数格納します。「笑顔」モーフターゲットは、笑った構成でのすべての顔頂点の位置を格納します。「眉上げ」モーフターゲットは、眉を上げた構成での位置を格納します。ランタイムでは、各ターゲットの寄与にウェイトを付け、ベースメッシュと一つ以上のモーフターゲットの間を線形にブレンドします。笑顔ウェイトが 0.7 なら、すべての頂点がレスト位置から笑顔位置へ 70% 動きます。笑顔（ウェイト 0.5）と眉上げ（ウェイト 0.8）が同時にアクティブなら、変位は加算され、頂点は合成位置へ動きます。

これが **Linear Blend Shapes** モデルであり、glTF が `WEIGHTS` アニメーションチャネルと呼ぶものです。

== glTF がモーフターゲットに格納するもの

glTF 仕様はモーフターゲットを、ターゲットごとのアクセサのリストとして格納し、それぞれが頂点ごとの変位ベクトルを含みます。具体的には、`N` 頂点と `K` 個のモーフターゲットを持つメッシュプリミティブについて、glTF ファイルは次を含みます。

- ベース頂点バッファ: 位置・法線・接線・テクスチャ座標を持つ N エントリ。
- K 個の追加の「スパース」または密なアクセサ配列。それぞれが N 個の位置変位ベクトル（各頂点について、モーフターゲット位置とベース位置の差）を含む。
- 任意で、表面曲率を変えるモーフターゲット（頬を膨らませるなど）向けに、K 個の追加の法線変位配列と K 個の追加の接線変位配列。

アニメーションシステムはモーフターゲットごとに一つ、範囲 [0, 1] の `K` 個のウェイトを駆動します。最終頂点位置は次です。

`final_position = base_position + sum(weight[i] * displacement[i])`（すべての K ターゲットについて）

これは線形ブレンドであり、スケルタル線形ブレンドスキニングと同じ美点があります。GPU で自明に並列化できることです。すべての頂点は独立で、すべての変位は単なるベクトル加算とスカラー乗算です。

== メモリの課題

モーフターゲットには深刻な問題が一つあります。メモリです。50,000 頂点の顔メッシュと 50 個のモーフターゲット（完全な FACS—Facial Action Coding System—カバレッジを持つフォトリアルキャラクターでは妥当な数）がある場合、50 × 50,000 = 2,500,000 個の変位ベクトルがあります。3D float ベクトルあたり 12 バイトで、単一キャラクターの顔の変位データだけで 30 メガバイトです。法線変位なら倍の 60 MB です。同時に話すキャラクターが複数いるゲームでは、急速に積み上がります。

私たちが使うアプローチは **バインドレス** モーフターゲットバッファと **スパースアクティベーション** の組み合わせです。すべてのキャラクターのすべてのモーフターゲットデータを無条件に GPU 常駐バッファへ載せるのではなく、*現在アクティブな* モーフターゲットのデータだけを昇格し、Vulkan 1.4 のディスクリプタインデクシングで単一シェーダーからいずれにもアクセスします。これはマテリアル向けバインドレステクスチャアトラスと同じ設計思想です。リソースごとにディスクリプタをバインドする代わりに、大きなディスクリプタ配列を一度バインドし、プッシュ定数またはユニフォーム値でインデックスします。

== 本章の内容

モーフターゲットパイプライン全体を三段階で実装します。第一に、glTF からモーフターゲット変位データをパースしてアップロードします。これは glTF のスパースアクセサ形式の特別な扱いが必要で、実際に変わる頂点だけを格納します（ベースと異なるのがまぶた頂点だけの「まばたき」のようなモーフターゲットでは大きな最適化です）。第二に、バインドレス Vulkan ディスクリプタ基盤を構築します。スキニングコンピュートシェーダーが自由にインデックスできる大きなバッファ配列です。第三に、第 3 章のコンピュートスキニングシェーダーを拡張し、スケルタルスキニングの前にモーフ変位を適用して、表情と身体ポーズの両方を同時に反映する最終頂点バッファを生成します。

制作におけるモーフターゲットとスケルタルリグの関係も簡潔に議論します。プロのキャラクターは通常両方を使います。頭と顎の粗いジョイントリグ（口を開けるような大規模変形を扱う）と、細かい表面詳細（リップシンク、鼻翼、眉しわ）向けの密なモーフターゲットレイヤーです。この二層がどう協力するかを理解すると、キャラクターパイプラインで正しいトレードオフを選べます。
