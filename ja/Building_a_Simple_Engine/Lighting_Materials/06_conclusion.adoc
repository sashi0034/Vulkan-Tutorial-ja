= まとめ

この章では、3D レンダリングにおけるライティングとマテリアルの基礎を探り、metallic-roughness ワークフローを用いた Physically Based Rendering（PBR）を紹介しました。PBR の背後にある理論を扱い、glTF モデルで使えるシェーダーを実装し、Vulkan Ray Query を使った高品質シャドウを追加しました。また、マテリアルプロパティをシェーダーへ効率的に渡すプッシュ定数の使い方も学びました。

== 学んだこと

この章では、Vulkan エンジンで物理ベースレンダリングを実装するために必要な本質的概念を一通り見てきました。metallic‑roughness PBR ワークフローを紹介し、glTF マテリアルプロパティをシェーダー入力に対応付け、ディスクリプタの頻繁な更新なしにドローごとのマテリアルパラメータを駆動するプッシュ定数を使いました。BRDF の各部がどう連携してエネルギーを保存し妥当なライティングを生み出すか、そしてモデルがエンドツーエンドで正しくレンダリングされるようシェーダーを vk::raii ベースのパイプラインにどう組み込むかを見ました。最後に、リアリズム向上のためハードウェア加速のレイトレースシャドウを統合しました。

== 理解を定着させる: この PBR パイプラインのメンタルモデル

高レベルでは、フレームを、物理的入力を表示可能なピクセルへ変換するリニアライト計算として考えてください。

- リニア空間の入力: 物理っぽい単位の強度を持つライト、マテリアルからの baseColor/metallic/roughness、normal/AO/emissive マップ。ヘッドルームを失わないようリニア HDR で作業する。
- BRDF の役割: D はハイライトの形を決め（roughness がローブ幅を制御）、G はマイクロファセット上のマスキング／自己影を強制し、F は掠め角で反射率を上げ F0 経由でマテリアル型に反射率を結びつける。エネルギー保存は鏡面（kS）と拡散（kD）を結び、入射量を超えないようにする。
- マテリアルつまみとしてのレバー:
  * Roughness: 鏡面ローブを広げ／狭め、G 経由でピーク強度も下げる。
  * Metallic: 誘電体挙動（色付き拡散 + 中立鏡面）と導体挙動（色付き鏡面、拡散なし）の間をクロスフェードする。
  * Base color: 誘電体では拡散アルベド、金属では色付き鏡面。
- Normal/AO/emissive の文脈: 法線マップは局所向きを摂動してディテールを追加し、AO は平坦な溝を避けるため間接／環境光を減衰させ、emissive はライト非依存の発光を追加する。
- 出力ステージング: 環境／間接と直接ライティングを合計したあと、トーンマッパー（例: Reinhard/ACES）で HDR を圧縮し、その後で初めてディスプレイに合わせてガンマを適用する。ガンマはちょうど一度だけ（シェーダーの pow か sRGB フレームバッファのどちらか）。

結果がおかしく見えるときの素早い推論ループ:

1. 空間と順序を確認: リニアライティング ➜ トーンマップ ➜ ガンマ。二重ガンマしていないか確認する。
2. BRDF を探る: すべてがプラスチックっぽい？ Roughness が低すぎるか、metallic で kD が減っていない。薄く泥んだハイライト？ Roughness が高すぎるか露出が低すぎる。
3. 法線／TBN を検証: 反転した緑チャネルや誤った接線の左右勝手は、奇妙なシェーディングや継ぎ目を起こす。
4. 露出／トーンマップを校正: 白が厳しくクリップするなら、露出制御を追加し、より滑らかなロールオフのため ACES/Hable に切り替える。
5. AO と emissive を適切に使う: AO は環境／IBL に影響すべきで直接鏡面には影響しない。emissive は加算的でライト非依存。

このメンタルモデルは、任意の入力の変更がパイプラインを通って画面にどう現れるかを予測する助けになり、「リストに従う」だけでなく「理解する」ことの核心です。

== 改善の余地

PBR パスは堅実なベースラインです。最も効果の大きいアップグレードはイメージベースドライティング（環境／間接用の環境マップ）といくつかのマテリアル拡張（例: clear coat や anisotropy）です。パフォーマンス面では、ライト数が増えたら clustered forward やディファードパスを検討してください。HDR チェーンを構築するなら、ブルームとよりフィルム的なトーンマッパー（ACES/Hable）がプレゼンテーションを整えます。

== 次のステップ

1 本の糸を選び深く掘り下げてください。ライティングなら、時間の許す範囲で GI/AO/ボリュメトリクスを探る。マテリアルなら、glTF（とカスタム）パラメータをシェーダーにきれいに対応付けるデータ駆動システムを設計する。ビジュアルなら、ポストエフェクト（フォグ、ブルーム、DoF）を試作する。パフォーマンスなら、まずプロファイルし、ホットスポット——特にモバイル——を最適化する。

ライティングは多くのアプローチと技法を持つ複雑なトピックであることを忘れないでください。この章で扱った実装はほんの始まりです。エンジン開発を続けるにつれ、プロジェクト固有のニーズに合わせてライティングシステムを洗練・拡張したくなるでしょう。
