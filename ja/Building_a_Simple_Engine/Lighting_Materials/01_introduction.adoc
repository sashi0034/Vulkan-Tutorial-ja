= ライティングとマテリアル入門

この章では、Physically Based Rendering（PBR）に焦点を当て、3D レンダリングにおけるライティングとマテリアルの基礎を探ります。ライティングは、リアルで視覚的に魅力的な 3D シーンを作るうえで重要な側面です。適切なライティングがなければ、最も詳細なモデルでさえ平坦で生気のない見た目になります。

image:../../../images/bistro.png[Bistro scene with PBR, width=600, alt=Rendering the Bistro scene at night with PBR pass]


[NOTE]
====
*PBR 参照について*: 本チュートリアル全体で、この章に到達する前に PBR（Physically Based Rendering）への言及に出会うことがあります。PBR は、物理原理に基づいて光が表面とどう相互作用するかをシミュレートするモダンなレンダリング手法です。続く節で PBR を詳しく扱うので、まだこれらの概念に慣れていなくても心配いりません。
====

この章は、光が異なるマテリアルと物理的に正確な方法でどう相互作用するかを理解するための基盤となります。ここで学ぶ概念は、PBR マテリアルを持つ glTF モデルのレンダリングにこの知識を使う Loading_Models の章を含む、後の章で適用されます。

エンジン実装全体を通じて、vk::raii ダイナミックレンダリングと {cpp}20 モジュールを使います。vk::raii 名前空間は Vulkan オブジェクトに対する Resource Acquisition Is Initialization（RAII）ラッパーを提供し、リソース管理を助け、コードをより簡潔にします。ダイナミックレンダリングは明示的なレンダーパスとフレームバッファを不要にし、レンダリング処理を単純化します。{cpp}20 モジュールは従来のヘッダファイルと比べ、コード構成、コンパイル時間、カプセル化を改善します。

== なぜライティングが重要か

コンピュータグラフィックスにおけるライティングは、いくつかの重要な目的を果たします。

1. *視覚的リアリズム*: 適切なライティングは、3D オブジェクトをよりリアルに見せる影、ハイライト、グラデーションを作ります。
2. *空間理解*: ライティングは、視聴者がシーン内のオブジェクト間の空間関係を理解する助けになります。
3. *ムードと雰囲気*: 異なるライティング設定は、シーンのムードと雰囲気を劇的に変えられます。
4. *焦点と注意*: ライティングは、シーンの重要な要素に注意を引くために使えます。

== Physically Based Rendering（PBR）

=== PBR 入門

Physically Based Rendering（PBR）は、過去 10 年のリアルタイムグラフィックスにおける最も重要な進歩の一つです。場当たり的なシェーディングモデルを使った従来のレンダリング手法と異なり、PBR は物理学の原理に基づき、現実世界で光が表面とどう相互作用するかをシミュレートすることを目指します。

=== リアルタイムレンダリングの進化

PBR を正しく評価するには、リアルタイムレンダリングがどう進化してきたかを理解すると役立ちます。

1. *固定機能パイプライン（1990 年代）*: 初期の 3D ハードウェアは、限定的なマテリアルプロパティを持つ Gouraud や Phong シェーディングのような固定ライティングモデルを使いました。

2. *プログラマブルシェーダー（2000 年代）*: シェーダープログラミングの導入により、開発者はカスタムライティングモデルを実装できましたが、これらはしばしば異なるライティング条件下で一貫性を欠きました。

3. *Physically Based Rendering（2010 年代）*: レンダリングを物理原理に基づかせることで、PBR は異なる環境でも一貫した、よりリアルな結果を提供します。

PBR の主な利点には次があります。

* *リアリズム*: マテリアルはあらゆるライティング条件下で正しく見える
* *一貫性*: アーティストはすべての環境で機能するマテリアルを作成できる
* *直感性*: マテリアルパラメータに物理的意味があり、理解しやすい
* *効率*: モダンな PBR 実装はリアルタイム性能向けに最適化されている

=== PBR の中核原則

PBR は、以前のレンダリング手法と区別するいくつかの主要原則の上に築かれています。

==== エネルギー保存

現実世界では、表面は受け取った光より多くの光を反射できません。このエネルギー保存の原則は PBR の基本です。

* 拡散反射と鏡面反射の合計は 1.0 を超えてはならない
* 表面がより金属的になるにつれ、拡散反射は減る
* 表面がより粗くなるにつれ、鏡面ハイライトは大きくなるが強度は下がる

==== マイクロファセット理論

PBR は表面の粗さをモデル化するためにマイクロファセット理論を使います。この理論は、表面が向きの異なる無数の完全反射マイクロファセットから成ると仮定します。

* 滑らかな表面はほとんど揃ったマイクロファセットを持ち、鋭い反射を作る
* 粗い表面はランダムな向きのマイクロファセットを持ち、光を散乱させてぼやけた反射を作る
* これらのマイクロファセットの分布は roughness パラメータで制御される

==== フレネル効果

フレネル効果は、反射率が視線角度によってどう変わるかを記述します。

* すべての表面は、視線方向が表面にほぼ平行になる掠め角でより反射的になる
* この効果は滑らかな表面でより目立つ
* 法線入射（F0、光が表面に垂直に当たるとき）での基本反射率は、マテリアルの屈折率で決まる
* 金属では、F0 は色付き（金属の特性に基づく）
* 非金属（誘電体）では、F0 は通常およそ 0.04（4%）

==== Metallic-Roughness ワークフロー

glTF と多くのモダンエンジンの PBR 実装は、次の主要パラメータでマテリアルを定義する metallic-roughness ワークフローを使います。

* *Base Color*: 表面のアルベドまたは拡散色
* *Metallic*: 表面がどれだけ「金属らしい」か（0.0 = 非金属、1.0 = 金属）
* *Roughness*: 表面がどれだけ滑らか／粗いか（0.0 = 鏡面、1.0 = 粗い）

このワークフローはアーティストにとって直感的で、リアルタイムレンダリングに効率的です。

=== PBR における BRDF

Bidirectional Reflectance Distribution Function（BRDF）は PBR の中核です。次を考慮して、光が表面からどう反射されるかを記述します。

* 入射光方向
* 出射視線方向
* 表面法線
* マテリアルプロパティ

PBR では、BRDF は通常 2 つのコンポーネントに分割されます。

* *拡散 BRDF*: 表面に侵入し、散乱し、出ていく光を扱う
* *鏡面 BRDF*: 表面から直接反射する光を扱う

==== 拡散 BRDF

最も単純な拡散 BRDF は Lambertian モデルです。

[source]
----
f_diffuse = albedo / π
----

ここで:

* albedo は表面の基本色
* π は正規化係数

特に粗い表面でリアリズムを高めるため、Disney の拡散や Oren-Nayar のようなより高度なモデルも使えます。

==== 鏡面 BRDF

鏡面コンポーネントでは、PBR は通常マイクロファセット BRDF を使います。

[source]
----
f_specular = D * F * G / (4 * (n·ωo) * (n·ωi))
----

ここで:

* D は Normal Distribution Function（NDF）
* F はフレネル項
* G はジオメトリ項
* n は表面法線
* ωo は出射（視線）方向
* ωi は入射（光）方向

一般的な実装には次があります。

* *D*: GGX（Trowbridge-Reitz）分布
* *F*: Schlick の近似
* *G*: Smith シャドウイング・マスキング関数

== コンピュータグラフィックスにおけるマテリアル

マテリアルは、表面が光とどう相互作用するかを定義します。異なるマテリアルは光を異なる方法で反射・吸収・透過します。リアルなレンダリングを作るうえで、マテリアルの理解は重要です。

=== マテリアルプロパティ

コンピュータグラフィックスでは、マテリアルはさまざまなプロパティで定義されます。

* *Base Color/Albedo*: 拡散照明下での表面の色
* *Metalness*: 表面がどれだけ金属的か（鏡面反射と拡散吸収に影響）
* *Roughness/Smoothness*: 表面がどれだけ粗いか／滑らかか（鏡面ハイライトの大きさと鋭さに影響）
* *Normal Map*: 幾何複雑さを増やさずに表面ディテールを追加
* *Ambient Occlusion*: 表面点が受け取る環境光の量を近似
* *Emissive*: 表面の一部を発光させる
* *Opacity/Transparency*: マテリアルの透明度を制御
* *Refraction*: 光がマテリアルを通過するときの屈折を制御

=== 一般的なマテリアル型

異なる種類のマテリアルには異なる特性があります。

* *金属*: 高い鏡面反射、色付き鏡面、拡散反射なし
* *誘電体（非金属）*: 低い鏡面反射、白い鏡面、強い拡散反射
* *半透明マテリアル*: 光を通過させ内部で散乱させる（例: 肌、ワックス、大理石）
* *透明マテリアル*: 散乱を最小限にして光を通過させる（例: ガラス、水）
* *異方性マテリアル*: 方向によって光の反射が異なる（例: ブラシドメタル、髪）

=== マテリアルプロパティ用のプッシュ定数

実装では、マテリアルプロパティをシェーダーへ効率的に渡すためにプッシュ定数を使います。

プッシュ定数は、ディスクリプタセットを作成・管理せずに少量のデータをシェーダーへ送る方法です。マテリアルプロパティのような頻繁に変わるデータに最適です。
