= Opacity Micromaps: 現実世界のための賢いシャドウ
:imagesdir: ../../../images

ここまで Simple Game Engine チュートリアルを完了してきたあなたは、本当に美しいものを作り上げています。レイトレースによるシャドウは床や壁に物理的な精度で落ちます。固体オブジェクトははっきりしたシルエットを落とし、ソフトシャドウはその影をさらに魅力的に見せます。光はジオメトリの位置を知り、それに応じて暗闇が続きます。シンプルな建築シーン — 箱型の部屋、テーブル、数脚の椅子 — では、結果はほぼ完璧で、特別な苦労もなく得られます。

しかし現実世界は箱と椅子だけではありません。外に出れば、単純なレイと三角形の交差ではうまく扱えない世界にすぐに遭遇します。木はシャープな矩形の影を落としません。錬鉄のフェンスは光を一様に遮りません。カーテンは光の隙間を生みながら揺らめきます。髪は一枚の表面ではなく、半透明な何百万本もの髪の毛です。自然の視覚的な豊かさ — そしてそれを模倣する人間の世界の豊かさ — のほとんどは、幾何学的には単純でも光学的には複雑なオブジェクトから生まれます。

リアルタイムレンダリングでは、この複雑さを **アルファテスト**（アルファマスキングとも呼ばれる）という技法で扱います。木の葉一枚一枚を実ジオメトリでモデリングする代わりに、広く平坦な四角形を取り、そこに葉のテクスチャを描きます。テクスチャは **アルファチャンネル** — 各テクセルがどれだけ不透明かを 0 から 1 の値で示す — を持ちます。葉の縁ではアルファは 0、つまり透明で、このピクセルは無視します。葉の本体ではアルファは 1、つまり不透明で、このピクセルは存在します。この工夫により、一枚の平坦な三角形が説得力のある複雑なシルエットを表現できます。

ラスタライズでは、アルファテストは安価です。GPU は各スクリーンピクセルでアルファを評価し、透明なものを単に破棄するだけです。シャドウマップでも同様の近道が使えます。デプスのみのアルファテスト付きシャドウパスはよく理解され、よく最適化されています。

レイトレーシングはこの方程式を変えます。レイトレースシャドウシステムでは、シャドウレイはテクスチャをスクリーンに投影するのではなく、三次元空間でジオメトリと交差します。そのレイが葉の三角形に当たったとき、「何か当たったか？」と聞くだけでは足りません。「この正確な点で、実際に固体であるものに当たったか？」と聞かなければなりません。その問いに答えるには、交差点の正確な位置でアルファテクスチャを参照する必要があります。しかも、レイが通過しうるすべての三角形についてこれを行わなければなりません。何千もの葉三角形を持つ一本の木、多くの木があるシーン、高解像度でピクセルあたり複数のシャドウレイ — 計算はすぐに膨大になります。

これは仮説上の問題ではありません。アルファテストされた植生は、リアルタイムレイトレースシーンにおけるパフォーマンスボトルネックの最も一般的な原因の一つです。プロファイリングツールは一貫して、any-hit shader の呼び出し — このアルファ参照を担当するシェーダーステージ — が、自然が画面に現れるたびに GPU 時間を支配することを示します。

**Opacity Micromaps（OMM）** は、`VK_KHR_opacity_micromap` 拡張を通じて提供される、この問題に対する Vulkan の答えです。中核のアイデアは優雅で、一度理解すればほぼ自明です。実行時に GPU に各三角形の各部分の不透明度を発見させる代わりに、最初のフレームが描画される前に、その情報を GPU のアクセラレーション構造へ事前ベイクします。各三角形を小さな **マイクロ三角形** のグリッドに分割し、それぞれを恒久的に不透明、恒久的に透明、または境界付近の不明として分類します。ハードウェアのトラバーサルユニットはこの分類を直接読み取り、シェーダーコードを一切実行せずに、レイが遮られるかどうかを即座に判断します。

その結果、高価な any-hit shader の発火は劇的に減ります — 確実性が本当に曖昧な縁ピクセルの細いリングに対してだけです。葉の表面の圧倒的大部分については、答えはあらかじめ分かっています。ハードウェアはその知識を単一サイクルで適用します。

本コースでは、なぜこの問題が存在するのか、micromap がどう解決するのか、そしてシンプルエンジンのソースコードでの実装がどう見えるのかについて、完全な概念的理解を与えます。まずシャドウそのものの視覚言語 — なぜあのように見えるのか、なぜ植生が高速シャドウアルゴリズムの前提を壊すのか — から始めます。次に GPU のレイトラバーサルハードウェアに降りて、パフォーマンスコストが正確にどこから来るかを理解します。そこから micromap の概念をゼロから組み立て、分割モデル、2-state と 4-state の分類フォーマット、micromap データがアクセラレーション構造にどう結び付くかを歩いていきます。最後にエンジンの `OpacityMicromapBuilder` 実装を巡り、この最適化がいつ元を取るか — そしていつ取らないか — を議論します。

GPU ハードウェア内部の事前知識は不要ですが、レイトレーシングの基礎には慣れているべきです。BVH とは何か、シャドウレイの仕組み、アクセラレーション構造が何をするか。これらの概念がしっかりしていれば、始める準備はできています。
