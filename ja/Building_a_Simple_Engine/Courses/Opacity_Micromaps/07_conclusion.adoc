= まとめ: アートとハードウェアをつなぐ

== 逆向きの旅

このコースは美しいシャドウと隠れた問題から始まりました。シャドウは正しく見えました。フレームはレンダリングされていました。しかし視覚的正しさの下で、GPU は膨大な量の冗長な発見に従事していました — 毎秒六十回、すべての葉のどの部分が固体でどれが空気かを再学習していました。その冗長性を GPU のトラバーサルハードウェアの根まで辿り、なぜ any-hit shader が必要か、なぜ高価か、なぜダイバージェンスとキャッシュミスがコストを悪化させるか、なぜレンダリングを最も視覚的に魅力的にするまさにその機能とともに問題が成長するかを正確に理解しました。

次に解決策を完全に学びました。Opacity Micromaps は、GPU の繰り返しの問いに対する答えを事前ベイクします。各三角形をマイクロ三角形のグリッドに分割し、それぞれに恒久的な状態を分類します。その分類はアクセラレーション構造自体に格納され、固定機能トラバーサルハードウェアが、プログラマブルシェーダードメインに一切入ることなく、アルファテスト交差の圧倒的大部分を解決できるようにします。any-hit shader はどこにでもいる働き者から稀な専門家へ降格され、本当に不確かな縁領域に対してだけ呼ばれます。

その結果、植生の多いシーンのシャドウは劇的に速く走り、見た目は同一で、シャドウシェーダーへの変更は不要です。節約はソフトシャドウのサンプル数とともに増え、視覚品質にとって最も重要なレンダリング構成でまさに最も価値が大きくなります。

== グラフィックスに繰り返されるテーマ

ここでの問題から解決策への旅は、リアルタイムグラフィックス最適化の歴史を通じて現れるパターンに従います。作業を実行時から前処理へ、汎用コンピュートから専用ハードウェアへ、発見から知識へ移すことです。

ライトマップは動的な光評価をオフラインベイクへ移しました。事前計算放射輝度転送は複雑なライティング積分を事前計算へ移しました。メッシュシェーダーはジオメトリ増幅をよりハードウェア意識の高い段階へ移しました。どの場合も、根底の洞察は同じです。答えがあらかじめ知り得るなら、コストを一度払い、可能な限りハードウェアがアクセスしやすい形で結果をキャッシュせよ。Opacity Micromaps はこの原則の例外的にきれいな実例です。事前ベイクデータは小さく、利益は大きく、既存のレンダリングパイプラインへの統合はほぼ継ぎ目がありません。

このパターンを理解することは、転用可能なスキルを与えます。次にレイトレースまたはラスタライズパイプラインでパフォーマンスボトルネックに遭遇したとき、最初に尋ねるべき問いの一つはこうです。「これはあらかじめ知り得た情報か？」答えがはいなら、最適化が見つかる可能性が高いです。

== 今あなたが理解していること

あなたは、真にモダンなハードウェアレベルの GPU 機能を理解しています。`VK_KHR_opacity_micromap` は micromap オブジェクトを標準の `VkAccelerationStructureKHR` ハンドルに統一し、純粋なデバイス側 API を使います — micromap 構築は、他のすべてのアクセラレーション構造ビルドと同じ `vkCmdBuildAccelerationStructuresKHR` コマンドを通ります。その設計は寿命管理を単純で一貫したものに保ちます。多くのグラフィックス開発者はまだ micromap に慣れていません。この拡張をよく理解することは意味のある差別化です。

あなたは API 表面だけでなく、API が存在する理由も理解しています。どの問題を解決するか、ハードウェアのどのレベルで動作するか、実際のトレードオフは何かを知っています。その深さの理解が、機能を使える人と、それを評価し、拡張し、他者に説明できる人を分けます。

== 次のステップ

`Courses/` 添付フォルダ内の `OpacityMicromapBuilder` ソースファイルを開いてください。本コースの概念的枠組みを新鮮に保ったまま、実装を通読してください。三つのフェーズ — 解析、分類、構築 — がコード構造にどうマップするかに注目してください。分割レベルのヒューリスティックを探し、レンダリングしているコンテンツに合うかどうかを考えてください。

レイクエリシェーダーを検査するときは、`OpacityMicromapKHR` SPIR-V 実行モードの宣言を探してください。micromap 有効なアクセラレーション構造をトラバースするシェーダーはこの実行モードを宣言しなければならず、省略すると SPIR-V 仕様により **未定義動作** になります。小さな詳細ですが、micromap 有効ジオメトリと相互作用するレイクエリシェーダーを読む・書くときに認識すべき重要なものです。

エンジンで設定フラグを使い OMM を有効・無効にし、GPU プロファイラ — NVIDIA Nsight Graphics または AMD Radeon GPU Profiler — を使って、植生の多いシーンで両方の構成をキャプチャしてください。any-hit shader の呼び出し回数、BVH トラバーサル段階の時間、ワープ占有率メトリクスを見てください。数字は、どんな説明も完全には置き換えられない直感を与えます。

さらに進む準備ができたら、`VK_EXT_displacement_micromap` を探ってください — 同様のマイクロ三角形分割モデルを不透明度ではなく幾何変位に使う兄弟拡張です。Opacity Micromaps がアルファをアクセラレーション構造にベイクできるようにするのに対し、Displacement Micromaps は細かい表面詳細 — 通常ディスプレースメントマップに格納される種類 — を、その詳細をメッシュにテッセレートするポリゴンオーバーヘッドなしに、直接 BVH にベイクできます。本コースで築いた概念的基盤はそのまま転用できます。

ハードウェアは巧妙です。それに沿って考えることを学ぶのが、より効果的なグラフィックスエンジニアになる方法です。
