== GUI: はじめに

== はじめに

「シンプルなエンジンの構築」シリーズの「GUI」の章へようこそ。前章でカメラシステムを実装したのに続き、ここでは Vulkan アプリケーションにグラフィカルユーザーインターフェース（GUI）を追加することに焦点を当てます。よく設計された GUI は、ユーザーが設定を操作し、情報を表示し、3D シーンとやり取りできるインタラクティブなアプリケーションを作るうえで不可欠です。

この章では、人気の即時モード GUI ライブラリである Dear ImGui を Vulkan エンジンに統合します。Dear ImGui は、そのシンプルさ、パフォーマンス、柔軟性からゲームやグラフィックス業界で広く使われています。従来の保持モード GUI システムの複雑さなしに、デバッグインターフェース、ツール、ゲーム内メニューを素早く作成できます。

本チュートリアルでは、プロフェッショナルな GUI システムを Vulkan エンジンに統合する手順を案内します。まず Dear ImGui を Vulkan とセットアップし、すべての GUI 機能の基盤を築きます。統合には Vulkan リソースの慎重な管理が必要です。既存のレンダリングシステムと干渉せずに並存する、専用のバッファ、テクスチャ、パイプラインを作成します。

ユーザー入力の処理は、3D シーンのナビゲーションと GUI 操作の両方をサポートする必要があるため、より複雑になります。3D ワールド向けの入力とインターフェース要素向けの入力を区別できるシステムを実装し、両方とのスムーズな操作を確保します。

ウィジェットの網羅的な例で圧倒するのではなく、GUI 機能を可能にする主要な統合概念に焦点を当てます。これらの原則を理解すれば、プロジェクトに必要な任意のインターフェース要素を実装できます。

レンダリング統合には興味深い課題があります。GUI は既存のパイプラインを乱さずに 3D シーンの上に描画する必要があります。レンダーパスを慎重に管理し、適切な深度テストとブレンディングを確保することで、これを解決します。

最後に、GUI と 3D シーンの橋渡しとなるオブジェクトピッキングを実装します。この機能により、ユーザーは 3D オブジェクトをクリックしてインターフェース上でそのプロパティを確認でき、一体感のある開発環境を作れます。

この章の終わりまでに、カメラの制御、レンダリング設定の調整、3D シーンとのやり取りに使える機能的な GUI システムが手に入ります。これは後の章で扱うマテリアルエディタ、シーン階層、デバッグツールなど、より高度な機能の基盤となります。

== 前提知識

この章は Camera & Transformations の章の直接の続きであり、そこで開発したカメラシステムを GUI 操作とシームレスに連携するよう拡張します。カメラ操作は、ユーザーがインターフェース要素とやり取りしているのか、3D シーンをナビゲートしているのかを認識する必要があります。

いくつかの中核的な Vulkan 概念もしっかり理解しておく必要があります。レンダリングパイプラインとコマンドバッファの知識は重要です。GUI レンダリングは既存の 3D レンダリングとの慎重な調整が必要で、同じコマンドバッファに GUI の描画コールを記録しつつ、異なるパイプライン状態を管理します。

バッファとイメージの作成スキルも不可欠です。Dear ImGui はジオメトリ用の専用頂点・インデックスバッファと、フォントやカスタム UI テクスチャ用のテクスチャリソースを必要とします。ディスクリプタセットとレイアウトの理解も重要で、3D シーンのディスクリプタと干渉しない GUI レンダリング専用のディスクリプタを作成します。

パイプライン作成の知識ですべてが結び付きます。3D パイプラインとは異なる頂点入力、シェーダー、レンダー状態を持つ、GUI レンダリング専用のグラフィックスパイプラインを構築します。

入力処理の基本概念に慣れておくと、3D ナビゲーションと GUI 操作を区別するデュアルモード入力システムの実装についていきやすくなります。
