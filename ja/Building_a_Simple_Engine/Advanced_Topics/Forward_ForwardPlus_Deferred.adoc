= Forward、Forward+、Deferred — 適切な経路の選び方

Vulkan では多くの種類のパイプラインを組めます。実務では、ほとんどのリアルタイムエンジンが 3 つのシェーディングアーキテクチャのどれかに寄ります: Forward、Forward+、Deferred。

このページではそれぞれが何か、このサンプルがなぜ Forward+ を選ぶか、関連する部品がコードのどこにあるかを説明します。

== Forward レンダリング

Forward は各オブジェクトを、そのライティング込みで単一パスで描きます。いちばん直接的なモデルです: マテリアルをバインドし、ライト（ユニフォームやテクスチャ／SSBO）をバインドし、描く。推論しやすく、透過や MSAA とも相性が良いです。

長所:

* シンプルで予測しやすい。
* 透過オブジェクトと MSAA に強い。
* ライト数が少ない場合やベイクド照明に向く。

短所:

* ライト数が増えるとピクセルごとのライトループが高くなり得る。
* ほとんどのライトがピクセルに影響しなくても評価してしまう。

== Forward+（動的ライトに使っているもの）

Forward+ は画面をタイルに分割し、コンピュートパスでライトをタイルに割り当てます。メインパスはそのピクセルのタイルに関連するライトだけでシェーディングします。このサンプルでは、コードを親しみやすく保つため、emissive／簡略ライトに焦点を当てた軽量な Forward+ を使います。

長所:

* 多くのローカルライトにスケールする; ピクセルに影響し得るライトだけを評価する。
* Forward の強み（透過／MSAA との相性）を保つ。

短所:

* タイルリスト構築のための pre-pass またはデプス情報とコンピュートディスパッチが必要。
* 素の Forward より動く部品が多い。

== Deferred シェーディング（検討するとき）

Deferred は最初のパスでマテリアルプロパティ（G-Buffer）を書き、2 番目のパスでそのバッファを照らします。照明コストを「照らされたピクセルごとのコスト」に変え、多くのライトで優れますが、透過と MSAA は難しくなります。

長所:

* 多くの動的ライトを高パフォーマンスで扱える。
* マテリアル／書き込みとライト／評価の分離が明確。

短所:

* 透過オブジェクトは別扱いが必要（しばしば Forward パスで）。
* MSAA はより複雑; メモリ帯域が高くなり得る。

== サンプルが使うもの（とその理由）

小さな動的ライトには Forward+、それ以外は Forward のマテリアル経路を使います。コードをコンパクトに保ちつつ、シーン周りに多くの小さなライトを置けるようにします。透過（ガラス）は順序とブレンディングが正しいよう、2 番目の Forward パスでシェーディングします。

プロジェクトが数百のシャドウ付きライトと複雑なポストライティングを必要とするなら、Deferred 経路やハイブリッド（不透明は Deferred、透過は Forward）を検討してください。

== このコードベースでの実装の要点

* 小さなコンピュートパスがタイルごとのライトリストを構築する。
* フレームごとの SSBO がタイルヘッダ／ライトインデックスを保持; メイン PBR パスはそれを読んで関連ライトだけをループする。
* ディスクリプタ更新はフレームの安全な時点で行い、使用中セットに触れない。

== コードの参照先

* Forward／Forward+ レンダーループ統合:
** `renderer_rendering.cpp`
* パイプライン + ディスクリプタレイアウトのセットアップ:
** `renderer_pipelines.cpp`
* メイン PBR シェーダー（Forward+ 有効時にタイルごとのライトリストを読む）:
** `shaders/pbr.slang`

NOTE: タイル／クラスタ構築シェーダーは `renderer_pipelines.cpp` に配線されています。そこから始め、Forward+ のライト割り当てパス用にどのコンピュートパイプラインが作られるかを追ってください。

== 自分のプロジェクトでの選び方

Forward を使う場合:

* ライト数が少なく、透過／MSAA が優先で、いちばん単純なパイプラインが欲しいとき。

Forward+ を使う場合:

* 多くのローカルライトが欲しいが、Forward の強みも欲しいとき。

Deferred を使う場合:

* 多くの動的ライトと複雑なライティングにスケールする必要があり、透過／MSAA を別途解く準備があるとき。

答えは一つではありません。ニーズを満たすいちばん単純なものを選んでください。パイプラインは後から成長させられます。

== 今後のアイデア

「読みやすいサンプル」を超えて照明系を広げたい場合:

* 2D タイルの代わりに clustered Forward+（デプススライスを使う 3D クラスタ）を追加する。
* シャドウを追加する（まず単一のディレクショナルシャドウマップ、次にポイント／スポットシャドウ）。
* 不透明専用の小さな Deferred 経路を追加する（透過は Forward のまま）。
* 完全な RT パイプラインを組まずに、選択的エフェクト（反射レイ、シャドウレイ、AO プローブ）向けの ray query ヘルパーを追加する。
