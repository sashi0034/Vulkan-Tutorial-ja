== Ray Query の反射と透過

Ray queries を使うと、完全なレイトレーシングパイプラインを採用せずに、レンダラーへ反射と屈折を足しやすくなります。このエンジンでは、Ray Query モードのコンピュートシェーダーがすでにプライマリの可視性を計算しており、反射・透過マテリアル向けに *セカンダリレイ* でそのシェーダーを拡張します。

このページでは、自分のプロジェクトでも再利用できる形で設計を説明します。

=== 2 つのトグル、1 つの明確なメンタルモデル

Ray Query モードは 2 つの機能トグルを公開します:

* *Reflections*: 最初のヒットから反射レイを有効にする。
* *Transparency/Refraction*: 透過マテリアル向けに屈折レイを有効にする。

品質用の小さなノブもあります:

* *Max secondary bounces*: `0` はセカンダリレイを完全に無効; `1` は 1 バウンスだけ有効。

バウンス上限の狙いは、物理ベースのシェーディングモデルに ray queries を重ねる様子を示しつつ、パフォーマンスを予測可能に保つことです。

=== 反射レイ（1 バウンス）

最初の表面ヒットでは次があります:

* 出射の視線方向 `V`
* 表面法線 `N`
* マテリアルパラメータ（roughness、metallic、Fresnel のベース反射率）

反射方向は標準の幾何学的反射です:

`R = reflect(-V, N)`

コンピュートシェーダーでは、自己交差を避けるために法線方向へわずかにオフセットした位置から新しいレイをトレースします:

* origin: `P + N * eps`
* direction: `R`

反射レイが何かに当たれば、プライマリレイと同じ PBR 経路でそのヒットをシェーディングします。ミスなら安定した空／背景関数を使います。

最終的な反射寄与は Fresnel で重み付けし、roughness で弱めます:

* グレージング角ほど強く反射する
* 粗い表面ほど反射は弱い

これにより結果は直感的で安定します。

=== 薄いガラスの屈折（1 バウンス）

透過マテリアルには「薄いガラス」モデルを実装します:

* 屈折レイは表面を *通した* 見え方を与える
* 反射レイは表面上の見え方を与える
* Fresnel が両者をブレンドする

屈折は Snell の法則で計算し、単純な全反射フォールバックを付けます。

屈折レイは次を使います:

* origin: `P + refrDir * eps`（屈折方向へのオフセット）
* direction: `refrDir`

透過結果は Fresnel で反射とブレンドし、マテリアルの transmission 係数でベース表面色へ混ぜます。

=== アルファマスク表面（植生など）

多くの実シーンは植生や薄いジオメトリに *アルファマスキング* を使います。アルファマスキングは通常のブレンディングとは違います:

* 表面はピクセルごとに存在するかしないか
* 判定は baseColor のアルファテクスチャと `alphaCutoff` で駆動される

伝統的なレイトレーシングパイプラインでは、アルファマスキングは any-hit シェーダーで実装されることが多いです。ray queries では、どの候補交差をコミットするかを制御することで同じ考え方を実装できます。

手順は:

1. アルファマスク付きインスタンスに対し、非不透明な候補を許可する。
2. 各候補三角形ヒットについて:
   * 候補 UV を計算する
   * baseColor アルファをサンプルする
   * `alpha >= alphaCutoff` のときだけ候補を受け入れる

これによりプライマリレイでマスク付きジオメトリの可視性が正しくなり、同じトラバーサルルーチンを使うため反射／屈折も一貫します。
