= モデルの読み込み: はじめに

== はじめに

「シンプルなエンジンの構築」シリーズの「モデルの読み込み」の章へようこそ。前章でエンジンアーキテクチャとカメラシステムを探ったあと、エンジンフレームワーク内で 3D アセットを扱うことに焦点を当てる準備が整いました。

この章では、モダンな 3D アセットを扱える堅牢なモデル読み込みシステムを構築します。確立したエンジンアーキテクチャと実装したカメラシステムの上に、複雑な 3D モデルを読み込んでレンダリングする能力を追加します。本編チュートリアルの xref:15_GLTF_KTX2_Migration.adoc[glTF と KTX2 の章] では、OBJ から glTF 形式への移行と glTF モデル読み込みの基礎を学びました。ここでは、その知識をエンジン構造に統合し、より完成度の高い実装を作ります。

この章は、単純なモデル読み込みから洗練されたエンジンレベルのシステムへと、3D アセット処理の理解を変えます。まずシーングラフを構築します。これは複雑な 3D シーンに必要な階層的な整理を提供します。この基盤により、オブジェクトを論理的にグループ化し、異なるレベルで変換を適用し、シーンの複雑さを効率的に管理できます。

アニメーションサポートは、モダンな 3D アプリケーションの重要な部分です。glTF のスケルタルアニメーションを扱えるシステムを実装し、滑らかなキャラクター移動、オブジェクトアニメーション、複雑な多部品システムを通じて 3D モデルに命を吹き込みます。

作成する PBR マテリアルシステムは、前章のライティング概念と実世界のアセット統合の橋渡しをします。glTF のマテリアルプロパティをシェーダーへシームレスにマッピングする方法を見て、アーティストが理解し効果的に使えるワークフローを作ります。

異なる変換を持つ複数オブジェクトのレンダリングは、技術的にも組織的にも課題です。類似オブジェクトを効率的にバッチしつつ、オブジェクトごとの固有マテリアルと変換を扱える柔軟性を保つ、注意深いエンジンアーキテクチャでこれを解決します。

この実装全体を通じて、チュートリアル風の解決策ではなくエンジンレベルの考え方でコードを構成します。このアプローチは、プロジェクトの複雑さと規模が増すにつれて役立ち、アニメーション付きモデルを持つ複雑なシーンを作るための堅固な基盤を提供します。

== 前提条件

この章は、本編 Vulkan チュートリアル、特に第 16 章（複数オブジェクト）で確立した基盤の上に築かれます。より複雑なシーン構成とアセット管理を扱うために、それらの概念を拡張します。複数オブジェクトの章は異なるジオメトリのレンダリングの基本概念を紹介しており、ここではマテリアルとアニメーションを持つ完全な 3D モデルを扱えるようスケールアップします。

モデル読み込みシステムの背骨となるコア Vulkan 概念への確かな慣れが必要です。xref:03_Drawing_a_triangle/03_Drawing/01_Command_buffers.adoc[コマンドバッファ] は、異なるマテリアルを持つ複数モデルを扱うときにより複雑になります。ディスクリプタセットとプッシュ定数を効率的に管理する必要があるためです。xref:03_Drawing_a_triangle/02_Graphics_pipeline_basics/00_Introduction.adoc[グラフィックスパイプライン] の理解は、異なるマテリアルが異なるパイプライン構成を要する可能性があるため重要です。

xref:04_Vertex_buffers/00_Vertex_input_description.adoc[頂点] および xref:04_Vertex_buffers/03_Index_buffer.adoc[インデックスバッファ] の経験は、特定形式の頂点データを含む glTF ファイルをパースして GPU バッファへアップロードするモデル読み込みに直接つながります。xref:05_Uniform_buffers/00_Descriptor_set_layout_and_buffer.adoc[ユニフォームバッファ] の知識は、変換行列、ライティング情報、マテリアルプロパティに使うため不可欠になります。

xref:06_Texture_mapping/00_Images.adoc[テクスチャマッピング] のスキルは特に重要です。glTF モデルはしばしばマテリアルごとに複数のテクスチャ（diffuse、normal、metallic-roughness など）を含み、これらを効率的に読み込んでバインドする必要があるためです。

最後に、基本的な 3D 数学の理解（行列、ベクトル、クォータニオン）は、モデル変換、アニメーション、シーン階層の扱いに不可欠です。復習が必要なら、xref:Building_a_Simple_Engine/Camera_Transformations/02_math_foundations.adoc[カメラと変換の章] でこれらの数学概念を詳しく扱っています。
