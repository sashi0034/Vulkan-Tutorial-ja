= カメラと変換: 変換行列

== 変換行列

この節では、3D グラフィックスで使われる変換行列と、レンダリングパイプラインでの適用をより深く掘り下げます。

=== Model-View-Projection（MVP）パイプライン

頂点をオブジェクト空間からスクリーン空間へ変換するには、一連の行列乗算が関わり、一般に MVP パイプラインと呼ばれます。

[source,cpp]
----
// The complete transformation pipeline
glm::mat4 MVP = projectionMatrix * viewMatrix * modelMatrix;
----

これらの行列をそれぞれ詳しく見ていきましょう。

=== モデル行列

モデル行列は、頂点をオブジェクト空間からワールド空間へ変換します。ワールド内でオブジェクトを配置・回転・スケールします。

[source,cpp]
----
glm::mat4 createModelMatrix(
    const glm::vec3& position,
    const glm::vec3& rotation,
    const glm::vec3& scale
) {
    // Start with identity matrix
    glm::mat4 model = glm::mat4(1.0f);

    // Apply transformations in order: scale, rotate, translate
    model = glm::translate(model, position);

    // Apply rotations around each axis
    model = glm::rotate(model, glm::radians(rotation.x), glm::vec3(1.0f, 0.0f, 0.0f));
    model = glm::rotate(model, glm::radians(rotation.y), glm::vec3(0.0f, 1.0f, 0.0f));
    model = glm::rotate(model, glm::radians(rotation.z), glm::vec3(0.0f, 0.0f, 1.0f));

    // Apply scaling
    model = glm::scale(model, scale);

    return model;
}
----

=== ビュー行列

ビュー行列は、頂点をワールド空間からビュー空間（カメラ空間）へ変換します。カメラの位置と向きを表します。

[source,cpp]
----
glm::mat4 createViewMatrix(
    const glm::vec3& cameraPosition,
    const glm::vec3& cameraTarget,
    const glm::vec3& upVector
) {
    return glm::lookAt(cameraPosition, cameraTarget, upVector);
}
----

`lookAt` 関数は、カメラを `cameraPosition` に置き、`cameraTarget` を見、上方向を `upVector` で定義するビュー行列を作成します。

=== プロジェクション行列

プロジェクション行列は、頂点をビュー空間からクリップ空間へ変換します。3D 座標が 2D スクリーンへどう投影されるかを定義します。

==== 透視投影

透視投影は、オブジェクトが遠ざかるにつれて小さく見える様子をシミュレートし、私たちの目が世界を自然に知覚する方法と同じです。

[source,cpp]
----
glm::mat4 createPerspectiveMatrix(
    float fovY,
    float aspectRatio,
    float nearPlane,
    float farPlane
) {
    return glm::perspective(glm::radians(fovY), aspectRatio, nearPlane, farPlane);
}
----

パラメータ:

* `fovY`: 視野角（度、垂直）
* `aspectRatio`: ビューポートの幅を高さで割った値
* `nearPlane`: ニアクリッピング平面までの距離
* `farPlane`: ファークリッピング平面までの距離

==== 正射影

正射影には透視歪みがないため、2D レンダリングや技術図面に有用です。

[source,cpp]
----
glm::mat4 createOrthographicMatrix(
    float left,
    float right,
    float bottom,
    float top,
    float nearPlane,
    float farPlane
) {
    return glm::ortho(left, right, bottom, top, nearPlane, farPlane);
}
----

=== 法線行列

オブジェクトに非均一スケールを適用すると、モデル行列で変換した法線が正しくなくなることがあります。法線行列がこの問題を解決します。

[source,cpp]
----
glm::mat3 createNormalMatrix(const glm::mat4& modelMatrix) {
    // The normal matrix is the transpose of the inverse of the upper-left 3x3 part of the model matrix
    return glm::transpose(glm::inverse(glm::mat3(modelMatrix)));
}
----

=== シェーダーでの変換の適用

Vulkan では、通常これらの行列をユニフォーム変数としてシェーダーに渡します。

[source,glsl]
----
// Vertex shader
#version 450

layout(binding = 0) uniform UniformBufferObject {
    mat4 model;
    mat4 view;
    mat4 proj;
} ubo;

layout(location = 0) in vec3 inPosition;
layout(location = 1) in vec3 inNormal;
layout(location = 2) in vec2 inTexCoord;

layout(location = 0) out vec3 fragNormal;
layout(location = 1) out vec2 fragTexCoord;

void main() {
    // Apply MVP transformation
    gl_Position = ubo.proj * ubo.view * ubo.model * vec4(inPosition, 1.0);

    // Transform normal using normal matrix
    mat3 normalMatrix = transpose(inverse(mat3(ubo.model)));
    fragNormal = normalMatrix * inNormal;

    fragTexCoord = inTexCoord;
}
----

=== 階層的変換

親子関係を持つ複雑なオブジェクトやシーンでは、階層的変換を使います。

[source,cpp]
----
// Parent transformation
glm::mat4 parentModel = createModelMatrix(parentPosition, parentRotation, parentScale);

// Child transformation relative to parent
glm::mat4 localModel = createModelMatrix(childLocalPosition, childLocalRotation, childLocalScale);

// Combined transformation
glm::mat4 childWorldModel = parentModel * localModel;
----
