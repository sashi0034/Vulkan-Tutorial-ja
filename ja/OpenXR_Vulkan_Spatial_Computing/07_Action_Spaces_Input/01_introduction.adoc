= アクションスペースと入力からパイプラインへの流れ

従来の PC やモバイルゲームでは、入力を生の物理デバイスの観点で考えがちです。「A ボタンは押されたか？」「マウスデルタは何か？」しかし空間コンピューティングでは、このモデルは破綻します。標準ゲームパッドや 6DoF ハンドコントローラからハンドトラッキング、アイトラッキングまで、VR／AR コントローラの多様性は、エンジンがユーザーが押している *特定の* ボタンを気にすべきでないことを意味します。

代わりに OpenXR は **アクションベース入力システム** を使います。この抽象レイヤーにより、ユーザーに *何をしてほしいか*（「Grab」「Teleport」「Menu」など）を定義し、後でそれらのアクションを様々なハードウェアプロファイルへマップできます。これによりエンジンは将来も安心で、現在および将来のすべてのデバイスと互換になります。

しかし空間コンピューティングでは、入力はボタンだけではありません。**ポーズアクション** です。ユーザーが手を動かすと、ボタンイベントだけでなく、3D 空間における位置と向き——空間変換全体——を得ます。これらのアクションは、以前簡単に触れた **スペース** 内で追跡されます。

この章では次を探ります。
1. OpenXR アクションを物理コントローラプロファイルへ定義・マップする方法。
2. 入力とレンダリングパイプラインのギャップを埋める **アクションスペース** の概念。
3. 生の XrSpace ポーズを、インタラクション向けに Vulkan シェーダーが使える 4x4 変換行列へ変換する方法。

このセクションの終わりには、デジタル入力（ボタンクリック）と空間入力（手の位置）の両方を、クリーンでモジュール化され高性能な方法で扱う堅牢なシステムが整います。
