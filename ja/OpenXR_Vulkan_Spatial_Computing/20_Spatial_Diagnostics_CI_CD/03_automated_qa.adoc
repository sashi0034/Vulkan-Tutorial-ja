= 自動 QA とヘッドレス CI/CD

空間エンジンのテストは、**ヘッドレスランタイム** と AI 駆動の視覚分析を使って自動化でき、物理ヘッドセットなしで堅牢な CI/CD パイプラインを可能にします。

== エコシステムの視点

この章は次のカテゴリに属します: **OpenXR ランタイムと Vulkan の併用**。

`XR_EXT_headless` 拡張を使い空間エンジンを自動 CI/CD パイプラインへ統合することは、大規模なコード品質維持の標準要件です。高速ビルド系が管理できるディスプレイなしセッションを作るには、これらの拡張を使う必要があります。

== 概念: ヘッドレスランタイム

**ヘッドレスランタイム**（OpenXR Simulator など）により、物理ヘッドセットや GPU なしでエンジンを実行できます。

1.  **モックトラッキングデータ**: シミュレータが事前録画またはスクリプト化されたポーズとコントローラ入力を提供する。
2.  **スワップチェーンキャプチャ**: エンジンがスワップチェーンへ描画し、シミュレータがこれらのフレームをディスクへ直接キャプチャする。
3.  **視覚検証**: これらのフレームを「ゴールドスタンダード」ベースラインと比較し、回帰がないことを保証する。

このモードではランタイムが完全に決定論的なポーズを生成し、すべての CI 実行が同一になり、スレッドやハンドル問題を検査する診断レイヤの利用を容易にします。

== 回帰向け AI ビジョン分析

従来の「ピクセル完全」比較は、再投影や MSAA の微妙な変動のため、空間コンピューティングではしばしば失敗します。代わりに **AI ビジョンモデル** を使い、特定のアーティファクトを検出します。

*   **非対称フラスタムエラー**: 視界端での「伸び」や「つまみ」を検出する。
*   **マルチビュー不整合**: 一方の目から欠けた物体を検査する。
*   **時間的マスクフリッカ**: **セマンティックオクルージョン** マスクの不安定性を特定する。

== CI での性能監視

性能の「クリープ」を防ぐため、すべてのビルドで **空間性能メトリクス** を追跡します。

*   **提出レイテンシ**: `xrWaitFrame` と `xrEndFrame` の間の合計時間。
*   **Wait-to-Begin ギャップ**: レイトラッチング同期の待機に費やした CPU 時間。
*   **メモリフットプリント**: 4D **ライトフィールド** と **セマンティックオクルージョン** バッファの VRAM 使用量。

== 発展: 自動視覚テストとレンズシミュレーション

Vulkan は、標準ヘッドレス実行を超える自動 QA の深さを可能にします。

*   **自動視覚回帰**: Vulkan の **Offscreen Rendering** と **Image Readback** を使い、毎フレームをキャプチャし SSIM（Structural Similarity Index）でゴールド画像と比較し、人間介入なしで視覚的正しさを保証できる。
*   **物理レンズシミュレーション**: Vulkan **コンピュートシェーダー** を使い、CI パイプラインでレンズアーティファクト（God Rays、色収差）をシミュレートし、ハードウェアへデプロイする前に UI とコンテンツが物理ガラス越しにどう見えるかをテストできる。
