= ネイティブマルチビュー

**シングルパスステレオ** の背後にある技術が **ネイティブマルチビュー**（`VK_KHR_multiview`）です。この機能により、単一のドローコールをイメージ配列の複数レイヤーへブロードキャストできます。少なくとも 2 ビューをほぼ常に持つ空間コンピューティングでは、これは大きな性能向上です。

== エコシステムの視点

この章は次のカテゴリに属します: **OpenXR ランタイムと Vulkan を使う**。

Vulkan のネイティブマルチビューサポートを活用することは、OpenXR で高性能ステレオレンダリングを実装する業界標準の方法です。最小の CPU オーバーヘッドでランタイムのマルチビュー要件を満たせます。

== CPU オーバーヘッドの勝利: なぜループを嫌うか

マルチビューの価値を理解するには、代替案を見てください。**マルチパスループ** です。

1.  **ループ**: 各眼について（0 から 1）:
2.  **バインド**: その眼向けにビュー／投影 UBO を更新する。
3.  **ドロー**: シーンのすべてのドローコールを再発行する。

複雑なシーンでは CPU が二倍の仕事をし、しばしば **CPU バウンド** になります。**マルチビューはループを殺します。** ドローコールを *一度* 発行し、GPU の固定機能ハードウェアがブロードキャストを処理します。CPU オーバーヘッドは半分になり、GPU は飽和したままにできます。

[source,cpp]
----
// Enabling Multiview in Vulkan 1.3 using designated initializers
vk::RenderingInfo renderingInfo{
    .renderArea = {{0, 0}, {xrWidth, xrHeight}},
    .layerCount = 1, // Must be 1 when using multiview masks
    .viewMask = 0b11, // Broadcast to Layer 0 (Left) and Layer 1 (Right)
    .colorAttachmentCount = 1,
    .pColorAttachments = &colorAttachment
};

commandBuffer.beginRendering(renderingInfo);
----

== ビューマスクとブロードキャスト機構

標準の Vulkan パイプラインでは、ドローコールは特定のアタッチメントをターゲットにします。マルチビューが有効な場合、`vk::RenderingInfo` で **ビューマスク** を指定します。ビューマスクはビットマスクで、セットされた各ビットが出力配列のレイヤーを表します。

*   **マスク 0b11（3）**: レイヤー 0（左眼）とレイヤー 1（右眼）の両方へ描画。
*   **マスク 0b1111（15）**: 4 レイヤーへ描画（Quad-Views で使用）。

マルチビューを使うことで、ランタイムがデータを受け取ることを期待する方法にエンジンを揃えます。ほとんどのランタイムは、眼間のより良いハードウェアレベル並列化とコヒーレントキャッシュのため **配列ベーススワップチェーン** を好みます。

== ハードウェア実装: SV_ViewID

マルチビューがアクティブなとき、GPU はマスクでセットされた各ビットについて頂点シェーダーを実行します。シェーダー内では、`SV_ViewID` セマンティックで現在のビューインデックスにアクセスします。

[source,slang]
----
// Using the View ID in a Slang Shader
struct VertexOutput {
    float4 position : SV_Position;
    uint viewID : SV_ViewID;
};

VertexOutput main(VertexInput input, uint viewID : SV_ViewID) {
    VertexOutput output;
    // Load the correct eye matrix using the viewID
    float4x4 mvp = eyeMatrices[viewID];
    output.position = mul(mvp, float4(input.position, 1.0));
    output.viewID = viewID;
    return output;
}
----

== 発展: ビュー固有スケーリングとオクルージョンカリング

Vulkan では、標準の OpenXR マルチビュー統合を超えた粒度の細かい制御が可能です。

*   **Per-View 属性**: Vulkan の **Multiview Per-View Attributes** により、単一ドローコール内で各ビューインデックスに異なる解像度やシェーダー属性を使えます。周辺ビューに低解像度ジオメトリを使うなどの最適化が可能です。
*   **クロスビュー依存**: Vulkan の **サブグループ演算** を使い、異なるビュー間のシェーダー呼び出しでデータを共有できます。これにより **Inter-View Occlusion Culling** が可能になり、一方の眼の可視性テストを他方と共有し、冗長な作業を大幅に削減します。
