= ピクセル単位マスキングと安定した合成

仮想物体を現実世界と合成するには、オクルージョン境界での「ハロー」や「ジッタ」を防ぐため、安定したピクセル単位マスキングが必要です。

== エコシステムの視点

この章は次のカテゴリに属します: **OpenXR 標準の先へ**。

OpenXR は基本的な環境ブレンディングを管理しますが、標準は物理ベースの光輸送やブルームマスキングを管理しません。Vulkan のハードウェアステンシルパイプラインとコンピュートシェーダーを使い、標準 OpenXR コンポジタがネイティブに協調しない高度な物理ベース合成と時間的安定性を実装します。

== 概念: 合成方程式

Augmented Reality（AR）では、「背景」はライブカメラフィード（パススルー）です。現実的なオクルージョンを達成するため、セマンティックマスクを **ゲーティング関数** として適用します。

1.  **ハードオクルージョン**: 二値決定（0 または 1）。単純ですが、頭の動き中にシマーするエイリアスエッジにつながる。
2.  **ソフトオクルージョン**: 境界の滑らかな勾配。マスクを「フェザー」することで、仮想と現実を自然にブレンドし、ML パスの不完全さを隠す。

ランタイムはアルファブレンディングモードの選択を扱い、シャープネスを保証するため、提供したアルファチャネルに後期フェザーリングを適用することがあります。

== ステンシルベースマスキングの実装

Vulkan エンジンでは、**ステンシルバッファ** を使い高速マスキングを行います。ハードウェアが早期にピクセルを破棄できるため、フラグメントシェーダーの分岐より効率的です。

[source,cpp]
----
// Configuring stencil masking using designated initializers
vk::StencilOpState stencilOp{
    .failOp = vk::StencilOp::eKeep,
    .passOp = vk::StencilOp::eReplace,
    .depthFailOp = vk::StencilOp::eKeep,
    .compareOp = vk::CompareOp::eAlways,
    .compareMask = 0xFF,
    .writeMask = 0xFF,
    .reference = 0x1 // Mark real-world occluders
};

vk::PipelineDepthStencilStateCreateInfo depthStencil{
    .stencilTestEnable = VK_TRUE,
    .front = stencilOp,
    .back = stencilOp
};
----

まず ML セグメンテーションマスクでステンシルバッファを埋め、次にステンシルが 0x1 *でない* 場所に仮想シーンを描画します。

== 発展: 物理ベース合成とブルームマスキング

Vulkan により、標準アルファブレンディングを超えてオクルージョン品質を押し上げられます。

*   **物理ベース合成**: Vulkan の **Dual Source Blending** を使い、複雑な光輸送シミュレーションを行い、物理光が仮想面でどう反射すべきかをリアルタイム計算できる。
*   **光学ブルームマスキング**: Vulkan **コンピュートシェーダー** を使い、明るい物理光源を検出し、高コントラスト環境でも仮想オクルージョンが固体感を保つようカスタム「ブルームマスキング」を適用できる。

== 時間的安定性: エッジジッタ問題

ML マスクはフレームごとに生成されるため、フリッカし得ます。修正のため、エンジンは **時間的再投影（Temporal Reprojection）** を実装し、前フレームのマスクを取り、現在の頭ポーズ（OpenXR の予測表示時刻から）で再投影します。これにより現実と仮想の境界が安定したままになります。
