= エンジンへの組み込み

**ライトフィールド理論** とプレノプティックデータ構造の導入により、`simple_game_engine` は `renderer_core.cpp` で **Buffer Device Address（BDA）** を使い高密度メモリアレイを扱う必要があります。*現時点では、実装は読者への演習として残されています。*

== バックエンドでの BDA 有効化

`renderer_core.cpp` では、論理デバイス作成時に BDA 機能を有効にする必要があります。これにより、Slang シェーダーで GPU バッファを生 64 ビットポインタとして扱えます。

[source,cpp]
----
// renderer_core.cpp
bool Renderer::createLogicalDevice(bool enableValidationLayers) {
    // ...
    vk::PhysicalDeviceVulkan12Features vulkan12Features;
    vulkan12Features.bufferDeviceAddress = vk::True;

    // NEW: Also enable 1.3 features for easier memory management
    vk::PhysicalDeviceVulkan13Features vulkan13Features;
    vulkan13Features.dynamicRendering = vk::True;
    vulkan13Features.pNext = &vulkan12Features;
    // ...
}
----

== 4D ライトフィールドバッファの写像

`renderer_resources.cpp` では、ST/UV 平面向けの高密度バッファを作成します。標準のディスクリプタセットの代わりに、生のデバイスアドレスを取得します。

[source,cpp]
----
// renderer_resources.cpp
// Supporting Struct: Push Constants for BDA
struct LightFieldPushConstants {
    uint64_t lightFieldDataAddress;
    float4 uvScaling;
    // ...
};

void Renderer::createLightFieldBuffer(size_t totalBytes) {
    // 1. Create the buffer with BDA usage
    vk::BufferCreateInfo createInfo{
        .size = totalBytes,
        .usage = vk::BufferUsageFlagBits::eStorageBuffer | vk::BufferUsageFlagBits::eShaderDeviceAddress
    };
    lightFieldBuffer = vk::raii::Buffer(device, createInfo);

    // 2. Retrieve the raw 64-bit address
    vk::BufferDeviceAddressInfo addressInfo{ .buffer = *lightFieldBuffer };
    uint64_t gpuPointer = device.getBufferAddress(addressInfo);

    // 3. Pass this pointer to our Slang shader via Push Constants
    pushConstants.lightFieldDataAddress = gpuPointer;
}
----

== なぜこれらの変更が必要か？

ライトフィールドデータで従来のディスクリプタセットから BDA へ移ることで、何千もの個別ビューテクスチャをバインドするオーバーヘッドを避けられます。エンジンは巨大なプレノプティックデータセット（しばしばマルチギガバイト）を単一の連続メモリブロックとして管理できるようになります。これは合成パス中のキャッシュ局所性を最大化し、空間的没入感の維持に不可欠です。
