= キャンテッドディスプレイと非対称フラスタム

標準的なデスクトップレンダラでは、2 つの仮想カメラ（「目」）が互いに平行であると仮定するのが一般的です。これは **平行投影（Parallel Projection）** モデルとして知られています。しかし、Pimax や Valve Index のような多くの最新のハイエンドヘッドセットは、**キャンテッドディスプレイ（Canted Display）** アーキテクチャを採用しています。これらのデバイスでは、物理スクリーンとレンズが外側に傾けられ、より広い **視野角（Field of View, FOV）** を提供します。

この機械的なキャンティングは、レンダリングパイプラインに大きな複雑さを持ち込みます。ビュー行列が互いに単純な横方向オフセットである、とはもはや仮定できません。代わりに、各目は独自の向きと、焦点が画像の中心にない **非対称フラスタム（Asymmetric Frustum）** を持ちます。

この章では、次の手法でこれらの非平行アーキテクチャを扱います。

1.  **XR ポーズの分解**: `XrView` データを使い、キャンティング角を考慮した正しい 4x4 行列を構築する方法を理解する。
2.  **非対称フラスタムの計算**: OpenXR の `fov` タンジェントを使い、光学軸に合わせて「中心」がずれた投影行列を構築する。
3.  **ビューポートスウィズルによる最適化**: **VK_NV_viewport_swizzle** 拡張（またはその Vulkan 1.3 相当）を使い、これらの非直線投影のラスタライズを最適化する。

これらの概念を習得すれば、平行スクリーンのモバイル VR ヘッドセットから超広 FOV のエンスージアスト向けハードウェアまで、空間コンピューティングハードウェアの全スペクトルにエンジンを対応させられます。
