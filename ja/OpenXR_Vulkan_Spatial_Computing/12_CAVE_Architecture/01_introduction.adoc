= CAVE アーキテクチャとマルチ GPU 同期: はじめに

これまで主にヘッドマウントディスプレイ（HMD）に焦点を当ててきましたが、空間コンピューティングは **CAVE**（Cave Automatic Virtual Environment）のような大規模環境も含みます。CAVE は、壁・床・場合によっては天井が高解像度の投影面である部屋です。ユーザーは通常、軽量なシャッターメガネを装着し、カメラ群でトラッキングされ、共有された没入型 3D 空間が生まれます。

CAVE 向けのエンジン設計は、単一 GPU のヘッドセットとは根本的に異なります。6 台以上の 4K プロジェクタを 120Hz で駆動するには、しばしば **ネットワーククラスタ**——それぞれ 1 台以上のハイエンド GPU を備えた強力なワークステーション群——が必要です。ここに巨大な同期課題が生まれます。部屋の角での「ティアリング」を避けるため、すべてのフレームがすべての画面でまさに同じマイクロ秒に表示されなければなりません。

この章では、ルームスケールの空間コンピューティング技術に必要なエンジニアリングに踏み込みます。

1.  **プロジェクタベースの空間技術**: CAVE の幾何学を理解し、複数の非標準投影面を扱うエンジン設計。
2.  **ハードウェア同期**: **VK_NV_present_barrier** 拡張と **スワップグループ** を使い、ネットワーククラスタノード間でフレーム完全同期を実現する。
3.  **分散レンダリング**: クラスタの各ノードが 3D 世界の自分のスライスを描画しつつ、マスターシミュレーションとロックステップを保つデータ並列アーキテクチャを設計する。

== ルームスケール空間技術の課題

HMD では画面が頭に物理的に固定されています。CAVE では画面が壁そのものです。つまり、部屋を歩き回るにつれ、各投影面の **非対称フラスタム** を、トラッキングされた目の位置に基づいてリアルタイムで再計算しなければなりません。

さらに、複数プロジェクタから画像を継ぎ合わせるため、仮想世界が部屋の物理的な角を回り込んでも継ぎ目なく見えるよう、**幾何補正（Geometric Correction）** と **エッジブレンディング**——次の章で扱う概念——を扱う必要があります。

== クラスタの同期

CAVE エンジニアリングの「聖杯」は **Genlock**（Generator Lock）と **Swaplock** です。Genlock は、すべての GPU からのビデオ信号がまさに同じ時刻にプロジェクタへ送られることを保証します。Swaplock は、フロント／バックバッファをスワップするコマンドがクラスタ全体で同時に起きることを保証します。

これらのハードウェアレベルの保証がなければ、CAVE の「没入感」はジッタと視覚的不連続ですぐに壊れます。Vulkan 1.3 の機能と専用ハードウェア拡張で、この精度をどう達成するかを見ていきます。

この章の終わりまでに、空間コンピューティングエンジンを単一ユーザーのヘッドセットから、マルチノードのルームスケール没入環境へどうスケールさせるかを理解できるでしょう。
