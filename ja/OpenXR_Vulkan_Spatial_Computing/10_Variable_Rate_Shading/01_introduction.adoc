= 周辺最適化のための Variable Rate Shading（VRS）

高忠実度空間コンピューティングの核心的な課題は、押し出すべきピクセル数の膨大さです。ヘッドセットが眼あたり 4K 解像度で 90Hz や 120Hz へ向かう中、シーン全体をフル解像度で描画するだけでは計算上しばしば非現実的です。しかし人間の視覚は独自の最適化機会を与えます。**フォビエイテッドレンダリング** です。

眼は **中心窩（fovea）** と呼ばれるごく小さな中心領域でのみ高ディテールを知覚します。視野の周辺へ向かうにつれ、細かいディテールを解像する能力は大幅に低下します。この章では、知覚品質を犠牲にせず周辺領域のフラグメント処理を知的に削減するため、強力な Vulkan 拡張である **Variable Rate Shading（VRS）** をどう活用するかを探ります。

Variable Rate Shading により、シェーディングレートをピクセルレートから切り離せます。ピクセルごとに一度フラグメントシェーダーを実行する代わりに、ピクセル群（例: 2x2 または 4x4 タイル）に対して一度実行するようハードウェアに伝えられます。この「粗いシェーディング」は GPU の **ALU**（Arithmetic Logic Unit）負荷を大幅に削減し、複雑な空間シェーダーではしばしばそれが主要なボトルネックです。

2 つの主要な戦略に焦点を当てます。

1.  **静的周辺最適化**: 光学歪みと色収差がすでにディテールを隠しているレンズ端でシェーディングレートを下げる。
2.  **動的視線駆動シェーディング**: アイトラッキングのテレメトリを使い、ユーザーが現在見ている場所に高解像度領域を中心付ける。

この章の終わりには、**VK_KHR_fragment_shading_rate** 拡張（またはモバイル版の **VK_EXT_fragment_density_map**）を空間パイプラインに統合する方法と、リアルタイムに更新されるシェーディングレートマップの管理方法を理解します。
