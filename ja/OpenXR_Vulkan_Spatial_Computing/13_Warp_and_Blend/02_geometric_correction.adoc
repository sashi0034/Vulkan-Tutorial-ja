= 幾何補正とエッジブレンディング

幾何補正は、直線画像を「ワープ」した画像へ変換し、曲面スクリーンや角など物理面へ投影したときに正しく見えるようにします。

== エコシステムの視点

この章は次のカテゴリに属します: **OpenXR 標準の先へ**。

標準的な HMD ランタイムは内部でワーピングを扱いますが、カスタムプロジェクタ構成や曲面ディスプレイウォールでは、独自の幾何補正を実装する必要があります。Vulkan のコンピュートシェーダーとサブグループ演算を使い、標準の HMD 中心 OpenXR モデルを超えるピクセル完全なエッジブレンディングとワーピングを実装します。

== 概念: 逆引き

ワーピングを理解するには、逆に考える必要があります。「このソースピクセルはどこへ行くか？」ではなく、**「壁上のこのピクセルについて、描画フレームのどこを見るべきか？」** と問います。

1.  **直線ソース**: 標準の 3D レンダー（直線、90 度の角）。
2.  **物理デスティネーション**: 曲面スクリーン、または壁が交わる角。
3.  **ワープマップ**: すべてのデスティネーションピクセルをソースレンダー内の $(u, v)$ 座標へ写すルックアップテーブル（LUT）。

ランタイムは「ワープメッシュ」の永続データベースを維持し、物理面向けに画像を「正方化」するのに必要な射影変換（キーストニング）を計算します。

== ワープマップの実装

この処理の「頭脳」は **ワープマップ** で、通常は浮動小数点データを含む `vk::raii::Image` です。コンピュートシェーダーで逆引きを行います。

[source,cpp]
----
// Configure the warp map texture binding using designated initializers
vk::DescriptorImageInfo warpMapInfo{
    .sampler = *pointSampler,
    .imageView = *warpMapView,
    .imageLayout = vk::ImageLayout::eShaderReadOnlyOptimal
};

vk::WriteDescriptorSet descriptorWrite{
    .dstSet = *warpDescriptorSet,
    .dstBinding = 0,
    .descriptorCount = 1,
    .descriptorType = vk::DescriptorType::eSampledImage,
    .pImageInfo = &warpMapInfo
};
----

Slang では、コンピュートカーネルがデスティネーションピクセルをソース座標へ写します。

[source,slang]
----
// Compute-based warping kernel in Slang
void main(uint3 dispatchThreadID : SV_DispatchThreadID) {
    // 1. Look up where this wall pixel should sample from the rendered image
    float2 sourceUV = warpMap.Load(int3(dispatchThreadID.xy, 0));

    // 2. Sample and write to the final projector output
    outputTarget[dispatchThreadID.xy] = sourceRender.SampleLevel(linearSampler, sourceUV, 0);
}
----

== 発展: ピクセル完全ブレンディングと動的キーストニング

Vulkan は標準キャリブレーションを超える精度を可能にします。

*   **ピクセル完全エッジブレンディング**: Vulkan の **Subgroup Operations** を使い、ピクセル完全精度のリアルタイムエッジブレンディングを実装できる。重なるプロジェクタ領域の「ホットスポット」を動的に除去できる。
*   **動的キーストニング**: コンピュートシェーダーと OpenXR からのトラッキング物体データを使い、セッション中にプロジェクタが物理的に動いてもワープマップをリアルタイム更新する動的キーストニングを実装できる。

== 輝度の加算とエッジブレンディング

マルチプロジェクタ構成では端が重なります。この領域は壁の他の部分の 2 倍明るくなります。解決のため、ワープマップに滑らかな勾配を定義するアルファチャネルを含めます。最終色にこのアルファを掛けることで、壁全体で均一な輝度を保ちます。

== 精度: バイキュービック対 Sinc フィルタリング

ワーピングは非整数座標でテクスチャをサンプリングするため、フィルタリング品質が極めて重要です。線形フィルタリングは高速ですが「ぼやけ」につながり得ます。ハイエンドの空間コンピューティングエンジンは、座標変換中にシャープネスを保つため、コンピュートシェーダー内でカスタム **Bicubic** または **Lanczos** フィルタを実装することが多いです。
