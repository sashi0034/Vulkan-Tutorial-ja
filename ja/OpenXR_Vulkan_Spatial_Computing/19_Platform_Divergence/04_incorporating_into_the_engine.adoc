= プラットフォーム分岐のエンジンへの組み込み

**プラットフォーム分岐** を統合すると、`simple_game_engine` はハイエンドデスクトップと低消費電力モバイルハードウェアの両方向けに空間パイプラインを自動最適化できます。`renderer_core.cpp` の `Renderer` に構成レイヤを追加して実現します。*現時点では、実装は読者への演習として残されています。*

== ハードウェアプロファイルの定義

`renderer_core.cpp` では、検出されたハードウェアに基づきレンダリングプリセットを適用するメソッドを導入します。このロジックにより、エンジンがプラットフォームの制約に合わせて忠実度をスケールすることが保証されます。

[source,cpp]
----
// renderer_core.cpp
void Renderer::applySpatialPlatformPresets() {
    // Note: platform is our Engine's platform abstraction
    if (platform->isMobile()) {
        // 1. Mobile Mastery: Optimize for Tile-Based GPUs
        renderScale = 0.75f; // Lower initial resolution for stability

        // 2. Explicitly set Load/Store ops for tile memory efficiency
        // Discarding depth/multisample data saves massive bandwidth
        mainColorAttachment.storeOp = vk::AttachmentStoreOp::eDontCare;
        mainDepthAttachment.storeOp = vk::AttachmentStoreOp::eDontCare;

        // 3. Enable Application SpaceWarp for synthetic frame generation
        xrConfig.enableASW = true;
    } else {
        // 4. Desktop High-End: Maximum Fidelity
        renderScale = 1.0f;
        useHDR10 = true; // Use 10-bit color formats

        // Enable high-bandwidth features like Ray Query
        vk::PhysicalDeviceRayQueryFeaturesKHR rayQueryFeatures{};
        // ... request features from the LUID-matched physical device ...
    }
}
----

== デスクトップでの Direct Mode の扱い

デスクトップヘッドセットでは、エンジンのウィンドウ系（GLFW）がヘッドセットの **Direct Mode** 表示を妨げないことを保証する必要があります。`main.cpp` でウィンドウ作成ロジックを構成します。

[source,cpp]
----
// main.cpp
void setupEngineWindow() {
    if (engine.isXRMode()) {
        // On Desktop, the headset is a "hidden" display.
        // We create a small "Mirror Window" for the monitor
        // while the main spatial rendering bypasses the OS compositor.
        glfwWindowHint(GLFW_VISIBLE, GLFW_FALSE);
        engine.createMirrorWindow("XR Mirror", 1280, 720);
    } else {
        engine.createStandardWindow("Simple Engine", 1920, 1080);
    }
}
----

== なぜこれらの変更が必要か？

プラットフォーム固有プリセットと Direct Mode 処理を実装することで、ハイエンドデスクトップ GPU でもモバイル SOC でも、エンジンが性能を保つことが保証されます。`simple_game_engine` はすでにモジュール化された `Platform` 抽象を利用しており、これを活用してデスクトップ固有ロジック（10 ビット HDR など）とモバイル固有最適化（**Tile-Based** 破棄など）を切り替えます。

このアーキテクチャ的柔軟性こそが、コアレンダリングロジックを書き直さずに、最高忠実度から最も効率的なモバイルスタンドアロン性能まで空間アプリケーションをスケールさせます。
