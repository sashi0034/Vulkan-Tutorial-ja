= デスクトップハイエンド: GPU の解放

ハイエンドデスクトッププラットフォームでは、空間コンピューティングエンジンは電池寿命や厳しい熱制限にあまり束縛されません。これにより絶対的な視覚的忠実度に集中でき、モダン GPU の全力を使い高解像度ヘッドセットを駆動できます。デスクトップでの目標は **飽和（Saturation）**——GPU のメモリバスとコンピュートユニットをできるだけ満たし、ハードウェアが達成できる最も没入感のある体験を届けることです。

== 概念: Direct Mode

標準アプリケーションでは、Vulkan ウィンドウは OS のウィンドウマネージャ（Windows の **DWM** など）によって管理されます。OS がウィンドウを他のデスクトップ要素と合成するため、レイテンシの層が追加されます。空間コンピューティングでは、この余分なレイテンシは許容できません。

**Direct Mode** を使います。これにより XR ランタイムが表示出力の排他制御を取り、OS デスクトップコンポジタを完全に迂回します。

*   **レイテンシの利点**: OS コンポジタを除くことで、フレーム配信時間を 1〜2 フレーム短縮し、**モーション・ツー・フォトン** の安定性に不可欠。
*   **ストロボ同期**: Direct Mode により完全な「ディスプレイストロボ」が可能になる。ヘッドセットのバックライトが GPU のスワップ信号と正確に同期して点滅し、急速な頭の動き中のモーションブラーを除去する。

== HDR10 と 10 ビット色: 画面の先へ

モバイルプラットフォームは優れた HDR サポートでしばしば先行しますが、ハイエンドデスクトップ GPU では、はるかに高い内部解像度とビット深度で **High Dynamic Range（HDR）** を活用できます。XR では物理的リアリズムに不可欠です。仮想の太陽は仮想のろうそくの何千倍も明るくあるべきです。HDR がなければ、エンジンはこれらの値を同じ狭い範囲へ「トーンマップ」せざるを得ず、スケール感が失われます。

実装のため、`VulkanContext` は OpenXR ランタイムと 10 ビット色フォーマット——`VK_FORMAT_A2B10G10R10_UNORM_PACK32` など——を交渉します。これにより 10 億色超が得られ、夜のシーンや洞窟のような暗い空間環境での「バンディング」アーティファクトを除去します。

[source,cpp]
----
// Configuring HDR10 for high-end desktop XR using designated initializers
vk::SwapchainCreateInfoKHR createInfo{
    .imageFormat = vk::Format::eA2b10g10r10UnormPack32,
    .imageColorSpace = vk::ColorSpaceKHR::eHdr10St2084EXT,
    // ... other parameters ...
};
----

== 高 PCIe 帯域の活用

**PCIe 4.0** または **5.0** リンクにより、CPU と GPU 間の高速データ転送が可能で、複雑な空間アセットのストリーミングに不可欠です。しかし XR に必要な高フレームレートでは、慎重に管理しないとこれらのリンクさえボトルネックになり得ます——特に PCIe レーンが限られたシステムで。

*   **非圧縮ライトフィールド**: レンダーループを止めずにギガバイト級のプレノプティックデータを GPU へ直接アップロードできる。
*   **レイトレーシング**: 複雑でハイポリな環境向けに毎フレーム **Acceleration Structures（AS）** を構築する高いオーバーヘッドを許容でき、リアルタイムの空間反射と大域照明が可能になる。

デスクトップでは、空間コンピューティングの心構えは、ユーザーと仮想世界の境界を溶かすために、あらゆるワットとあらゆるギガバイトの帯域を使い切ることです。
