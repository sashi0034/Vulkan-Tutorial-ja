= エンジンへの組み込み

レイトラッチングは、GPU 投入直前の可能な限り最後の瞬間にビューと投影行列を更新することで「スウィム」を最小化する強力な最適化です。これを `simple_game_engine` に実装するには、`renderer_rendering.cpp` の `Render` ループを修正します。実装例は link:../../../attachments/openxr_engine/renderer_rendering.cpp[renderer_rendering.cpp] にあります。

== 後期更新の実装

`renderer_rendering.cpp` では、最終的な UBO 更新を `vkCmdDraw` 呼び出しの直前まで遅らせる必要があります。これには、最後の瞬間の行列が準備できるまでコマンドバッファ投入をゲートするため **タイムラインセマフォ** を使います。

[source,cpp]
----
// renderer_rendering.cpp
void Renderer::Render(...) {
    // ...
    if (xrMode) {
        // Retrieve the ABSOLUTE LATEST view poses from OpenXR
        // Note: This happens AFTER we've already waited on the frame fence
        auto latestPoses = xrContext.getLatestViewPoses();

        // Directly update the mapped UBO memory for each eye
        // Since we use persistent mapping (vk::MemoryPropertyFlagBits::eHostVisible),
        // we can write directly to the GPU-resident buffers.
        for (uint32_t eye = 0; eye < 2; ++eye) {
            updateUniformBuffer(currentFrame, eye, &latestPoses[eye]);
        }

        // Use Timeline Semaphores to signal that the late update is finished
        vk::TimelineSemaphoreSubmitInfo timelineInfo{
            .signalSemaphoreValueCount = 1,
            .pSignalSemaphoreValues = &currentFrameValue
        };

        // Gate the graphics queue submission
        // The GPU will wait for 'currentFrameValue' before executing
        graphicsQueue.submit(submitInfo);
    }
}
----

== 低レイテンシ向け永続マッピング

`Renderer::createUniformBuffers` では、毎フレームのマッピングオーバーヘッドを避けるため、バッファを一度マップしてレンダラーの寿命中開いたままに保ちます。

[source,cpp]
----
// renderer_rendering.cpp
bool Renderer::createUniformBuffers(Entity* entity) {
    // ... create vk::raii::Buffer ...

    // NEW: Persistently map the buffer for last-second updates
    // entityRes.uniformBuffersMapped is a void* array for each frame-in-flight
    entityRes.uniformBuffersMapped[i] = entityRes.uniformBuffersAllocation[i]->map(0, VK_WHOLE_SIZE);
}
----

== なぜこれらの変更か？

`updateUniformBuffer` 呼び出しを `Render` 関数の末尾（投入直前）へ移すことで、GPU が可能な限り正確なヘッドトラッキングデータを受け取ることが保証されます。これにより、フレームを記録した時点からヘッドセットのディスプレイへ実際にスキャンアウトされるまでの「知覚レイテンシ」（Motion-to-Photon）が削減されます。
