= レイトラッチングとタイムラインセマフォ

空間コンピューティングでは、「頭はここにあると思う」と「ピクセルが実際にディスプレイに描かれる」の間のギャップを **Motion-to-Photon（M2P）** レイテンシと呼びます。前章で探った予測フレームループがあっても、シミュレーション開始後から GPU が描画を終える前までの間にユーザーの頭が動ける時間窓が残ります。古い頭のポーズを使うと、仮想世界がユーザーの実際の動きに対して「スウィム」したり遅れたりして感じられます。

ここで **レイトラッチング** が登場します。フレーム開始時に一度だけ頭のポーズを取得してすべてに使うのではなく、人間が可能な限り遅く——理想的には、それに依存するドローコールの GPU 実行開始直前に——最新のポーズに「ラッチ」したいのです。

Vulkan 1.3 で大きな CPU ストールを起こさずにこれを達成するため、**タイムラインセマフォ** を活用します。従来、CPU と GPU の同期には `vkDeviceWaitIdle` や、複雑なフレーム間で管理しにくいバイナリセマフォといった「重い」操作が必要でした。タイムラインセマフォは、CPU と GPU の両方がウェイトまたはシグナルできる、単調増加する 64 ビットカウンタを作れます。

Uniform Buffer Object（UBO）またはプッシュコンスタントの更新をタイムラインセマフォ値の後ろにゲートすることで、CPU が絶対最新のビューと投影行列の書き込みを終えた後にのみ GPU がドローコールを進めることを保証できます。これにより予測誤差が最小化され、仮想世界がユーザーの現実にしっかり固定されたままになります。

この章では次を見ます。

1. タイムラインセマフォを使い CPU／GPU ハンドシェイクをオーケストレーションする。
2. 投入直前の「最後のマイクロ秒」でシェーダーデータを更新する。
3. これらの後期更新をエンジンのコマンドバッファフローに統合する。
